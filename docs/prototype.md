# 我的城市探秘低保真原型

本目录交付一组可直接打开的 HTML 原型，用于评审「我的城市探秘」微信小程序的用户首探闭环和小程序内超级管理员交付链路。

## 打开方式

- 原型目录入口：`docs/prototype/city-quest-lofi/index.html`
- 每个页面都是独立 HTML，可直接在浏览器中打开。
- 页面状态通过 query 参数和 `localStorage` 保留，不依赖接口、构建工具或网络。

## 原型范围

用户端主流程：

1. 用户收到线下交付的探秘入口码。
2. 进入会员欢迎页，点击「开始探索」后成为有效会员。
3. 进入首页地图，查看公开百科和会员宝箱热区。
4. 扫实体宝箱码，校验会员、定位和领取限制。
5. 加入背包或立即打开，查看奖励入账。
6. 在背包、图鉴、副本和我的页面查看后续资产与进度。

管理员端主流程：

1. 管理工作台进入高频任务。
2. 投放宝箱、开通或续期会员、售卖解密地图、组合售卖、发放道具。
3. 管理员提交前看到确认弹窗。
4. 成功后展示入口小程序码，管理员转交给用户。

## 页面清单

用户端：

- `welcome.html`：会员、解密地图、VIP+地图组合三类欢迎页。正常入口码链路会进入固定欢迎页；直接打开时随机模拟入口类型和状态，需要固定评审时可追加 `?type=membership|map|bundle&code=valid|done|invalid`。
- `home.html`：首页地图、百科图层、宝箱热区、扫码入口。
- `explore.html`：宝箱、百科、副本、线索分段列表。
- `encyclopedia.html`：公开百科详情、收藏、分享。
- `scan-result.html`：扫码成功、定位、范围、已领取、已领完、无效 token 和会员阻断。
  正常从首页进入时随机模拟一种扫码结果；需要固定评审场景时可追加 `?scenario=success|location|range|claimed|empty|invalid`。
- `box-open.html`：开箱扫描、奖励揭示、入账反馈。
- `inventory.html`：宝箱、道具、线索、空箱记录。
- `cards.html`：图鉴系列进度、已获得和未获得剪影。
- `quest.html`：副本线索时间线、答案提交、罗盘入口。
- `profile.html`：用户码、会员状态、管理员入口。

管理员端：

- `admin-workbench.html`：小程序内超级管理员工作台。
- `admin-deploy-chest.html`：投放实体宝箱。
- `admin-open-membership.html`：开通或续期会员。
- `admin-sell-map.html`：售卖解密地图。
- `admin-bundle.html`：VIP+解密地图组合售卖。
- `admin-grant-prop.html`：发放道具。
- `admin-welcome-code.html`：入口码交付页。

## 状态覆盖

页面左侧控制台可切换以下状态：

- 未登录：只显示公开百科相关页面；我的页显示登录入口，不显示用户码或收藏列表；收藏时提示登录。
- 普通用户：只显示公开百科相关页面，可收藏和分享公开百科。
- 待激活会员：可进入本人欢迎页，未开始前不能使用会员能力。
- 有效会员：可查看热力图、扫码领取、开箱、使用罗盘和推进副本。
- 过期会员：历史资产可见，会员动作只读或阻断。
- 超级管理员：可进入小程序管理工作台。

可见性规则：

- 未登录、普通用户只显示首页、探索、公开百科详情和我的；欢迎、扫码、开箱、图鉴、背包、副本及管理页都不暴露入口或页面标题。
- 未登录和普通用户的「我的」页分开处理：未登录只显示登录入口和设置；普通用户显示用户码、收藏百科和设置。
- 待激活会员只额外显示本人欢迎页；点击开始前不显示会员能力入口。
- 未登录、普通用户、待激活会员只显示公开百科相关入口；宝箱热区、宝箱分段、副本分段、线索分段、图鉴和背包入口均隐藏。
- 过期会员可查看历史图鉴、背包、历史副本和历史线索，但不显示宝箱热区和宝箱分段。

## 原型规则与评审说明

这些说明不进入手机画板，只用于评审和实现对齐。

欢迎入口：

- 欢迎页只承接线下已完成的权益，不展示价格、不展示线上开通或支付入口。
- 会员欢迎码点击「开始探索」后才激活会员，到期时间从点击成功时起算。
- 解密地图欢迎码点击「开始解密」时再次校验有效会员。
- VIP+解密地图组合码点击「开始探索」后同时完成会员激活和地图开始。
- 已完成欢迎码本人复扫时展示「继续探索」，不重复激活会员、不重复开始地图。
- 无效、过期、作废或非归属欢迎码统一提示「无法进入该探秘入口」，随后返回首页。

首页与探索：

- 首页默认展示公开百科地图；未登录、普通用户和待激活会员不显示宝箱热区图层切换。
- 有效会员可切换宝箱热区，热区只展示漂移坐标。
- 过期会员可看到过期提醒，但不显示宝箱热区入口。
- 探索页分段按状态生成；未登录、普通用户和待激活会员只显示百科分段。
- 未登录、普通用户和待激活会员只展示公开百科；需解锁百科不出现在列表或详情直达结果中。
- 需解锁百科、宝箱、副本、线索、图鉴和背包不向未登录或普通用户暴露入口。

扫码结果：

- 原型中未登录、普通用户和待激活会员直接打开扫码结果页时回到公开百科兜底页，不展示扫码、宝箱或领取信息。
- 过期会员扫码实体宝箱时展示续期方向和用户码，不展示宝箱详情。
- 有效会员扫码成功后可「加入背包」或「立即打开」。
- 定位未授权、超出范围、已领取、已领完、无效 token 都不领取宝箱，也不消耗领取次数。
- 原型从首页进入扫码页时随机模拟一种结果；固定评审时使用 `scenario` 参数。

资产与玩法：

- 有效会员可打开背包中的未打开宝箱；领取后可在任意位置打开。
- 会员过期后保留历史背包、图鉴、线索和副本进度，但不能领取、开箱或使用罗盘。
- 图鉴列表中已获得项显示卡面和具体标题，未获得项显示剪影；列表不额外显示「已获得/未获得」标签。
- 图鉴详情展示卡面、所属系列、等级、系列进度、编号、获得来源和获得时间。
- 寻宝罗盘一次消耗 1 个；对同一对象已解锁后再次查看不重复消耗。
- 真实坐标、精确指引和投放照片只在罗盘解锁后显示。

管理端：

- 管理端可出现实收金额，用户端不可出现价格或支付。
- 开通会员、售卖解密地图、组合售卖和发放道具都需要确认弹窗。
- 会员开通、解密地图售卖和组合售卖成功后展示小程序码，由管理员转交用户。
- 单独售卖解密地图要求目标用户为有效会员；非有效会员应使用 VIP+解密地图组合。
- 管理员创建临时用户后，用户首次扫码欢迎码并登录时绑定对应权益。

## 视觉原则

- 低保真线框为主，但保留「城市刻痕地图」识别。
- 只在首页地图、扫码结果和开箱页使用刻痕/扫描线。
- 用户端不展示价格、不出现线上支付、不出现购买会员文案。
- 地图宝箱只展示漂移坐标；真实坐标仅在罗盘解锁弹窗中出现。
- 手机画板只放真实用户或管理员会看到的界面文案；业务规则、评审说明和状态解释放在本文档或设计文档里。

## 评审检查点

- 从 `index.html` 能进入管理工作台，并完成开通会员到入口码交付。
- 从入口码能进入用户欢迎页，并点击开始探索到首页。
- 有效会员能从首页进入扫码结果，并完成开箱。
- 普通、待激活、过期状态均能触发对应阻断。
- 管理端金额只出现在管理页面。
- HTML、CSS、JS 均为本地相对路径，无外部依赖。
