## 1. 摘要

C 端采用 **平台内核（core）+ 特性模块（features）+ 共享层（shared）**。

- 依赖方向遵循整洁架构：Presentation → Application → Domain；Infrastructure 实现 Domain 端口并依赖 core。
- Domain / Application **不依赖** `wx.*` 与 Page。
- 首发业务（百科地图闭环）是第一个垂直切片，用于交付产品与验证分层；**不定义**架构上限。
- 新业务能力 = 新 `features/<name>/` + 可选分包 + 在组装根注册；**不修改** core 公共语义；**不**为未立项能力预建空页面或空 feature 目录。

组装入口：

- 运行时：`miniprogram/app.ts` + `miniprogram/app-context.ts`
- 路由：`miniprogram/app.json`

## 2. 架构目标

| 目标 | 度量 |
|------|------|
| 可扩展 | 新业务 = 新 feature（+ 可选 subpackage），不改 core 公共 API |
| 可替换 | http / storage / session / state / map 的**实现**可替换；调用方依赖端口或门面 |
| 可测 | domain、application、部分 core 纯逻辑可在 Node 侧单测；domain 无 `wx.*` |
| 可运行 | 首发可交付 PRD 百科闭环；主包体积可控（微信单包 ≤ 2MB） |
| 可演进 | 状态库、UI 库、npm 工具有明确接入契约，可局部升级 |

**非目标**

- 实现未立项业务
- 在 `app.json` 注册未使用的「未来页面」
- 重开全栈选型（Workers / Hono / Neon / R2 等以技术方案为准）
- 为假想跨端引入 Taro / uni-app

## 3. 软件设计原则（强制）

### 3.1 SOLID

| 原则 | 约束 |
|------|------|
| **S** 单一职责 | 一文件一职责；Page 不做 HTTP；Infrastructure / Repository 不做 Toast；Storage 不做业务规则 |
| **O** 开闭 | 新能力通过新模块 + 注册扩展；禁止在 core 内堆积 `if (feature === '…')` |
| **L** 里氏替换 | Repository / Gateway 实现可替换；调用方只依赖接口形状 |
| **I** 接口隔离 | Feature 仅经 `public.ts` 外露；禁止跨 feature deep import 内部文件 |
| **D** 依赖倒置 | Domain / Application 依赖端口（interface）；由 Infrastructure 与 core 提供实现 |

### 3.2 分层

```text
┌──────────────────────────────────────────────────────────┐
│  Presentation：pages / components / bindings             │
├──────────────────────────────────────────────────────────┤
│  Application：use cases（features/*/application）        │
├──────────────────────────────────────────────────────────┤
│  Domain：entities / pure rules / ports                   │
├──────────────────────────────────────────────────────────┤
│  Infrastructure：repository 实现、wx 适配                │
│  Core：http / session / storage / state / map / nav …    │
└──────────────────────────────────────────────────────────┘
依赖方向：Presentation → Application → Domain
          Infrastructure → Domain ports + Core
          Core 不依赖任何 feature
```

### 3.3 其它规范

| 规范 | 要求 |
|------|------|
| 高内聚低耦合 | 按**业务能力**切 feature，禁止无界全局 `utils` / `common` 垃圾场 |
| DRY | 跨 feature 复用进入 `shared` 或 `core`；feature 私有逻辑不随意上提 |
| YAGNI（实现） | 不写未使用的抽象类、空分包页；扩展能力用**端口与目录纪律**表达 |
| KISS | 端口 = TypeScript `interface` + 工厂函数；不上 IoC 容器框架 |
| 不可变 | 状态更新返回新对象；禁止直接修改从 storage 读出的对象 |
| 显式边界 | HTTP、Storage、用户输入处校验/窄化；错误类型显式 |
| 职责分离 | 传输层抛错；用例决定是否降级；UI 决定 Toast / 文案 |

### 3.4 反模式黑名单

1. Page 内直接 `wx.request` 或散落 baseURL  
2. 巨型无界 `utils.ts` / `common.ts`  
3. Feature 互相 deep import 内部实现  
4. Domain 引用 `wx` 或组件  
5. 用无类型 `getApp().globalData` 充当状态总线  
6. 为未立项业务注册空页面  
7. 登录成功后自动续办写操作（违反 PRD；默认不重放）  
8. Secret（如 `WECHAT_SECRET`、`JWT_SECRET`）进入小程序包  
9. 非 GCJ-02 坐标直接交给 `<map>`  
10. 无门面隔离即引入重型全局状态库或 UI 全家桶  

### 3.5 Code Review 清单

- 依赖方向是否被破坏？  
- 是否出现跨 feature 内部引用？  
- 新逻辑是否应下沉到 use case / domain，而非堆在 Page？  
- 是否引入无门面的全局可变状态？  
- 页面是否落在正确的主包 / 分包？  
- 是否把业务规则写进 storage 或 http 层？

## 4. 架构风格

小程序 = **App Shell** + **Feature Modules**。

```text
                    ┌──────────────────┐
                    │ app.ts           │  hydrate、启动
                    │ app-context.ts   │  composition root
                    │ app.json         │  已启用页面 / 分包
                    └────────┬─────────┘
           ┌─────────────────┼─────────────────┐
           ▼                 ▼                 ▼
     ┌──────────┐     ┌──────────┐     ┌──────────────┐
     │  core/   │     │ shared/  │     │ features/*   │
     │ 内核     │     │ 通用层   │     │ 业务模块     │
     └──────────┘     └──────────┘     └──────────────┘
```

| 区域 | 职责 | 可否依赖 feature |
|------|------|------------------|
| **core** | http、session、storage、state、map、config、navigation、error | **否** |
| **shared** | 与具体业务无关的 UI 组件与纯工具 | **否** |
| **features/\<name\>** | 单一业务能力的 domain + application + infrastructure + presentation | 仅可依赖他 feature 的 **public.ts** |
| **组装根** | 创建依赖、注入 use case、暴露 `getAppContext()` | 可依赖各 feature 的 public |

### 4.1 扩展标准路径

1. 新建 `features/<name>/`，按 §6.1 模板分层  
2. 导出 `public.ts`（用例工厂、类型、路由常量）  
3. 若包体需要：独立 subpackage，页面仅属该能力  
4. 在 `app.json` 注册页面；在 `app-context.ts` 组装依赖  
5. 不修改其它 feature 内部；协作时优先抽 **core 端口** 或 **shared 契约**，其次对方 `public.ts`  

### 4.2 首发启用模块

| Feature | 职责 |
|---------|------|
| `encyclopedia` | 百科点位、详情、本地浏览历史、收藏 API |
| `account` | 登录 / 退出、ensureAuthenticated、协议与账户相关路由 |

不预建其它 feature 目录；模块名是边界划分，不是产品路线图列表。

## 5. 技术栈

| 层 | 选型 | 说明 | 升级触发 |
|----|------|------|----------|
| 语言 | TypeScript strict | `tsconfig.json` | — |
| 运行时 | 微信原生小程序，基础库 3.x+ | 地图 / 登录等为平台能力 | 跟官方 |
| 组件框架 | glass-easel + Component | `app.json`：`componentFramework` | — |
| 代码注入 | `lazyCodeLoading: requiredComponents` | 减启动注入 | — |
| 渲染 | 默认 WebView；Skyline 可配置 | 关键路径真机门禁后再开 | 真机矩阵通过 |
| 样式 | WXSS + `app.wxss` CSS 变量 | 设计令牌见 UIUX | 设计系统膨胀再工具化 |
| 状态 | Store 端口 + 内存实现 | `core/state` | 跨 feature 订阅成本过高 → 换实现，端口不变 |
| 网络 | `core/http` 封装 `wx.request` | 统一 envelope / 鉴权 / 错误 | — |
| 持久化 | `core/storage` 版本化 key | `cq/v1/...` | — |
| 地图 | `core/map` 视图模型 + viewport | 业务产出 MarkerVm | 聚合 / 视野加载时换策略或适配 |
| 路由 | `app.json` + feature 路由常量 | 分包按能力切 | 重模块独立包 |
| 校验 | 边界 type guard | 默认不上小程序 Zod | 复杂表单 feature 可局部引入 |
| 测试 | Vitest | `tests/**` 测纯模块 | — |
| 类型检查 | `tsc --noEmit` | `npm run typecheck` | — |
| 包管理 | npm | 默认零业务运行时依赖 | feature 显式引入 |
| 多环境 | `core/config/env.ts` | 按 `envVersion` 映射 API | 正式域名发布时配置 |

### 5.1 第三方接入契约

| 接入 | 契约 |
|------|------|
| 状态库 | 只替换 `core/state`（或具体 store）的**实现**；对外保持 `getState` / `setState(reducer)` / `subscribe` |
| UI 组件库 | 进入 `shared/ui` 或 feature 的 presentation；用业务组件包装第三方；Page 不直接铺满第三方标签 |
| 工具 npm | 经 `core` 或 `shared` 再导出；禁止 Page 直接依赖一长串无封装包 |

### 5.2 工程命令

```bash
cd city-quest-wxmp
npm install
npm run typecheck
npm test
```

微信开发者工具打开 `city-quest-wxmp` 目录（`project.config.json` 中 `miniprogramRoot` 为 `miniprogram/`）。

### 6.2 Feature 内部模板

```text
features/<name>/
  domain/
    entities.ts
    ports.ts
    rules/*.ts          # 纯函数，可单测
  application/
    *.ts                # use case 工厂：依赖 ports，返回可调用函数
  infrastructure/
    *-repository.ts     # 实现 ports；调用 core/http 或 storage
  presentation/         # 可选：复杂 UI 下沉；简单页可薄挂在 pages/
  public.ts             # 唯一外露 API
```

**依赖规则**

| 层 | 允许依赖 |
|----|----------|
| domain | 无 `wx`；仅类型与纯逻辑（可读 core 的**常量类型**时需保持可测） |
| application | domain + ports |
| infrastructure | domain ports + core |
| presentation / pages | application（经 public 或 context）、shared、core 的 navigation/error 等展示辅助 |
| 其它 feature | **仅** `features/<x>/public.ts` |

### 6.3 分包策略

| 包 | 内容 | 理由 |
|----|------|------|
| 主包 | `pages/map`、`pages/mine`、`pages/detail` + core + shared + 启动所需代码 | Tab 与分享冷启动详情 |
| `package-account` | history、favorites、profile、about、agreement、privacy | 低频路径，按需下载 |
| 新增能力 | 按 feature 边界增 subpackage | 不预注册空包 |

`app.json` 中 `preloadRule`：进入 `pages/map` 或 `pages/detail` 时可预下载 `account` 分包。

详情页留在主包：分享路径冷启动不依赖分包下载。

### 6.4 核心文件职责

| 路径 | 职责 |
|------|------|
| `miniprogram/app-context.ts` | 创建 kv、session、http、各 repository 与 use case；`getAppContext()` |

| `miniprogram/core/http/client.ts` | 唯一 API 请求出口；解包 envelope；401 清 session |

## 7. 核心子系统

### 7.1 HTTP（`core/http`）

| 项 | 约定 |
|----|------|
| 职责 | 唯一出站 API 通道 |
| 信封 | `{ success, data, error, meta }`，与技术方案 / server 对齐 |
| 错误 | `HttpError { status, code, message }`（`errors.ts`） |
| 鉴权 | `auth: true` 时注入 `Authorization: Bearer <token>` |
| 401 | `session.clear()`；**不**在 http 层弹登录或跳转 |
| 重试 | 写操作不重试；读操作默认不重试（可按需扩展幂等读） |
| 超时 | `HTTP_TIMEOUT_MS`（constants） |
| 资产 | `requestAsset` 拉静态 JSON（如 `/config/encyclopedia-types.json`），不按信封解包 |

页面与 domain **禁止**直接 `wx.request`。

### 7.2 Session（`core/session`）

| 项 | 约定 |
|----|------|
| 端口 | `hydrate` / `getToken` / `getUser` / `isLoggedIn` / `setSession` / `clear` / `subscribe` / `getSnapshot` |
| 持久化 | `cq/v1/token`、`cq/v1/user` |
| 登录 UI | `features/account` |
| 产品规则 | `ensureAuthenticated` 引导登录后**默认不自动重放**原写操作 |

### 7.3 Storage（`core/storage`）

| 项 | 约定 |
|----|------|
| 适配 | `KvStorage`：`getJson` / `setJson` / `remove` |
| 实现 | `createWxKvStorage()`；测试可用 `createMemoryKvStorage()` |
| Key | `StorageKeys`，前缀 `cq/v1/` |
| 业务规则 | 不在 storage 层；例如浏览历史 MRU 在 encyclopedia domain rules |

### 7.4 State（`core/state`）

```ts
interface Store<S> {
  getState(): S
  setState(reducer: (prev: S) => S): void
  subscribe(listener: () => void): () => void
}
```

- 默认：`createStore` 内存实现  
- 替换第三方状态库时：实现同一端口或保持 feature facade，不改调用方语义  

### 7.5 Map（`core/map`）

```ts
interface MapMarkerVm {
  id: string
  lng: number  // GCJ-02
  lat: number
  styleKey: string
  selected?: boolean
  title?: string
}

interface MapViewport {
  includePoints?: Array<{ lng: number; lat: number }>
  center?: { lng: number; lat: number }
  scale?: number
}
```

- core 只提供与玩法无关的视图模型与 `viewportFromMarkers`  
- 具体「数据 → MarkerVm[]」由 feature domain/application 完成（如 `encyclopedia` 的 `toMapMarkers`）  
- 默认中心：大理 GCJ-02（`DEFAULT_MAP_CENTER` / `DEFAULT_MAP_SCALE`）  

### 7.6 Config（`core/config`）

| 文件 | 职责 |
|------|------|
| `env.ts` | `getEnv` / `apiUrl` / `assetUrl`；develop / trial / release |
| `constants.ts` | 超时、历史上限、默认地图中心、`STORAGE_NS` |
| `capabilities.ts` | 能力开关读取与合并；默认可扩展位关闭 |

禁止将个人局域网 IP 作为仓库默认提交值；develop 默认 `http://127.0.0.1:8787`，trial/release 配置正式 API 域名。

### 7.7 Navigation（`core/navigation`）

- `navigateTo` / `navigateBack` / `switchTab`  
- 登录为页内授权弹窗，**不再**使用 `returnUrl` 跳转独立登录页

### 7.8 Error（`core/error`）

- `messageForCode` / `messageFromUnknown`：code → 用户文案  
- **禁止**在 infrastructure / http 内 `wx.showToast`  

### 7.9 Shared（`shared/lib`）

| 模块 | 职责 |
|------|------|
| `load-seq.ts` | 异步竞态序号，避免过期 `setData` |
| `result.ts` | 可选 `Result` 类型糖 |

## 8. 首发 Feature

### 8.1 encyclopedia

| 层 | 内容 |
|----|------|
| domain/entities | 列表项、详情、类型、浏览记录、收藏列表 DTO |
| domain/ports | `EncyclopediaRepository` / `FavoriteRepository` / `BrowseHistoryRepository` |
| domain/rules | `upsertBrowseHistory`（MRU、去重、上限 200）；`toMapMarkers`（纯 VM，无 wx）；`normalize-list`（列表边界窄化） |
| domain | `type-defaults`（类型默认单源，供 type-label / fallback 共用） |
| application | `loadMapPoints`（支持 `forceRefresh`）、`openDetail`、`list/add/removeFavorite` |
| infrastructure | HTTP repository、**TTL 缓存装饰器** `cached-encyclopedia-repository`、本地 browse-history、类型 fallback |
| presentation | `type-label`；`wx-markers`；`marker-icon-style`（纯几何/缓存键）；`marker-icon-service`（Canvas 生成 PNG + 缓存）；`marker-label-style`（右侧常显名称，纯样式） |
| public.ts | 对外导出工厂、类型、`EncyclopediaRoutes`、marker 适配、图标服务、缓存装饰器 |

主要用例：

| 用例 | 行为 |
|------|------|
| 加载地图点位 | 拉 published 列表 + 类型配置 → `MapMarkerVm` + viewport；可选 `forceRefresh` 失效仓储 TTL |
| 渲染地图 marker | 按类型 `color` Canvas 画圆点 → 用户目录 PNG → `iconPath`；按颜色缓存（含选中态预热）；`toWxMapMarkers` 挂载右侧常显 `label`（透明底、ink 字；选中仅加大字号） |
| 打开详情 | 拉详情并写入本地浏览历史 |
| 收藏 | 调用 `/api/v1/me/favorites*`（调用方须先 ensureAuthenticated） |

**列表 / 类型缓存（infrastructure 装饰器，§9）**

| 项 | 约定 |
|----|------|
| 形状 | `createCachedEncyclopediaRepository(inner)`，调用方仍依赖 `EncyclopediaRepository` |
| 列表 TTL | 默认 60s（`ENCYCLOPEDIA_LIST_TTL_MS`） |
| 类型 TTL | 默认 1h（`ENCYCLOPEDIA_TYPES_TTL_MS`） |
| 并发 | 同 key inflight 合并；返回浅拷贝，禁止污染缓存 |
| 失效 | `invalidateCache('list' \| 'types' \| 'all')`；`loadMapPoints({ forceRefresh: true })` 用于用户重试 |
| 详情 | `getById` **不**经此缓存（`isFavorited` 随登录态变化） |

**地图 marker 图标（presentation）**

| 项 | 约定 |
|----|------|
| 平台约束 | 微信 `<map>` 的 `iconPath` **不支持 SVG**，须为本地/临时图片路径 |
| 视觉 | 实心圆 + 白描边；填充色 = 类型 `color`（与 UIUX 一致）；选中略放大 |
| 生成 | `createOffscreenCanvas('2d')` 绘制 → `canvasToTempFilePath` → `USER_DATA_PATH/cq-markers/{color}_{0\|1}.png` |
| 缓存 | 内存 key：`#rrggbb\|0/1`；磁盘文件命中则跳过重绘；失败回退 `assets/markers/default.png`（**不**永久缓存失败结果） |
| 扩展 | 新增/变更类型只需配置 `key`+`color`，**无需**再发版打包分类 PNG |
| 名称 label | 关闭系统 POI 后靠右侧常显名辨识；`marker-label-style` 生成微信 `label`（无底色）；`bindmarkertap` / `bindlabeltap` 共用选中逻辑 |
| 页面流 | `pages/map`：**两阶段**渲染——先用已缓存 path / fallback `setData(markers)`，再 `ensureAll` 后 patch；点选用 `patchMarkersSelection` 仅 path-setData 变更的 1–2 个 marker；`createMarkerIdMap` 保证 numeric id 稳定；圆点 `anchor` 中心居中放大 |

路由常量：

- 主包：`/pages/map/map`、`/pages/detail/detail`（`EncyclopediaRoutes`）  
- 分包：`/package-account/pages/history/history`、`.../favorites/favorites`、`.../profile/profile`（`AccountRoutes.profile`）

### 8.2 account

| 层 | 内容 |
|----|------|
| domain/ports | `AuthRepository`、`ProfileRepository` |
| application | `login`、`logout`、`ensureAuthenticated`、`loadProfile`、`updateProfile`（含 `isValidPhone`） |
| infrastructure | `POST /api/v1/auth/wechat/login`；`GET/POST /api/v1/me`；`POST /api/v1/me/avatar`（`wx.uploadFile`） |
| routes / public | `AccountRoutes`（含 `profile`）、对外工厂 |

`ensureAuthenticated`：

1. 已登录 → `true`  
2. 未登录 → `wx.showModal`「授权登录」（允许 / 取消）  
3. 用户允许 → `wx.showLoading('登录中')` → `login()`（仅 `wx.login` code，不上传头像/昵称）→ `hideLoading` → toast 结果  
4. 取消、失败或**刚登录成功**均返回 `false`，留在当前页  
5. **不**在登录成功后自动执行原写操作（调用方需用户再次点击）

资料编辑（`updateProfile`）：

1. 客户端校验昵称非空 ≤64、手机号空或 `^1\d{10}$`  
2. 若有本地新头像路径 → `uploadAvatar` → 得 `avatarKey`  
3. `POST /me` 提交 `nickname` / `phone` / `avatarKey?`  
4. 成功后 `session.setSession(token, user)` 同步本地  

### 8.3 组装（`app-context.ts`）

`AppContext` 对外提供（页面只依赖 context 或 public，不 new repository）：

- `session`  
- `loadMapPoints` / `openDetail`  
- `listFavorites` / `addFavorite` / `removeFavorite`  
- `listBrowseHistory` / `clearBrowseHistory`  
- `ensureAuthenticated` / `login` / `logout`  
- `loadProfile` / `updateProfile`  
- `getEncyclopediaTypes` / `setEncyclopediaTypes`（map 预热类型，detail 共用）  

组装：`createEncyclopediaRepository` → `createCachedEncyclopediaRepository` → `createLoadMapPoints(repo, { invalidateCache })`；`createProfileRepository(http, session)` → `createLoadProfile` / `createUpdateProfile`。  

### 8.4 页面职责

| 页面 | 包 | 职责 |
|------|-----|------|
| `pages/map` | 主包 Tab | 首访 hard load + **一次** `includePoints` fit；再 `onShow` soft refresh（TTL 缓存、不打断视野）；两阶段 marker 图标；预览卡 |
| `pages/mine` | 主包 Tab | 会话展示（头像 key→`fileUrl`）、`ensureAuthenticated` 主动登录、用户区进 profile、入口跳转、退出 |
| `pages/detail` | 主包 | `openDetail(id)`；收藏前 `ensureAuthenticated`；类型展示优先用 context 中 map 预热的 types；分享落地 |
| history / favorites | 分包 | 浏览记录 / 收藏列表（收藏页未登录时弹授权登录，取消则回「我的」） |
| profile | 分包 | 编辑头像（`chooseAvatar`）/ 昵称 / 手机号；`loadProfile` + `updateProfile` |
| about / agreement / privacy | 分包 | 合规文案（可从「我的」进入；登录不再强制勾选） |

页面保持薄：事件 → context use case → `setData`；地图完整交互与视觉按 PRD / UIUX 实现。

## 9. 通用扩展机制

| 类型 | 做法 |
|------|------|
| 新业务模块 | §4.1 标准路径 + §6.2 模板 |
| 包体控制 | feature 级 subpackage + `preloadRule` |
| 列表缓存 | Repository 装饰器 `createCachedEncyclopediaRepository`（TTL + remote），调用方不改；`forceRefresh` 走 invalidate |
| 地图策略 | 替换 feature 内加载策略或 map 适配；保持 `MapMarkerVm` |
| 横切日志 / 埋点 | 装饰 http 或 session，不进业务 if 链 |
| 第三方 UI / 状态 | §5.1 |
| API 变更 | 隔离在 infrastructure repository |
| 多城 / 上下文 | 以 core 上下文或独立 feature 注入查询参数，禁止页面散落写死 |
| Feature 协作 | 优先提升共享端口到 core / shared；其次 `B.public`；禁止循环依赖 |

**纸面接入清单（Feature X）**

1. `features/x/{domain,application,infrastructure,presentation?,public.ts}`  
2. 需网络 → repository 使用 `core/http`  
3. 需登录 → `account/public` 的 `ensureAuthenticated`  
4. 需地图 → 产出 `MapMarkerVm[]`，用 `viewportFromMarkers` 或自有策略  
5. 更新 `app.json` 与 `app-context.ts`  
6. 不修改其它 feature 内部文件
