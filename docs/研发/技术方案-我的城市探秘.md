## 1. 摘要

「我的城市探秘」v1 是个人主体微信小程序：用户在地图上浏览运营录入的大理「百科」点位，登录后可云端收藏；浏览记录仅存本地。运营通过简易 Web 后台维护百科与上下架。

技术侧采用 **Cloudflare 边缘栈**：Workers（Hono API）+ Neon PostgreSQL（Drizzle）+ R2 图片 + Pages（Vue3 后台），客户端为微信小程序 TypeScript。v1 按「全量已上架点位可一张地图打完」设计，不做搜索、类型筛选、路线、多城、UGC。

## 2. 背景、目标与非目标

### 2.1 背景

- 仓库已按三端拆分：`city-quest-wxmp` / `city-quest-server`（submodule）/ `city-quest-console`，业务代码基本为空（绿场）。
- PRD §7.3 不锁实现栈，要求独立技术方案。
- 部署与运维偏好：**git + Cloudflare**，少自建机。

### 2.2 技术目标

| 目标 | 说明 |
|------|------|
| 可浏览 | 小程序地图展示全部已上架百科，预览 → 详情，本地浏览记录 |
| 可登录收藏 | 微信登录识别用户；收藏落库；「我的收藏」可读可取消 |
| 可运营 | Web 后台账号密码登录，百科 CRUD、多图、上下架 |
| 可上线 | 合法域名、合规文案页、首发内容可运营填充 |

对齐产品 KR：KR1 外网可开、KR2 主路径无阻断、KR3 后台可独立维护、KR4 首发可读内容（建议 ≥20 条）。

### 2.3 非目标（v1 明确不做）

- 用户发布、评论、点赞
- 搜索、类型筛选、路线规划、多城市
- 浏览记录云同步、推荐算法、推送
- 复杂权限/审核流/数据分析大盘
- 管理端多角色、OAuth 第三方登录

### 3.2 数据归属

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| 百科、用户、收藏 | Neon | 服务端权威 |
| 百科图片 | R2 | key 存于百科 `images` |
| 百科类型 | 静态 JSON | 不入库 |
| 浏览记录 | 小程序本地 | 不上云 |
| 用户/管理员会话 | JWT（客户端持有） | 服务端无 session 表 |

### 3.3 运行时边界

- **一个 Worker** 同时服务小程序公开/用户 API 与管理端 API（路径前缀区分 + JWT role 隔离）。
- 小程序 **不直连** Neon / R2 写接口；读图走 R2 公网域名（或 Worker 代理，见 §9）。
- 管理端为 SPA，所有业务经 API；无服务端渲染要求。

## 4. 技术选型与理由

| 层 | 选型 | 理由 |
|----|------|------|
| API 运行时 | Cloudflare Workers | 与 git+CF 部署一致；全球边缘；绑定 R2 自然 |
| HTTP 框架 | **Hono** | 轻量、TS 友好、Workers 一等公民、路由/中间件清晰 |
| 数据库 | **Neon PostgreSQL** | 托管 PG、分支方便；个人项目够用 |
| ORM / 迁移 | **Drizzle** + drizzle-kit | 类型安全、迁移可检视 SQL；配合 `@neondatabase/serverless` |
| 对象存储 | **Cloudflare R2** | 与 Worker 同生态、无出站流量费友好 |
| 运营后台 | **Vue 3 + Vite + Vue Router + Pinia + Tailwind CSS** | 用户指定；SPA 足够 CRUD |
| 后台托管 | **Cloudflare Pages** | git 集成部署简单 |
| 小程序 | 微信原生 **TypeScript** | 已有脚手架；v1 不引入 Taro/uni 等跨端 |
| 用户/管理鉴权 | **JWT（jose，HS256）** | 无状态，适合 Worker；user / admin 分 `role` claim |
| 管理密码 | env 单管理员 + **明文密码** | YAGNI：无 admin 表；由平台环境变量/密钥类型托管 |
| 校验 | **Zod** | 边界入参校验 |

### 4.1 刻意不选

| 项 | 原因 |
|----|------|
| 微信云开发 | 与选定 CF 栈不一致；后台扩展形态受限 |
| 自建 VPS 跑 Node/Go | 运维重于当前偏好 |
| Monorepo 共享 package（v1） | 三端发布节奏不同；类型配置以 server Assets 为权威，客户端仅 fallback 常量，避免过早抽象 |
| Session 表 / Redis | JWT 足够；少一个状态存储 |

### 5.3 部署拓扑

| 组件 | 方式 | 产物 / 命令（示意） |
|------|------|---------------------|

### 5.4 域名与小程序合法域名

| 用途 | 建议 |
|------|------|
| API | **Workers 自定义域**，例如 `https://api.example.com` |
| 图片 | R2 自定义域，例如 `https://img.example.com` |
| 后台 | Pages 域，例如 `https://console.example.com` |

小程序后台配置：

- request 合法域名：API 自定义域
- downloadFile 合法域名：图片自定义域（若用 R2 直链）

**锁定**：不要把生产流量默认绑在 `*.workers.dev` 上赌审核/合法域名可用性；v1 默认自定义域。

## 6. 领域模型与数据库

### 6.1 术语映射

| 产品（PRD） | 代码 / 表 |
|-------------|-----------|
| 百科 | `encyclopedia` / 表 `encyclopedias` |
| 类型 | `encyclopedia-types.json` 的 `key` |
| 上架 / 下架 | `status = published \| unpublished` |
| 游客 | 无有效 user JWT |

### 6.2 表结构

#### users

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | uuid | PK，默认 gen_random_uuid() | |

| nickname | text | NULL | 登录页采集，可空 |
| avatar_url | text | NULL | |
| created_at | timestamptz | NOT NULL | |
| updated_at | timestamptz | NOT NULL | |

#### encyclopedias

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | uuid | PK | |
| name | text | NOT NULL | |
| type_key | text | NOT NULL | 对应类型配置 key |
| lng | double precision | NOT NULL | **GCJ-02** |
| lat | double precision | NOT NULL | **GCJ-02** |
| intro | text | NOT NULL | 详情正文；列表/预览可截断 |
| address | text | NULL | 空则客户端不展示导航 |

#### favorites

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | uuid | PK | |

- `UNIQUE (user_id, encyclopedia_id)`
- `favorites_user_created_idx` ON `(user_id, created_at DESC)`

### 6.3 不入库的内容

| 内容 | 策略 |
|------|------|
| 管理员账号 | 环境变量 |
| 浏览记录 | 小程序 storage |
| 百科类型 | 静态 JSON |
| JWT 黑名单 | v1 不做；退出仅清客户端 token |

### 6.4 类型静态配置

权威文件：`city-quest-server/src/assets/config/encyclopedia-types.json`

```json
[
  { "key": "food", "name": "美食", "color": "#F97316" },
  { "key": "scenic", "name": "景点", "color": "#22C55E" },
  { "key": "goods", "name": "其他", "color": "#3B82F6" }
]
```

规则：

- 每条百科 **单选** 一个 `type_key`。
- 写入百科时校验 `type_key` 属于配置表。
- 小程序地图圆点颜色 = 类型 `color`（Canvas 动态生成，非按类型静态 PNG）。
- v1 **不**按类型筛选，仅着色；新增类型无需小程序发版加图标资源。

**同步策略（推荐）**

1. 权威：server `src/assets/config/encyclopedia-types.json`（Workers Assets，对外 `GET /config/encyclopedia-types.json`，**不计入 Worker 调用**）。  
2. Worker 内部：`import` 同一文件做 `type_key` 校验；管理端可选 `GET /admin/types` 兜底。  
3. 小程序：进地图时拉静态 Assets；失败用 `config/encyclopedia-types.ts` 常量。  
4. Console：优先拉静态 Assets；失败走 `/admin/types`；再失败用 `config/encyclopedia-types.ts` 常量（与小程序同形）。

### 6.5 图片 key 约定

```text
encyclopedias/{yyyy}/{mm}/{uuid}.{ext}
```

- `images[0]`：封面与分享封面。
- 空数组：详情不展示图；分享用默认图或仅标题（实现时定一种兜底）。

### 6.6 删除与下架语义

| 操作 | 行为 |
|------|------|
| 下架 | `status = unpublished`；地图与公开详情不可见；**收藏行保留** |
| 上架 | `status = published` |
| 物理删除 | admin DELETE；CASCADE 删除相关 favorites；R2 孤儿文件 v1 **可接受**（可后续脚本清） |

运营默认走下架，少物理删。

### 6.7 Drizzle 示意（非最终代码）

```ts
// 概念示意：实现以 schema.ts 为准
// users, encyclopedias, favorites — uuid PK, 上述列
// encyclopedias.images / tags: jsonb().$type<string[]>()
// favorites: uniqueIndex(userId, encyclopediaId)
```

迁移：drizzle-kit generate / migrate；生产对 Neon 执行迁移后再发 Worker。

## 7. API 设计

### 7.1 约定

- Base：`https://<api-domain>/api/v1`
- JSON；UTF-8
- 鉴权头：`Authorization: Bearer <token>`
- 统一信封：

```json
{
  "success": true,
  "data": {},
  "error": null,
  "meta": null
}
```

失败：

```json
{
  "success": false,
  "data": null,
  "error": { "code": "NOT_FOUND", "message": "百科不存在或已下架" },
  "meta": null
}
```

### 7.2 错误码

| code | HTTP | 含义 |
|------|------|------|
| `UNAUTHORIZED` | 401 | 未登录或 token 无效 |
| `FORBIDDEN` | 403 | role 不符（如 user token 调 admin） |
| `NOT_FOUND` | 404 | 资源不存在或对当前角色不可见 |
| `VALIDATION_ERROR` | 400 | 参数校验失败 |
| `WECHAT_LOGIN_FAILED` | 401/502 | code2session 失败 |
| `CONFLICT` | 409 | 唯一约束等冲突（收藏可改幂等 200） |
| `INTERNAL_ERROR` | 500 | 未分类服务端错误 |

### 7.3 公开接口（游客可用）

| Method | Path | Auth | 说明 |
|--------|------|------|------|
| GET | `/health` | 无 | 探活 |
| GET | `/config/encyclopedia-types.json` | 无 | **静态 Assets** 类型数组（客户端权威来源） |
| GET | `/public/encyclopedias` | 无 | **仅 published**；地图全量 |
| GET | `/public/encyclopedias/:id` | 可选 user | 仅 published 返回全文；否则 404；若带合法 user JWT 则附 `isFavorited` |

**地图列表建议字段**（控制 payload）：

`id, name, typeKey, lng, lat, intro`  
（intro 可服务端截断到 N 字供预览卡；详情再取全文）

**详情字段**：上表全部业务字段 + `images` 转为 **可访问 URL 列表**（key → `IMAGE_PUBLIC_BASE_URL + key`）+ `tags` + `status`（公开仅 published）+ `isFavorited?`。

### 7.4 用户鉴权与收藏

| Method | Path | Auth | 说明 |
|--------|------|------|------|
| POST | `/auth/wechat/login` | 无 | body: `{ code, nickname?, avatarUrl? }` → `{ token, user }` |
| GET | `/me` | user | 当前用户资料 |
| GET | `/me/favorites` | user | 收藏列表，**按 created_at 倒序**；含已下架 |
| POST | `/me/favorites` | user | body: `{ encyclopediaId }`；目标须 **published**；已存在则幂等成功 |
| DELETE | `/me/favorites/:encyclopediaId` | user | 取消收藏；**下架项也允许** |

**收藏列表项**（满足 PRD 下架展示）：

```json
{
  "encyclopediaId": "...",
  "name": "...",
  "typeKey": "...",
  "coverUrl": null,
  "intro": "...",
  "status": "published",
  "favoritedAt": "2026-07-13T00:00:00.000Z"
}
```

`status` 为 `unpublished` 时客户端置灰/角标「已下架」；点击提示不可用，并允许在列表取消收藏。

### 7.5 管理端

| Method | Path | Auth | 说明 |
|--------|------|------|------|
| POST | `/admin/login` | 无 | `{ username, password }` → `{ token }` |
| GET | `/admin/types` | admin | 类型配置兜底（推荐直接拉静态 Assets） |
| GET | `/admin/encyclopedias` | admin | 全部状态；query: `page`, `pageSize`, `status?` |
| POST | `/admin/encyclopedias` | admin | 创建；校验必填与 type_key |
| GET | `/admin/encyclopedias/:id` | admin | 详情（含 unpublished） |
| PATCH | `/admin/encyclopedias/:id` | admin | 部分更新；含 `status` 上下架 |
| DELETE | `/admin/encyclopedias/:id` | admin | 物理删除 |
| POST | `/admin/uploads` | admin | `multipart/form-data` 文件 → `{ key, url }` |

**创建/更新 body 字段**（与 PRD 对齐）：

`name, typeKey, lng, lat, intro, address?, businessHours?, avgPrice?, phone?, tags?, images?, status?`

校验要点：

- 必填：name, typeKey, lng, lat, intro；创建时 status 可默认 `unpublished`
- images 为 key 数组（先 upload 再提交表单）
- 单文件大小上限建议 **5MB**；超限 `VALIDATION_ERROR`
- 类型：jpeg/png/webp

### 7.6 下架相关 API 语义汇总

| 场景 | API 行为 |
|------|----------|
| 地图 | list 不含 unpublished |
| 公开详情 / 分享进入 | unpublished → 404；客户端展示失效页 |
| 新收藏 | 仅 published |
| 取消收藏 | 任意已收藏 id |
| 收藏列表 | join 百科，返回 status 与摘要，不因下架而消失 |

## 8. 鉴权与权限

### 8.1 小程序用户

1. 客户端 `wx.login` 取 `code`。
2. 登录页：头像（`chooseAvatar` 等能力）、昵称、协议勾选；未勾选不可登录。
3. `POST /auth/wechat/login`：Worker 用 `WECHAT_APPID/SECRET` 调 `jscode2session` 得 `openid`。
4. 按 openid upsert `users`；可更新 nickname/avatar。
5. 签发 JWT：`{ sub: userId, role: "user", openid }`，TTL 建议 **30 天**。
6. 客户端 `wx.setStorageSync` 存 token；`Authorization: Bearer` 带出。
7. 退出：清本地 token 与会话；服务端不吊销（v1）。

**通用登录拦截（产品硬规则）**

```text
点击需登录能力（如收藏）
  → 未登录：弹窗说明 → 确认则 navigateTo 登录页（可带 returnUrl）
  → 登录成功：navigateBack / 回 returnUrl
  → 不自动执行刚才动作（不自动收藏）
```

「我的」页提供主动登录入口，同一套登录页。

### 8.2 管理员

1. `POST /admin/login`：username 与 `ADMIN_USERNAME`、password 与 `ADMIN_PASSWORD` 均做常量时间比较。
2. 签发 JWT：`{ sub: "admin", role: "admin" }`，TTL 建议 **7 天** 或更短。
3. admin 中间件：校验签名且 `role === "admin"`。
4. **禁止** user JWT 访问 `/admin/*`（FORBIDDEN）。

### 8.3 权限矩阵（实现对照）

| 能力 | 游客 | 登录用户 | 管理员 |
|------|------|----------|--------|
| 地图/预览/上架详情 | ✓ | ✓ | ✓（走公开或 admin） |
| 本地浏览记录 | ✓ | ✓ | — |
| 收藏读写 | ✗ | ✓ | — |
| 百科维护/上传 | ✗ | ✗ | ✓ |

## 9. 图片与 R2

### 9.1 上传

1. 管理端选图 → `POST /admin/uploads`（Bearer admin）。
2. Worker 校验类型与大小 → 写入 R2 key（§6.5）→ 返回 `{ key, url }`。
3. 表单把 key 数组写入百科 `images`。

### 9.2 读取

**首选**：R2 bucket 绑定自定义域 `IMAGE_PUBLIC_BASE_URL`，URL = base + key。

备选：Worker `GET /files/*` 代理（增加 Worker 流量，仅当自定义域不便时）。

### 9.3 客户端

- 详情：有 images 才展示轮播/封面。
- 分享：标题默认百科名称；封面默认 `images[0]` URL。
- 小程序 downloadFile 合法域名包含图片域。

### 9.4 CORS

- API：允许 console 源；方法含 GET/POST/PATCH/DELETE/OPTIONS；头含 Authorization、Content-Type（Hono `cors`，仅 `run_worker_first` 路径如 `/api/*`）。
- 静态 Assets（`GET /config/*`）：不经 Hono，CORS 由 `src/assets/_headers` 配置（公开配置可用 `Access-Control-Allow-Origin: *`）。
- R2 公网读：以浏览器后台预览需要配置；小程序侧主要校验合法域名。

## 10. 小程序架构

### 10.1 信息架构（实现）

```text
Tab
├── 首页 map
└── 我的 mine
      ├── 登录入口 / 头像昵称 + 退出
      ├── 浏览记录 → history
      ├── 我的收藏 → favorites（仅登录显示入口）
      └── 关于/协议/隐私

栈页：detail | login | ...
```

### 10.2 首页地图

1. `GET /public/encyclopedias` 一次拉全量 published；并行拉类型配置（Assets `/config/encyclopedia-types.json`，失败用本地 fallback）。
2. markers：实心圆点 + 白描边；填充色 = 类型 `color`。
   - 微信 `<map>` `iconPath` 不支持 SVG；运行时用 **Canvas 2d 离屏绘制** 导出 PNG，按「颜色 × 选中态」缓存（内存 + `USER_DATA_PATH/cq-markers/`）。
   - 分类扩展只改类型配置，**不**再为每个 `type_key` 打包静态 marker 图。
   - 绘制/写盘失败回退包内 `assets/markers/default.png`；失败结果不永久占用缓存，下次可重试。
   - 关闭系统 POI 后，右侧常显点位名（原生 `label`，无底色纸片；选中仅加大字号）。点名称与点圆点均可选中（`labeltap` / `markertap`）。
3. `include-points`（或等价）使全部点位入视野；无点时 fallback 大理中心 + 默认 scale（实现写死常量）。
4. 点 marker → 预览卡（名称 + 介绍截断）→ 进详情；选中态用更大显示尺寸（预热 selected 图标，点选同步切换；圆点中心锚点居中放大）。

v1 不做：搜索、类型筛选、路线。

### 10.3 详情

- 进入页即写本地浏览记录（§10.4）。
- 展示规则：有值才展示地址/电话/人均/营业时间/标签/图片。
- 收藏按钮：未登录走 login-guard；已登录调 POST/DELETE；展示 `isFavorited`。
- `onShareAppMessage` / 朋友圈：标题 + 封面；path 带百科 id；落地若 404 则失效提示。

### 10.4 浏览记录（本地）

| 规则 | 实现 |
|------|------|
| 触发 | 进入详情 |
| 存储 | `wx.setStorageSync` 专用 key |
| 结构 | `{ id, name, typeKey, coverUrl?, intro?, viewedAt }[]` |
| 去重 | 同 id 一条；再访更新并移到最前 |
| 排序 | 最近在前 |
| 容量 | **最多 200**；超出丢弃最旧 |
| 清空 | 一键清空 |
| 失效 | 列表可显示缓存名；进详情若 404 则提示下架/不可用 |

模块：`storage/browse-history.ts`，纯函数 + storage 副作用隔离，便于测。

> 说明：若 PRD 验收摘要与正文容量表述不一致，技术方案与实现统一为 **200**（与 PRD §7.2.5 正文一致）。

### 10.5 网络层

- `services/http.ts`：封装 `wx.request`；挂 token；解析 envelope；统一 toast/错误码。
- 401：清会话，按需引导登录（收藏路径不自动重试写操作）。

### 10.6 坐标

全链路存储与小程序展示使用 **GCJ-02**（与微信地图一致）。**禁止把未转换的 WGS84 直接写入库或交给小程序**，避免点位偏移。

运营后台为便于选点，可使用开源 OSM 底图（WGS84）：

| 环节 | 坐标系 | 说明 |
|------|--------|------|
| 库 / API / 小程序 `<map>` | GCJ-02 | 权威数据 |
| 后台 `MapPicker` 底图 | WGS84（OSM） | 瓦片 `https://tile.openstreetmap.de/{z}/{x}/{y}.png` |
| 点击 / 拖标记 → 表单 | WGS84 → GCJ-02 | `utils/coord.ts` `wgs84ToGcj02` |
| 表单 / 加载 → 标记位置 | GCJ-02 → WGS84 | `gcj02ToWgs84`（迭代反算） |
| 经纬度输入框 | GCJ-02 文本 | 主路径为地图选点，输入仅作微调 |

实现要点：`city-quest-console` 使用 Leaflet + 上述瓦片；默认中心大理（GCJ-02 `100.19, 25.69`）；空串 / 非数字不得被 `Number("")` 当成 `0`。

|----|------|

## 15. 风险与合规

| 风险 | 影响 | 缓解 |
|------|------|------|
| 个人主体类目/能力限制 | 地图、用户信息、审核失败 | 提前选类目；头像昵称用官方推荐组件；准备精简方案 |
| 合法域名 | 小程序无法调 API/图 | **强制自定义域**；上线检查单含域名项 |
| code2session 出网 | 登录失败 | Worker fetch 微信域名；超时与错误码 `WECHAT_LOGIN_FAILED`；日志不落 secret |
| Neon 冷启动 | 首请求慢 | 精简 list 字段；可接受 MVP 延迟；必要时预热 |
| 坐标用错坐标系 | 点位偏移 | 存库/小程序锁定 GCJ-02；OSM 选点必须经 WGS84↔GCJ-02 转换后再写表单 |
| OSM 瓦片不可达（大陆网络） | 后台地图空白 | 经纬度输入仍可用；可换可达瓦片源或要求运营侧网络；CRS 策略与底图绑定 |
| R2 公网/CORS | 图裂、后台预览失败 | 自定义域 + 合法域名；上传类型限制 |
| workers.dev 依赖 | 审核/网络不稳 | 不用作生产主域 |
| 管理口令泄露 | 内容被改 | secrets 管理；强密码；仅一人掌握 |
| submodule 未 init | 部署缺 API 代码 | README/CI 明确 submodule |
| R2 孤儿文件 | 轻微存储浪费 | v1 接受；后续再清 |
| Skyline / map 兼容 | 真机异常 | 真机验收；必要时关 Skyline 特性 |
| 登录后误自动收藏 | 违反 PRD | login-guard 单测/清单勾选「不自动续办」 |
| 内容为空 | KR4 失败 | 阶段 C 明确运营录入任务 |
