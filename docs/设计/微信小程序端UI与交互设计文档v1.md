# 我的城市探秘微信小程序端 UI 与交互设计文档 v1.0

## 1. 设计目标

### 1.1 产品定位

「我的城市探秘」不是旅游攻略，也不是纯游戏化抽奖工具。它的第一屏应让用户感觉自己正在城市现场打开一张可探索的地图，围绕公开百科、实体宝箱、图鉴、线索和副本形成线下探秘闭环。

本设计覆盖两个端内角色：

- 用户端：游客、普通登录用户、有效会员、过期会员。
- 管理员端：同一小程序内的超级管理员工作台，用于投放宝箱、开通或续期会员、发放道具。

### 1.2 单一体验目标

让用户在首页完成三件事：

1. 看见当前城市有哪些可探索内容。
2. 理解自己当前能做什么。
3. 在会员状态允许时快速进入扫码、宝箱、线索和副本流程。

### 1.3 设计边界

- 用户端不展示价格。
- 用户端不提供线上支付入口。
- 普通用户扫码宝箱不展示宝箱详情、奖励信息或真实坐标。
- 会员开通后先进入待激活状态；用户扫会员欢迎码并在欢迎页点击「开始探索」后才获得会员能力。
- 解密地图作为道具进入专用欢迎页，必须在有效会员状态下开始。
- VIP+解密地图组合售卖只交付一个组合欢迎码。
- 热力图只展示漂移坐标。
- 罗盘解锁后展示真实坐标、文字指引、照片，但不提供地图导航。
- 过期会员保留历史资产可见性，但会员动作只读或阻断。

## 2. 视觉方向

### 2.1 设计主题

主题：城市刻痕地图。

视觉来源不是奇幻宝箱，而是城市现场能真实看到的材料：路牌、旧建筑铭牌、纸质地图折痕、运营贴纸、档案标签、坐标记录、投放照片。界面应克制、清晰、有线下行动感。

### 2.2 审美风险

风险点：首页地图叠加一层低透明度的「折线刻痕」网格，而不是常见的圆形热力气泡或渐变背景。

理由：

- 漂移坐标是产品核心安全机制，地图点位本来就不是精确坐标。
- 刻痕网格能表达「这里有线索，但不会把真实位置直接告诉你」。
- 它比宝箱插画更成人化，比普通地图 Pin 更有产品识别度。

执行约束：

- 刻痕只出现在首页地图、宝箱热区、开箱动效三处。
- 不在列表、表单、个人中心重复使用，避免变成装饰噪音。

### 2.3 色彩系统

| Token | Hex | 用途 |
|---|---:|---|
| `ink` | `#17201B` | 主文字、图标、强调线 |
| `paper` | `#F6F3EA` | 页面底色、地图抽屉底色 |
| `mist` | `#E1E8E2` | 分割线、禁用态、地图低层面 |
| `moss` | `#3F6B53` | 主操作、会员状态、有效点位 |
| `brass` | `#B4823D` | 罗盘、稀有奖励、管理员确认 |
| `signal` | `#D94E2E` | 错误、过期提醒、危险状态 |

用色规则：

- 页面不能被单一绿色控制。`moss` 只负责行动和会员点位，列表与大面积底色使用 `paper`、`ink`、`mist`。
- `brass` 只给罗盘、奖励和关键确认，避免所有按钮都变成金色。
- `signal` 只用于阻断、过期、失败，不用于营销。

### 2.4 字体系统

小程序实现优先使用系统字体，必要时通过 `wx.loadFontFace` 后置加载。

| 角色 | 字体建议 | 用法 |
|---|---|---|
| Display | Source Han Serif SC / Songti SC fallback | 只用于首页城市名、图鉴系列名、副本标题 |
| Body | WeChat Sans / PingFang SC / system-ui | 正文、列表、表单、按钮 |
| Utility | DIN Alternate / SF Mono / monospace fallback | 用户码、坐标、宝箱编号、时间 |

字号建议：

| Token | Size | Weight | 场景 |
|---|---:|---:|---|
| `hero` | 40rpx | 600 | 首页城市名、开箱结果标题 |
| `title` | 34rpx | 600 | 页面标题、详情标题 |
| `section` | 30rpx | 600 | 模块标题、卡片标题 |
| `body` | 28rpx | 400 | 正文、列表主信息 |
| `meta` | 24rpx | 400 | 标签、状态、辅助信息 |
| `code` | 30rpx | 600 | 用户码、坐标、编号 |

字距规则：

- 中文正文不使用负字距。
- 编号、坐标、用户码可使用等宽或数字字体，提高可扫读性。
- 大标题不做过度装饰，只用字体差异和留白形成识别。

### 2.5 图标与形状

- 导航图标使用线性图标，笔画 2px，避免填充式游戏 UI。
- 地图点位形状：
  - 百科：小圆点加短标签。
  - 宝箱热区：断续环形刻痕，不使用真实 Pin。
  - 已解锁真实指引：实心点加坐标标签。
  - 副本：折角标签。
- 卡片圆角不超过 8px。
- 不做卡片套卡片。页面分区使用底色、间距、分割线表达。

## 3. 信息架构

### 3.1 用户端底栏

使用自定义底栏，根据会员状态动态显示。

| 用户状态 | 底栏 |
|---|---|
| 未登录 / 普通用户 | 首页、探索、我的 |
| 待激活会员 | 首页、探索、我的 |
| 有效会员 | 首页、探索、图鉴、背包、我的 |
| 过期会员 | 首页、探索、图鉴、背包、我的 |

规则：

- 普通用户不展示图鉴和背包入口。
- 未登录、普通用户、待激活会员不展示任何会员能力入口；包括宝箱热区、宝箱分段、副本分段、线索分段、图鉴和背包。
- 「不展示」表示入口和分段 Tab 隐藏，不是显示后置灰或点击阻断。
- 未登录和普通用户底栏相同，但「我的」页内容不同：未登录显示登录入口，普通用户显示用户码和收藏。
- 有效会员首页显示常驻扫码按钮。
- 普通用户不显示小程序内扫码按钮，但可通过微信直接扫线下小程序码进入扫码结果页。
- 管理员入口只在「我的」页出现，不进入用户底栏。

### 3.2 管理员端入口

管理员身份由服务端识别。管理员打开「我的」页后看到「管理工作台」入口。

工作台不替换用户端底栏，而是进入独立二级页面。管理员完成任务后返回工作台或用户端。

### 3.3 页面清单

用户端：

| 页面 | 目的 |
|---|---|
| 首页 | 城市地图、百科点、会员宝箱热区、扫码入口 |
| 探索 | 按用户状态动态展示百科、宝箱、副本、线索分段 |
| 百科详情 | 阅读公开或已解锁百科，收藏和分享 |
| 扫码结果 | 解析小程序码，按身份分流，领取或开箱 |
| 会员欢迎页 | 展示套餐欢迎内容，点击开始探索后激活会员 |
| 解密地图欢迎页 | 展示指定地图内容，点击开始解密后进入地图起点 |
| 组合欢迎页 | 展示 VIP+解密地图组合内容，点击开始探索后同时激活会员并开始地图 |
| 开箱结果 | 全屏动效、奖励揭示、入账反馈 |
| 图鉴 | 系列进度、图鉴墙、详情 |
| 背包 | 宝箱、道具、线索、空箱记录 |
| 副本详情 | 线索时间线、答案提交、完成奖励 |
| 我的 | 登录入口、用户码、收藏、设置、会员状态或管理员入口，按状态动态展示 |

管理员端：

| 页面 | 目的 |
|---|---|
| 管理工作台 | 三项高频任务与最近操作 |
| 投放宝箱 | 绑定已有宝箱编号，采集现场信息 |
| 开通/续期会员 | 定位用户或创建临时用户、选套餐、填实收与备注、确认、交付欢迎码 |
| 售卖解密地图 | 校验有效会员、选择地图、填实收与备注、确认、交付欢迎码 |
| 组合售卖 | 创建 VIP+解密地图组合权益、确认、交付一个组合欢迎码 |
| 发放道具 | 定位用户、选道具、填数量与备注、确认 |

## 4. 关键布局

### 4.1 首页全屏地图

首页是产品主体验，不做营销页。

下图为有效会员状态；未登录、普通用户和待激活会员只展示公开百科地图，不展示宝箱热区图层切换和小程序内扫码按钮。

```text
┌─────────────────────────┐
│ 城市 上海      搜索/筛选 │
│                         │
│   百科点       刻痕热区  │
│        地图主体         │
│                         │
│             [扫码]      │
│ ┌─────────────────────┐ │
│ │ 底部抽屉：附近百科  │ │
│ │ 标签/距离/收藏状态  │ │
│ └─────────────────────┘ │
│ 首页 探索 图鉴 背包 我的 │
└─────────────────────────┘
```

状态规则：

- 首次进入请求定位。
- 用户拒绝定位时，显示手动选择城市入口，地图加载默认城市内容。
- 默认图层为百科。
- 有效会员可切换宝箱热区图层。
- 未登录、普通用户、待激活会员不展示宝箱热区图层切换。
- 过期会员看到常驻过期提醒条，但不展示宝箱热区图层切换。

地图控件：

- 左上：城市名和区域切换。
- 右上：搜索、筛选。
- 右侧：回到当前位置。
- 右下：扫码按钮，仅有效会员可见。
- 底部：内容抽屉，默认半展开。

首页地图点位：

- 公开百科：所有用户可见。
- 已解锁百科：仅对应用户可见。
- 锁定百科：不显示点位，避免内容泄露。
- 宝箱热区：仅有效会员可见，只显示漂移点位。

### 4.2 探索页分段列表

分段 Tab 按状态动态生成。下图为有效会员状态；未登录、普通用户和待激活会员只显示百科分段。

```text
┌─────────────────────────┐
│ 搜索城市内容             │
│ 区域  标签  状态         │
│ [宝箱][百科][副本][线索] │
│                         │
│ 列表卡片                 │
│ 列表卡片                 │
│ 列表卡片                 │
└─────────────────────────┘
```

分段可见性：

| 用户状态 | 宝箱 | 百科 | 副本 | 线索 |
|---|---|---|---|---|
| 未登录 | 不显示 | 显示公开百科 | 不显示 | 不显示 |
| 普通用户 | 不显示 | 显示公开百科 | 不显示 | 不显示 |
| 待激活会员 | 不显示 | 显示公开百科 | 不显示 | 不显示 |
| 有效会员 | 显示 | 显示 | 显示 | 显示已解锁 |
| 过期会员 | 不显示 | 显示 | 显示历史只读 | 显示历史只读 |

规则：

- 表格中的「不显示」表示对应分段 Tab 不出现在界面中。
- 未登录和普通用户只看到公开百科；需解锁百科、宝箱、副本、线索、图鉴和背包均不出现入口。

宝箱卡片：

- 显示区域、漂移距离、状态、是否已用罗盘解锁。
- 未解锁时展示「使用罗盘」。
- 已解锁时展示真实坐标、文字指引、照片入口。
- 不展示奖励池、真实 GPS 或内部编号。

百科卡片：

- 标题、封面、标签、收藏状态。
- 锁定百科不出现在普通列表中，除非用户已有解锁记录。

副本卡片：

- 名称、区域、难度、进度状态。
- 状态：未开始、进行中、已完成、未解锁。
- 有效会员可进入完整流程。
- 过期会员进入只读历史。

线索卡片：

- 只展示已解锁线索。
- 显示所属副本、状态、是否有精确指引。
- 不展示未解锁线索标题，避免谜题泄露。

### 4.3 扫码结果页

入口：

- 有效会员从首页扫码按钮进入。
- 任意用户可用微信直接扫线下小程序码进入。

流程：

```text
打开小程序码
-> 专用扫码结果页
-> 校验登录
-> 校验会员状态
-> 校验宝箱 token、状态、定位、领取限制
-> 展示结果
```

普通用户：

- 不进入可领取详情。
- 使用中性提示后回首页。
- 不展示扫码、宝箱或领取信息。

过期会员：

- 显示续期面板。
- 展示用户码、过期时间、线下续期提示。
- 不展示宝箱详情。

有效会员：

```text
┌─────────────────────────┐
│ 找到一个城市宝箱          │
│ 距离 7m / 领取范围 10m   │
│                         │
│ [加入背包] [立即打开]    │
└─────────────────────────┘
```

异常反馈：

| 场景 | UI |
|---|---|
| 定位未授权 | 明确提示需要定位才能领取，按钮「去开启定位」 |
| 不在范围内 | 显示距离不足，不展示真实位置 |
| 已领取 | 提示已领取，引导查看背包 |
| 已领完 | 提示已被领取完 |
| token 无效 | 提示宝箱不存在或已失效 |

### 4.4 欢迎入口页

欢迎入口页只承接线下已完成的权益，不做售卖解释、不展示价格。三类欢迎页共用同一信息骨架，但标题、封面、说明和开始目标由后台按套餐或解密地图配置。

会员欢迎页：

```text
┌─────────────────────────┐
│ 城市刻痕封面             │
│ 欢迎来到城市探秘          │
│ 套餐名 / 待激活           │
│                         │
│ 你将从这里开始本次探索    │
│ 到期时间将在开始后计算    │
│                         │
│ [开始探索]               │
└─────────────────────────┘
```

解密地图欢迎页：

```text
┌─────────────────────────┐
│ 地图封面 / 区域标签       │
│ 解密地图名               │
│ 难度 / 预计时长 / 区域    │
│                         │
│ 简短背景说明              │
│                         │
│ [开始解密]               │
└─────────────────────────┘
```

组合欢迎页：

```text
┌─────────────────────────┐
│ 组合封面                 │
│ VIP 探索已准备好          │
│ 解密地图名               │
│                         │
│ 开始后将激活会员并进入地图 │
│                         │
│ [开始探索]               │
└─────────────────────────┘
```

校验与分流：

- 用户扫码后先登录并校验欢迎码归属。
- 已有用户欢迎码必须匹配当前登录用户。
- 临时用户欢迎码首次扫码后自动绑定当前微信用户。
- 欢迎码无效、过期、作废或用户不匹配时，用户端统一展示「无法进入该探秘入口」，随后返回首页。
- 会员欢迎页点击「开始探索」后才激活会员，到期时间从点击成功时起算。
- 解密地图欢迎页点击「开始解密」时校验有效会员，不满足则直接阻断并回首页。
- 组合欢迎页点击「开始探索」后同时激活会员、标记地图已开始，并进入指定地图起点。
- 已完成欢迎码本人复扫时展示已激活或继续探索状态，不重复执行激活或开始。

### 4.5 开箱结果页

动效方向：

- 全屏开箱动效，可跳过。
- 背景使用短暂刻痕扫描，不使用烟花或金币雨。
- 奖励逐个揭示，稀有图鉴和罗盘使用 `brass` 强调。

结构：

```text
┌─────────────────────────┐
│      开箱中              │
│   刻痕扫描 / 宝箱轮廓    │
│           跳过           │
└─────────────────────────┘

┌─────────────────────────┐
│      收获已入账          │
│ [图鉴] 城市记忆          │
│ [道具] 寻宝罗盘 x1       │
│                         │
│ [查看背包] [继续探索]    │
└─────────────────────────┘
```

奖励类型展示：

| 类型 | 表达 |
|---|---|
| 图鉴 | 图像、标题、稀有度、所属系列 |
| 道具 | 图标、数量、用途短句 |
| 线索 | 所属副本、线索状态 |
| 百科解锁 | 标题、可查看入口 |
| 虚拟宝箱 | 宝箱卡片、稍后打开 |
| 空箱 | 克制反馈，引导继续探索 |

### 4.6 图鉴页

主结构：系列优先。

```text
┌─────────────────────────┐
│ 图鉴                     │
│ 系列进度  海派建筑  1/8 │
│ [全部][稀有][未获得]     │
│ ┌────┐ ┌────┐ ┌────┐   │
│ │已得│ │剪影│ │剪影│   │
│ └────┘ └────┘ └────┘   │
└─────────────────────────┘
```

规则：

- 已获得图鉴展示主图、标题、等级、系列。
- 未获得图鉴展示剪影，不泄露完整内容。
- 图鉴详情支持分享。
- 过期会员可查看历史图鉴。
- 普通用户无入口。

### 4.7 背包页

背包不重复展示图鉴，图鉴归属图鉴页。

分段：

- 宝箱
- 道具
- 线索
- 空箱

宝箱：

- 未打开：有效会员显示「打开宝箱」。
- 未打开：过期会员显示过期阻断。
- 已打开：只展示历史结果入口。

道具：

- 罗盘数量明显展示。
- 解密地图以地图卡片展示名称、区域、状态和继续入口。
- 使用记录可查看。
- 罗盘使用入口只出现在具体宝箱或线索上下文，不在道具列表里泛化使用。

线索：

- 展示已解锁线索。
- 可进入所属副本或线索详情。
- 过期会员只读。

空箱：

- 展示获得时间和来源。
- 不做负面情绪文案。

### 4.8 副本详情页

结构：线索时间线。

```text
┌─────────────────────────┐
│ 副本名 / 难度 / 区域      │
│ 进度  2/5                │
│                         │
│ ● 线索一  已完成          │
│ ● 线索二  进行中          │
│ ○ 线索三  未解锁          │
│                         │
│ 答案输入框                │
│ [提交答案] [使用罗盘]     │
└─────────────────────────┘
```

线性副本：

- 前置线索未完成时，后续线索不可提交答案。
- 未解锁节点显示锁定状态，不显示正文。

自由探索副本：

- 已解锁线索可并行查看和答题。

答题反馈：

- 正确：标记完成，推进进度。
- 错误：提示「答案不匹配，可以再试一次」。
- 不展示接近度。
- 不限制提交次数。

完成结果：

- 展示完成状态和奖励入账。
- 副本完成结果可分享。

### 4.9 我的页

内容：

- 未登录展示登录入口、设置、协议和隐私。
- 普通用户展示头像、昵称、用户码、收藏百科和设置。
- 未登录不展示用户码或收藏列表；普通用户不展示会员状态、到期时间、罗盘余额或权益入口。
- 用户码只在已登录状态展示，使用等宽数字字体。
- 待激活会员展示待开始状态和会员欢迎入口。
- 有效会员展示到期时间。
- 过期会员展示常驻续期提醒。
- 收藏百科。
- 设置、协议、隐私。
- 管理员显示「管理工作台」入口。

会员状态表达：

| 状态 | 表达 |
|---|---|
| 未登录 | 登录入口、设置、协议和隐私，不展示用户码或收藏列表 |
| 普通用户 | 用户码、收藏百科和设置，不展示会员状态或权益入口 |
| 待激活会员 | 显示「待开始探索」，提供进入会员欢迎页入口 |
| 有效会员 | 到期时间、罗盘余额、最近探索 |
| 过期会员 | 过期时间、用户码、线下续期提示 |

## 5. 交互规则

### 5.1 登录

采用按需登录。

触发登录的动作：

- 我的页点击登录按钮。
- 收藏百科。
- 扫会员欢迎码、解密地图欢迎码或组合欢迎码。
- 扫码结果页继续领取。
- 查看图鉴、背包。
- 使用罗盘。
- 提交答案。
- 查看用户码。

未登录可做：

- 浏览公开百科地图。
- 查看公开百科详情。
- 分享公开内容。

### 5.2 定位

- 首页首次进入请求定位，用于城市定位和地图中心。
- 拒绝定位后进入手动选城市状态。
- 扫码领取时仍必须有定位授权。
- 宝箱热区不请求真实坐标，只使用漂移坐标。

### 5.3 会员阻断

普通用户：

- 小程序内不出现会员能力入口。
- 通过外部小程序码进入隐藏能力页时，使用中性提示并回首页。
- 不展示价格。
- 不展示扫码、宝箱或领取信息。

待激活会员：

- 「我的」页展示待开始状态。
- 可进入本人会员欢迎页。
- 点击欢迎页「开始探索」前，不显示会员底栏和会员能力入口。
- 触发宝箱热力、开箱、罗盘、副本完整玩法时按非有效会员阻断。

过期会员：

- 首页和我的页常驻过期提醒。
- 触发会员动作弹续期面板。
- 续期面板展示用户码、过期时间、线下续期提示。

### 5.4 欢迎码与开始探索

欢迎码类型：

- 会员激活码：进入会员欢迎页。
- 解密地图开始码：进入解密地图欢迎页。
- VIP+解密地图组合码：进入组合欢迎页。

进入规则：

- 扫码后先登录并校验欢迎码归属、有效期、状态和权益记录。
- 已绑定用户欢迎码必须匹配当前登录用户。
- 临时用户欢迎码首次扫码后自动绑定当前微信用户。
- 验证失败统一显示「无法进入该探秘入口」，不展示具体失败原因，随后返回首页。
- 后台保留具体失败原因供管理员查看。

开始规则：

- 会员欢迎页点击「开始探索」后激活会员，激活成功后进入首页地图。
- 解密地图欢迎页点击「开始解密」后校验有效会员，成功后进入指定地图或副本起点。
- 组合欢迎页点击「开始探索」后同时激活会员和开始地图，成功后进入指定地图或副本起点。
- 已完成欢迎码本人复扫时，按钮显示「继续探索」，不重复激活或重复开始。

后台交付规则：

- 管理员完成开通、地图售卖或组合售卖后，结果页直接展示小程序码。
- 管理员可保存小程序码图片、查看记录或在记录详情重新生成。
- 小程序码有效期按套餐或解密地图后台配置展示给管理员。

### 5.5 罗盘

入口：

- 宝箱热区详情。
- 探索页宝箱卡片。
- 线索详情。

解锁前：

- 展示消耗 1 个寻宝罗盘。
- 展示当前余额。
- 明确对象名称。

解锁后：

- 不重复消耗。
- 宝箱展示真实坐标、精确指引、投放照片。
- 线索展示精确指引。
- 不提供地图导航。

### 5.6 分享与收藏

收藏：

- 仅百科详情支持收藏。
- 未登录时按钮文案为「登录后收藏」，点击后触发登录。

分享：

- 百科详情支持分享。
- 图鉴详情支持分享。
- 副本完成结果支持分享。
- 宝箱、线索、背包物品不做分享入口，避免泄露玩法内容。

### 5.7 空状态

空状态口径：克制、给下一步、不营销。

示例：

| 场景 | 文案 |
|---|---|
| 无百科结果 | 没有找到相关内容。换个关键词或区域试试。 |
| 无宝箱 | 当前区域没有可探索宝箱。可以切换区域查看。 |
| 无线索 | 还没有解锁线索。完成副本或打开宝箱后会出现在这里。 |
| 无图鉴 | 还没有获得图鉴。打开宝箱后会点亮收藏。 |
| 无背包宝箱 | 背包里还没有宝箱。去首页附近看看。 |
| 无解密地图 | 还没有可开始的解密地图。完成线下开通后会出现在这里。 |

### 5.8 错误状态

错误不道歉，不模糊。

| 场景 | 文案 |
|---|---|
| token 无效 | 宝箱不存在或已失效。 |
| 定位缺失 | 需要开启定位后才能领取宝箱。 |
| 超出范围 | 当前位置不在领取范围内。 |
| 已领取 | 你已领取过这个宝箱。 |
| 已领完 | 这个宝箱已被领取完。 |
| 罗盘不足 | 寻宝罗盘数量不足。 |
| 会员过期 | 会员已过期，续期后可继续使用。 |
| 欢迎码不可用 | 无法进入该探秘入口。 |
| 待激活会员能力阻断 | 开始探索后可使用会员能力。 |
| 解密地图会员不足 | 当前无法开始该解密地图。 |

## 6. 动效与反馈

### 6.1 动效原则

- 只在关键节点使用动效：首页图层切换、欢迎页开始、扫码结果、开箱、罗盘解锁。
- 尊重系统减少动态效果设置。
- 动效不阻塞信息读取，开箱动效必须可跳过。

### 6.2 首页地图

- 地图加载后，百科点位以 120ms 间隔轻微浮现。
- 切换宝箱热区时，刻痕环从外向内短暂描边。
- 抽屉上拉下压使用 180ms ease-out。

### 6.3 扫码结果

- 校验中使用短线扫描，不使用转圈 loading 作为主视觉。
- 成功后宝箱轮廓出现，动作按钮延迟 80ms 进入。
- 失败直接显示原因，不播放失败动画。

### 6.4 欢迎页开始

- 欢迎页加载时只做封面轻微上浮，不做营销式大动画。
- 点击「开始探索」或「开始解密」后，按钮进入处理中状态，等待服务端确认。
- 会员激活成功时显示一次短暂刻痕亮起，随后进入首页或地图起点。
- 失败时立即显示统一阻断文案，不播放失败动画。

### 6.5 开箱

- 全屏 1.2s 标准动效。
- 提供「跳过」。
- 稀有奖励卡片轻微放大 1.03，再回落。
- 空箱无戏剧化反馈，只显示结果和下一步。

### 6.6 罗盘解锁

- 确认消耗后，罗盘刻度旋转 300ms。
- 解锁信息以坐标、指引、照片顺序展开。
- 已解锁再次查看不播放消耗动效。

## 9. 可访问性与小程序约束

- 所有按钮最小点击区域 88rpx。
- 文字与背景对比满足移动端阅读。
- 不用颜色单独表达状态，必须配合文字或图标。
- 表单错误定位到具体字段。
- 页面主操作固定在可触达区域，避免被安全区遮挡。
- iPhone 小屏下按钮文字不换行溢出。
- 减少动态效果时关闭地图点位浮现、开箱扫描、罗盘旋转。
- 管理员金额输入使用数字键盘。
- 用户码、坐标、宝箱编号、欢迎码记录编号支持长按复制。
- 欢迎页主按钮必须固定在安全区上方，避免小屏遮挡。
- 管理员小程序码结果页必须保留文本摘要，避免只依赖图片识别。

## 10. 自检与修订记录

### 10.1 非模板化自检

已避免的常见默认方案：

- 没有使用大面积奶油色加陶土色的文艺旅游风。
- 没有使用黑底霓虹色的重游戏风。
- 没有使用报纸式细线栏目作为主结构。
- 没有用营销首页替代实际地图体验。
- 没有把宝箱画成主要视觉符号，避免低龄化。

本方案的特异性来自：

- 漂移坐标对应刻痕热区。
- 用户端与线下运营端共享同一套城市现场语言。
- 坐标、用户码、宝箱编号作为真实操作材料进入界面。

### 10.2 待设计补充

后续进入高保真或代码实现前需要补齐：

- 首页地图真实视觉稿。
