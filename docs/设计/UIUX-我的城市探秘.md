## 1. 设计目标与原则

### 1.1 产品在界面上要完成的事

| 端 | 首要工作 | 一句话 |
|----|----------|--------|
| 小程序 | 在地图上发现策展点位 | 打开即地图，少噪音，能预览、进详情、登录后收藏 |
| 运营后台 | 稳定录入与上下架 | 密而静、可扫读、表单一次填完 |

### 1.2 体验原则（决策用）

1. **地图即首页**：首屏不讲故事、不放营销 Hero；用户打开就应看到点位。
2. **策展感，不是信息流**：内容像「本地清单贴在地图上」，不做瀑布流卡片墙。
3. **有值才展示**：空字段不占行；无图不占位大图。
4. **登录不抢戏**：游客可完整逛；登录只在收藏等能力处出现。
5. **登录后不自动续办**：回原页后由用户再次点击（PRD 硬规则）。
6. **两端分工**：小程序偏旅途气质；后台偏工具效率——共享色相，不共享装饰。

### 1.3 明确不做的视觉方向

- 紫蓝渐变 SaaS、玻璃拟态堆叠、装饰性 blob
- 奶油底 + 衬线大标题 + 陶土强调色的通用 AI 模板
- 纯黑底 + 荧光强调色的「暗黑科技」风
- 过度插画/贴纸化「旅游 App 模板」
- 类型筛选 Chip 栏、搜索条、路线控件（v1 产品范围外）

## 2. 设计方向（Design Direction）

### 2.1 命题

**「洱海晨雾里的白墙青瓦地图」**  
——大理不是喧闹的 neon 城市；产品气质应是**清爽、可信任的本地向导**，用地图与类型色点完成「一眼看全」，用克制的纸面卡片完成「点开即读」。

### 2.2 受众扫描优先级

| 优先级 | 用户要先看到什么 |
|--------|------------------|
| P0 | 地图上全部点位 + 类型色含义可感知 |
| P1 | 点位预览（名称 + 介绍） |
| P2 | 详情完整信息与收藏/分享 |
| P3 | 我的：浏览 / 收藏回访 |

### 2.3 调性关键词

| 要 | 不要 |
|----|------|
| 清、静、可信、清单感 | 喧、花、广告感、游戏化 |
| 纸面 + 地图工具 | 厚重拟物 / 夸张 3D |
| 类型色只服务图例 | 全局彩虹 UI |

### 2.4 签名元素（Signature）

**「点位签」Preview Tag**

- 点选地图 marker 后，底部（或 marker 旁安全区）浮出一条**横向点位签**：
  - 左侧 **4–6px 类型色竖条**（唯一强色）
  - 主文：名称（1 行截断）
  - 辅文：介绍（2 行截断）
  - 右侧轻量「详情」示意（chevron 或文字按钮）
- 不强制封面图；无图不留灰块。
- 签名只花在这里：其余页面保持安静的列表与表单。

记忆点：**类型色竖条 + 纸面签**，而不是大圆角彩色卡片。

## 3. 设计令牌（Design Tokens）

实现时：小程序写入 `app.wxss` CSS 变量（或 SCSS 变量）；Console 写入 Tailwind theme / CSS variables。

### 3.1 色彩

#### 品牌与中性（小程序 + 后台共用色相）

| Token | Hex | 用途 |
|-------|-----|------|
| `color.ink` | `#1A2332` | 主文字、图标默认 |
| `color.ink-secondary` | `#5C6570` | 次要文字、说明 |
| `color.ink-muted` | `#8B939E` | 占位、时间戳、禁用说明 |
| `color.paper` | `#F5F7F6` | 页面底（白墙+薄雾） |
| `color.surface` | `#FFFFFF` | 卡片、列表行、弹层 |
| `color.line` | `#E2E6EA` | 分割线、输入边框 |
| `color.line-strong` | `#C9D0D6` | 强调边框、未选中控件 |
| `color.fill` | `#E8EEF2` | 浅底块、Tag 默认底、骨架 |
| `color.primary` | `#2B4C7E` | 主按钮、关键链接、Tab 选中、焦点 |
| `color.primary-pressed` | `#1F3A63` | 主按钮按下 |
| `color.primary-soft` | `#E7EEF6` | 主色浅底（选中 chip、info 底） |
| `color.danger` | `#C23B3B` | 错误、破坏性操作 |
| `color.danger-soft` | `#FCECEC` | 错误浅底 |
| `color.warning` | `#B45309` | 警告文案（少用） |
| `color.success` | `#1F7A4C` | 成功反馈（偏深，保证对比） |
| `color.scrim` | `rgba(26,35,50,0.45)` | 弹窗遮罩 |
| `color.map-chrome` | `rgba(255,255,255,0.92)` | 地图上浮层底 |

#### 百科类型色（仅图例 / marker / 点位签竖条 / 类型小圆点）

| type_key | name | color | 文字叠色规则 |
|----------|------|-------|----------------|
| `food` | 美食 | `#F97316` | **禁止**橙底白字小标签若对比不足；优先「色点 + 墨色字」 |
| `scenic` | 景点 | `#22C55E` | 同上；大面积底改用 `success` 深色或浅底 |
| `goods` | 其他 | `#3B82F6` | 可用作小面积；正文仍用 `ink` |

**规则**

- 类型色 **不**作为页面主背景或大按钮底色。
- 地图 marker：实心圆点，外圈白描边 1–2px，保证压在深浅底图上可辨。
- 类型名展示：`● 美食`（● 为类型色，文字为 ink）。
- 下架态：整体降为 `ink-muted` + 角标「已下架」；取消类型高饱和色。
- 信息 **不单靠颜色**：类型必有名称；状态必有文字。

#### 对比度（设计验收）

- 正文 `ink` on `paper` / `surface`：目标 ≥ 4.5:1
- 次要 `ink-secondary` on `surface`：≥ 4.5:1（不足则加深）
- 主按钮白字 on `primary`：≥ 4.5:1
- 类型色仅作点缀时，不依赖颜色传达唯一信息（同时有类型名）
- 地图浮层与底图：正文与关键控件图形对比 ≥ 3:1

### 3.2 字体

| 角色 | 小程序 | Console | 规格建议 |
|------|--------|---------|----------|
| 展示/页标题 | 系统中文（PingFang SC / 系统默认） | 同左 + `system-ui` | 20–22px / 34–40rpx，Semibold |
| 卡片标题 | 同上 | 同上 | 16–17px / 32rpx，Medium–Semibold |
| 正文 | 同上 | 同上 | 15–16px / 28–30rpx，Regular，行高 1.55–1.65 |
| 辅助/元信息 | 同上 | 同上 | 12–13px / 24rpx，Regular，`ink-secondary` |
| 数字（人均、列表时间） | `tabular-nums` 若平台支持 | `font-variant-numeric: tabular-nums` | 与辅文同级 |

**不引入**自定义商用字体包（个人项目体积与审核成本）；用 **字重 + 字号 + 颜色** 做层级。

中文排版：

- 标题与短文案避免两端生硬断行；按钮文案 ≤ 6 字优先。
- 介绍截断：预览卡 2 行；列表 1–2 行；详情全文。

### 3.3 间距与圆角

基于 **8pt** 节奏（小程序用 rpx：约 `1px ≈ 2rpx` @375 设计宽，实现时按项目基准统一）。

| Token | 值 | 用途 |
|-------|-----|------|
| `space.1` | 4 | 图标与字、紧标签内边距 |
| `space.2` | 8 | 行内元素间隙 |
| `space.3` | 12 | 列表行内边距半级 |
| `space.4` | 16 | 页面左右边距、卡片内边距 |
| `space.5` | 24 | 区块间距 |
| `space.6` | 32 | 大分段 |
| `space.7` | 48 | 罕见大留白 |

| Token | 值 | 用途 |
|-------|-----|------|
| `radius.sm` | 6 | 小控件、Tag |
| `radius.md` | 10 | 输入框、次要按钮 |
| `radius.lg` | 14 | 点位签、列表卡片 |
| `radius.xl` | 20 | 底部弹层顶角、大图 |
| `radius.pill` | 999 | 少数胶囊（如已下架角标） |

**同心圆角**：外层圆角 ≈ 内层圆角 + 内边距（点位签内若有小图时遵守）。

### 3.4 阴影与描边

| Token | 值 | 用途 |
|-------|-----|------|
| `shadow.card` | `0 4px 16px rgba(26,35,50,0.08)` | 点位签、浮层 |
| `shadow.sheet` | `0 -4px 24px rgba(26,35,50,0.10)` | 底部 sheet |
| `border.default` | `1px solid color.line` | 列表分割可改用底边 |
| `focus.ring` | `0 0 0 3px color.primary-soft` + 边框 primary | Console 键盘焦点 |

地图浮层优先 **浅阴影 + 细边框**，避免厚重投影。

### 3.5 触控与尺寸

| 项 | 值 |
|----|-----|
| 最小点击热区 | **44×44px**（约 88×88rpx） |
| 主按钮高度 | 44–48px |
| 列表行最小高度 | 56px（有副文可更高） |
| 地图 marker 视觉直径 | 12–16px；热区由地图组件处理，密集时可接受略叠但点选以最近为准 |
| Tab 图标+文字 | 符合微信 Tab 规范，选中 `primary`，未选中 `ink-muted` |

### 3.6 动效

| 场景 | 建议 |
|------|------|
| 点位签出现 | 150–200ms，opacity + translateY(8→0) |
| 点位签消失 | 120–150ms，仅 opacity（更快、更静） |
| 按钮按下 | scale(0.98) 或背景变 `primary-pressed`，100–150ms |
| 页面转场 | 用系统导航默认；不自定义花哨路由动画 |
| 列表加载 | 骨架块 `fill` 色微闪（可选）；失败用文案+重试 |
| 尊重系统 | 若可检测减少动态，则取消非必要位移 |

**禁止** `transition: all`；只过渡具体属性。

## 5. 关键用户流程（界面层）

### 5.1 逛地图

```text
打开小程序
  → 首页全屏地图 + 加载态
  → 成功：markers + include-points 框住全部点
  → 失败：地图兜底中心（大理）+ 错误条「点位加载失败，点击重试」
点 marker
  → 选中态 marker（可略放大或描边加强）
  → 弹出点位签（名称 + 类型名 + 介绍截断）
点点位签 / 「查看详情」
  → 详情页
  → 写入本地浏览记录
```

### 5.2 收藏与登录（通用拦截）

```text
详情点「收藏」
  ├─ 已登录 → POST 收藏 → 按钮变「已收藏」
  └─ 未登录
        → Modal：「需要登录」
           正文说明登录用途 + 成功后需再次点击收藏
           主按钮：去登录 / 次按钮：取消
        → 登录页（可带 return 路径）
        → 成功 navigateBack
        → **不**自动收藏；可 toast「登录成功，请再次点击收藏」
```

「我的」主动登录：同一登录页，无「续办」语义。

### 5.3 下架内容

| 入口 | 界面 |
|------|------|
| 地图 | 无点 |
| 分享/直链详情 | 失效页：说明「该地点已下架或不可用」+ 返回首页 |
| 收藏列表 | 行置灰 + 角标「已下架」；点击 → 提示或失效详情；**可在列表取消收藏** |
| 浏览记录 | 可显示本地缓存名；进详情若失效则提示 |

### 5.4 运营录入

```text
登录 → 列表（默认可看全部状态）
  → 新建 / 编辑表单
  → 先上传图片拿 key，再保存
  → 上架后小程序地图可见（无需发版）
```

## 6. 小程序页面规格

### 6.1 首页 · 地图 `pages/map`

#### 布局（竖屏）

```text
┌─────────────────────────────┐
│ 导航栏：我的城市探秘          │
├─────────────────────────────┤
│                             │
│         全屏 map             │
│      ·  ·    ·   ·          │
│         ·  (markers)        │
│                             │
│  ┌───────────────────────┐  │
│  │▌ 点位名称              ›│  │  ← 点位签（有选中时）
│  │  ● 美食 · 介绍截断两行… │  │
│  └───────────────────────┘  │
├─────────────────────────────┤
│  首页          我的          │
└─────────────────────────────┘
```

#### 行为

| 项 | 规格 |
|----|------|
| 数据 | 一次拉全量 published |
| 视野 | `include-points`；无点时大理中心 + 默认 scale |
| Marker | 圆点；填充类型 `color`；白边；选中可放大 ~1.2 或加粗描边（实现：Canvas 动态生成 PNG 作 `iconPath`，非按类型静态图） |
| 点位签 | 距 Tab / 安全底 ≥ 16px；左右 margin 16；含类型名 |
| 二次点击同点 | 保持签；点地图空白 → 收起签并取消选中 |
| 加载 | 首次全屏轻量 loading 或顶部进度；不阻塞地图底图 |
| 空数据 | 地图兜底视野 + 居中说明「暂时还没有点位」+ 刷新 |
| 错误 | 可重试条，不使用阻塞整页死白 |
| 图例（推荐） | 角落或首次轻提示：三色点 + 美食/景点/其他 |

#### 点位签内容

- 左类型色条（固定宽）
- 名称 1 行
- 类型名（文字）+ 介绍 2 行，`ink-secondary`
- 整卡可点进详情；热区整卡 ≥ 44 高

#### v1 不出现

搜索框、类型 Filter、路线控件。

### 6.2 百科详情 `pages/detail`

#### 信息块顺序（上 → 下）

1. **图集**（仅 `images.length > 0`）：顶区宽图/swiper；首图为封面
2. **标题区**：名称；其下「类型色点 + 类型名」；标签行（有则 chip）
3. **介绍**：正文，可多段
4. **信息列表**（有值才渲染行）  
   - 地址（可点：打开地图/导航）  
   - 营业时间  
   - 人均  
   - 电话（可点拨号）
5. **底栏**（固定）：收藏按钮；分享依赖系统菜单（右上角）或 `open-type="share"`（二选一，推荐 **系统菜单 + 底栏收藏**）

#### 收藏按钮态

| 态 | 展示 |
|----|------|
| 未登录 / 未收藏 | 描边或浅底 + 图标 + 「收藏」 |
| 已收藏 | 实心 primary 或 ink + 图标 + 「已收藏」 |
| 加载中 | 禁用 + 短暂 loading，防连点 |
| 下架失效页 | 说明 + 「取消收藏」（从收藏进入时） |

状态 **文案 + 图标双编码**，不只换心形颜色。

#### 分享

- 标题：百科名称  
- 封面：`images[0]`；无图用小程序默认分享图（实现时提供一张中性默认图）

#### 浏览记录

进入有效详情后写入本地；失效页 **不**写入。

### 6.3 登录 `pages/login`

```text
┌─────────────────────────────┐
│ 登录                         │
│                             │
│        [ 选择头像 ]          │
│        昵称输入框            │
│                             │
│   ☐ 我已阅读并同意           │

│                             │
└─────────────────────────────┘
```

| 规则 | 说明 |
|------|------|
| 协议未勾选 | 主按钮禁用，或点击时 toast「请先同意用户协议和隐私政策」；勾选热区含整行文案 |
| 头像昵称 | 微信推荐能力；UI 引导填写 |
| 成功 | 回上一页；无栈则回「我的」 |
| 失败 | 「登录失败，请重试」，不暴露底层错误 |

视觉：居中内容区，背景 `paper`，少装饰。

### 6.5 浏览记录 `pages/history`

| 项 | 规格 |

## 7. 运营后台页面规格

### 7.1 总体视觉

- **工具型**：高密度、浅底、细边框表格/列表  
- 主色 `primary` 仅用于主按钮、链接、焦点  
- 类型色仅在列表「类型」列显示小圆点 + 类型名  
- 不使用点位签装饰；不需要旅游插画  

### 7.2 登录 `LoginView`

- 居中窄卡片（max-width ~360px）  
- 字段：用户名、密码；主按钮「登录」  
- 错误：卡片内 `danger-soft` 提示条，统一「账号或密码错误」  
- 已登录访问 `/login` → 重定向列表  
- 可见焦点环；label 与控件关联  

### 7.3 列表 `EncyclopediaListView`

| 列 | 说明 |
|----|------|
| 名称 | 主识别 |
| 类型 | 色点 + 名 |
| 状态 | 文字徽章：已上架 / 已下架（非仅色点） |
| 更新时间 | tabular 数字 |
| 操作 | 编辑；快捷上架/下架（按钮名含对象语义，避免一排无差别「编辑」） |

- 顶栏：标题「百科」+「新建」主按钮 + 可选状态下拉筛选  
- 分页：底部分页器  
- 空态：「还没有百科，点击新建」  

### 7.4 表单 `EncyclopediaFormView`

字段布局（单栏，max-width ~640–720）：

1. 名称 *  
2. 类型 *（select，展示类型名）  
3. **位置 ***  
   - 经度 / 纬度（并排输入框，**GCJ-02**；支持手动微调）  
   - 下方 **地图选点**（`MapPicker`）：主录入方式  
     - 底图：开源 OSM 瓦片 `https://tile.openstreetmap.de/{z}/{x}/{y}.png`  
     - 交互：点击地图或拖动标记选点  
     - 选点后回写经纬度输入框；改输入框（防抖）同步标记位置  
     - helper：底图为 WGS84，**保存坐标为 GCJ-02（与微信地图一致）**  
4. 介绍 *（textarea，较高）  
5. 地址  
6. 营业时间 / 人均 / 电话（可两列）  
7. 标签（TagInput：回车或逗号生成 chip，可删）  
8. 图片（Uploader：多图、可排序，**第一张为封面**；限制类型与 5MB）  
9. 状态：上架 / 下架（创建默认下架，避免误发布）  

底栏粘性：取消 / 保存；破坏性删除在编辑页次要位置，二次确认。

校验：字段级中文错误；经纬度须为有效数字且在合法范围（拒绝空串被当成 `0`）；提交失败时页顶摘要可聚焦。

### 7.5 图片上传

- 缩略图网格 + 序号角标（1 = 封面）  
- 上传中 per-item progress  
- 失败可重试/移除；删除控件有明确名称  

### 7.6 后台 a11y 要点

- 全键盘可达登录与 CRUD；无键盘陷阱  
- `:focus-visible` 高对比环  
- 路由切换后焦点落到主标题或主区域  
- 放大 200% 不裁切关键操作

## 10. 无障碍与包容性（设计约束）

### 10.1 不可协商

1. 颜色不单独表达信息（类型有文字，状态有文字）  
2. 可点击控件有可见名称；收藏/上下架双编码  
3. 触控目标小程序 ≥ 44×44；后台关键目标 ≥ 24×24 且间距充足  
4. 正文对比 ≥ 4.5:1；图标边框/选中环/地图关键 UI ≥ 3:1  
5. 空态 / 错态 / 登录门闸有中文说明 + 下一步  
6. 登录后不自动收藏，文案写清  
7. 下架可读「已下架」并允许清理收藏  
8. 图片有替代策略；无图不硬占位  

### 10.2 小程序

- 地图页有明确导航标题；预览卡承担 marker 的可访问摘要  
- 选中 marker 视觉明显；二次进详情热区足够  
- 弹窗主次按钮清晰；表单错误有文字  
- 安全区：底栏与点位签避开 home indicator  

### 10.3 Console

- 可见焦点、label 关联、状态徽章带文字  
- 全键盘完成登录与主要表单  
- 操作按钮语义区分（编辑《名称》）  

### 10.4 类型色

| 做法 | 要求 |
|------|------|
| 地图 | 色点 + 白/深描边 + 签/详情类型名 |
| 列表 | 色点 + 类型名 |
| 大按钮 | 不用类型色做底 |
| 状态色 | 与类型色分册，禁止混用 |

## 16. 决策记录（已锁定）

| 决策 | 选择 | 原因 |
|------|------|------|
| 视觉主题 | 白墙雾感 + 洱海靛主色 | 贴合大理与策展向导，避开通用 AI 模板 |
| 签名交互 | 点位签（类型色竖条） | 地图场景记忆点强，实现成本低 |
| 类型色职责 | 仅图例级 + 强制类型名 | 可读性与 a11y |
| 小程序字体 | 系统字体层级 | 体积与审核 |
| 后台风格 | 密而静工具风 | 运营效率；与 C 端气质同源不同貌 |
| 经纬度录入 | 地图选点为主 + 输入微调 | 纯手输几乎不可行；存库仍 GCJ-02 |
| 后台底图 | OSM（openstreetmap.de 瓦片） | 开源免 key；展示 WGS84，写入前转 GCJ-02 |
| 预览是否强制图 | 否 | 与 PRD 一致 |
| 登录续办 | 不自动收藏 | PRD 硬规则 |
| 预览卡类型 | 名称 + 类型名 + 介绍 | a11y：颜色不单独表达 |
