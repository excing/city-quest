## 1. 结论摘要

实体宝箱领取统一走**扫码链路**（小程序内扫码 + 微信扫一扫），不在首页宝箱点卡片上完成领取。

核心闭环：

**识别码参数 →（有效会员）进扫码结果页 Loading → 定位上报 + 服务端校验 → 通过则原子入包 → 成功待开 / 已领 / 分类失败 →「立即打开」在当前页弹「宝箱开启」弹窗**

| 身份 | 扫实体宝箱码 |
|---|---|
| 有效会员 | 进入扫码结果页，展示完整成功 / 失败信息 |
| 未登录 / 普通 / 待激活 / 过期 | **不进入结果页**，**静默回首页**；不提示、不展示宝箱信息、不领取、不强制登录 |

- 校验通过后**自动领取入包**（非「先预览再点领取」）。
- 前端**始终上报定位**；范围配置（含 `-1` 不限）仅服务端判定。
- 开箱（立即打开 / 背包打开 / 未来其它入口）统一为当前页**弹窗**，不整页跳转开箱结果。

## 2. 入口与码分流

### 2.1 扫码入口

| 入口 | 说明 |
|---|---|
| 首页悬浮扫码 | 仅**有效会员**展示（与导航规划一致） |
| 「我的」扫码 | 有效会员可用；**控件形态为普通按钮**（非悬浮）；页面布局另案，本链路行为与首页一致 |
| 微信扫一扫 | 外扫小程序码冷启动，与内扫**同一套业务规则** |

- 扫码界面本身：只开相机、识别小程序码，**不展示业务结果**；**进入内扫扫码页**时再申请相机（不在首页预申请；详见权限规划）。
- 未登录 / 普通 / 待激活 / 过期：**不展示**扫码入口（首页悬浮与「我的」按钮均不展示；与导航一致）。
- 相机权限拒绝：展示**相机权限中断页**（说明 + 去设置），不进入扫码结果页；去设置返回后 **自动重试**打开扫码；用户**返回/关闭 → 首页 Tab**（详见权限规划 §5）。
- 外扫（微信扫一扫）冷启动：**不经过**小程序相机权限；业务规则与内扫一致。

### 2.2 码参数与分流

- 小程序码携带**多参数**；前端将参数**原样上报**服务端，**不解析业务含义**。
- 服务端解析码型并分流：
  - 实体宝箱 → 本文件扫码结果页链路
  - 欢迎码 / 解密地图码 / 组合码 → 既有欢迎页链路
  - 无法识别 → 按码型失败策略处理
- 参数字段名与码生成由后台对齐；用户端不依赖 6 位运营编号作为查询凭证。

### 2.3 非有效会员处理

- 含未登录：扫实体宝箱码 → **静默回首页**。
- 不强制拉起登录（与收藏等「触发登录」场景区分）。
- 不返回宝箱敏感信息；后台仍记扫码失败日志。

## 3. 扫码结果页

仅**有效会员**进入。页面名统一为**扫码结果页**（偏开箱语义，不叫「领取成功页」）。

### 3.1 状态机

```
进入结果页
  → Loading（定位 + 校验 + 可能入包）
    → 成功待开（新领取，未打开）
    → 已领取（本人曾领过）
    → 分类失败（不入包）
  Loading 超时 / 网络失败 → 直接回首页（需再次扫码）
```

### 3.2 Loading

职责（同一阶段完成）：

1. 获取定位并上报 `lat/lng`（**凡走本链路一律定位**，前端不知宝箱是否配置 `-1`）；申请时机与全局失败分级见权限规划。
2. 将码多参数 + 定位提交服务端。
3. 服务端校验：会员有效、参数有效、宝箱状态、位置范围、单用户是否已领、总次数；通过则**原子领取入包**。
4. Loading 动效期间用户等待；结束后进入对应终态。
5. 定位拒权：去设置后返回结果页 **自动重试** Loading；定位系统失败 / 超时：可重试定位（与「距离不足」区分）。

失败时：**绝不入包、不扣可领取次数**。  
「已领取」为独立业务态，不重复扣次。

### 3.3 成功待开

条件：本次校验通过并新写入背包，宝箱未打开。

展示：

- 宝箱氛围信息（图片、名称/等级氛围等）
- 「已放入背包」类文案
- 主按钮：**立即打开**
- 次按钮：**查看背包**
- 可回首页

不展示：真实 GPS、运营 6 位编号、扫码参数 / 内部参数。

### 3.4 已领取

条件：该有效会员对该实体宝箱已有领取记录。

| 背包中该箱状态 | 表现 |
|---|---|
| 未打开 | 主「立即打开」+ 次「查看背包」 |
| 已打开 | 主「查看背包」+ 可回首页 |

- 不二次扣总次数、不重复入包。
- 可在本页对未打开箱直接弹窗开箱。

### 3.5 分类失败（有效会员可读）

| 失败类型 | 用户侧表现 | 是否入包 |
|---|---|---|
| 距离不足 | 提示需靠近宝箱；展示到真实点的**直线距离（米）**；可「重新定位」；**不展示真实坐标 / 地图钉** | 否 |
| 定位授权拒绝 | 需开启定位才能领取 + 去设置；**含范围 `-1` 箱也不可领** | 否 |
| 定位系统失败 / 超时（定位侧） | 可重试定位（与「距离不足」区分） | 否 |
| 次数已满 | 提示已被领取完 | 否 |
| 状态不可领 | 未投放 / 停用 / 已回收 / 异常等，提示不可领取；不暴露内部运维细节 | 否 |
| 扫码参数无效或不存在 | 提示宝箱不存在或已失效；不返回内部原因 | 否 |
| Loading 网络失败 / 接口超时 | **直接回首页** | 否（以服务端为准；不确定时靠再次扫码） |

- 内部错误码、堆栈、真实坐标、库存实现细节不对用户展示。
- 距离数字：服务端用上报点与**真实坐标**计算的直线距离（米）；**漂移坐标不参与领取与距离展示**。

### 3.6 次要导航

| 操作 | 落地 |
|---|---|
| 查看背包 | 进入背包列表（可定位到刚领取的宝箱，实现可选） |
| 回首页 | 回到首页 Tab |
| 重新定位 | 停留结果页，重新走 Loading 校验（未入包场景） |

## 4. 领取规则（服务端）

与总 PRD 对齐，本专题强调：

| 规则 | 说明 |
|---|---|
| 有效会员 | 领取时必须为有效会员；以当次接口查询状态为准 |
| 同用户同箱 | 同一用户对同一实体宝箱只能成功领取一次 |
| 总次数 | 原子扣减；`-1` 表示不限；默认 1 |
| 位置范围 | 默认 10 米，最小 10 米，`-1` 表示**有有效 lat/lng 时**不拦距离；仅服务端用真实坐标判定；**缺定位 / 拒权一律不可领（含 `-1`）** |
| 状态 | 仅「已投放」可领；待投放 / 停用 / 已回收 / 异常不可领 |
| 隐藏宝箱 | 不进首页地图；**扫码领取规则与普通实体箱一致** |
| 并发 | 事务保证防超发；失败方得到已领完 / 已领 / 业务失败语义 |
| 入包时机 | 校验全通过后自动入包；再展示成功待开 |

前端定位策略：

- 始终请求并上报定位（与权限规划一致：前台 `getLocation`，不做后台定位 / 轨迹画像）。
- 不根据精度字段拒领（v1 不做强防作弊）。
- 范围是否 `-1`、是否在圈内，**只信服务端结果**。
- **拒权 / 无 lat/lng 一律不入包**，即使该箱配置范围为 `-1`（`-1` ≠ 可无定位领取）。
- 拒权 → 去设置 + 返回自动重试 Loading；系统失败 / 超时 → 可重试（全局失败分级与恢复见权限规划）。

## 5. 开箱：「宝箱开启」弹窗

### 5.1 挂载方式

- **不在扫码结果页整页切换为开箱结果页**。
- 任意开箱入口在**当前页面**弹出「宝箱开启」弹窗：
  - 扫码结果页点「立即打开」
  - 背包内打开
  - 未来其它打开入口

### 5.2 弹窗内容（v1）

- 游戏化开启动效
- 奖励列表（图鉴 / 道具 / 线索 / 虚拟宝箱 / 空箱等，按奖励池结果）
- 按获得奖励类型提供跳转按钮（图鉴 / 线索 / 道具等）
- 去背包
- 关闭 / 回首页

### 5.3 开箱校验

- 打开时再次校验**有效会员**（与领取分开的两次校验）。
- 会员已过期：阻断打开，不发奖，宝箱保持未打开。
- 物品归属、未打开状态、奖励池执行规则沿用总 PRD。

### 5.4 与「立即打开」关系

- 本质：已在 Loading 成功时完成领取 → 用户点「立即打开」→ 调开箱 → 当前页弹窗展示奖励。
- 非「Loading 内自动开箱」。

## 6. 会员临界与权限边角

| 场景 | 处理 |
|---|---|
| Loading 时已非有效会员 | 按非有效会员：不进结果页或接口判定失败；不入包（静默策略优先于结果页） |
| 入包后、开箱前过期 | 开箱接口阻断；箱保持未打开 |
| 过期会员持有历史未开箱 | 不可在扫码链路打开；背包打开同样阻断（总 PRD） |
| 有效会员扫无效扫码参数 | 结果页失败态「不存在或已失效」 |
| 非有效会员扫无效扫码参数 | 与扫有效箱相同：**静默回首页**（不区分提示） |

说明：前后端均以**当次服务端会员状态**为准，不做客户端会话内过期容忍。

## 7. 日志与安全

### 7.1 扫码日志（全量）

服务端记录至少：

- 用户 ID（未登录可空）
- 上报多参数 / 码型
- 结果码与失败原因（内部）
- 定位（若有）
- 时间

非有效会员静默场景**也记日志**。

### 7.2 安全边界

- 所有会员、位置、次数、归属校验在服务端完成。
- 扫码参数不可枚举；6 位编号不作客户端查询凭证。
- 非有效会员响应不包含宝箱详情、真实坐标、投放照片等。
- v1 不做强防 GPS 作弊；接受玩法性定位篡改风险（总 PRD）。

## 8. 接口最小约定

| 能力 | 约定 |
|---|---|
| 扫码解析 + 领取 | 上报：码多参数 + lat/lng；返回：结果态枚举 + 展示字段（成功待开氛围 / 已领状态 / 失败类型 / 距离米等） |
| 开箱 | 共用开箱接口；驱动「宝箱开启」弹窗 |
| 分流 | 服务端识别非宝箱码并返回可路由类型（欢迎页等） |

结果态建议枚举（实现可命名对齐）：

- `CLAIMED_PENDING_OPEN` — 新领成功待开  
- `ALREADY_CLAIMED` — 已领取（附是否已打开）  
- `OUT_OF_RANGE` — 距离不足（附 distanceMeters）  
- `LOCATION_DENIED` / `LOCATION_FAILED` — 由客户端定位阶段处理为主；服务端可回显缺定位  
- `QUOTA_EXHAUSTED` — 次数已满  
- `UNAVAILABLE` — 状态不可领  
- `INVALID_CODE` — 参数无效  
- `FORBIDDEN_SILENT` — 非有效会员（客户端应静默回首页，即使误进页）

## 10. 验收场景

- 有效会员内扫：Loading → 成功待开 → 立即打开弹窗发奖；查看背包进列表。
- 有效会员外扫：与内扫相同规则。
- 范围外：失败 + 真实直线距离 + 重定位后可成功领取。
- 已领再扫：已领态；未开可弹窗开箱；已开仅查看背包。
- 总次数 = 1 双人并发：仅一人成功，另一人次数已满或等价失败。
- 非有效会员内外扫：静默首页，无结果页、无提示、无领取。
- 无效码：仅有效会员见失效失败态；非有效会员静默首页。
- 定位拒权 / 定位失败 / 相机拒权 / Loading 超时：拒权去设置返回后自动重试；系统失败可重试；相机中断页不进结果页，**返回/关闭 → 首页**；Loading 超时回首页（详见权限规划）。
- 隐藏箱：可扫码领取，规则同普通箱。
- 范围 `-1`：有定位时服务端不拦距离；前端仍上报定位；**拒权 + `-1` 仍不可领**。
- 领取后过期再开箱：阻断且箱未开。

## 11. 明确不做（本专题 v1）

- 首页宝箱卡片上完成领取。
- 领取前完整业务预览（未入包前展示可领详情）。
- Loading 内自动开箱。
- 非会员轻提示 / 会员引导 / 价格相关文案。
- 前端解析码业务含义或按前缀硬分流（允许仅透传）。
- 距离模糊化、展示真实坐标钉、强 GPS 精度门槛。
- 「我的」页完整 UI 信息架构（另案）。

## 12. 已锁定决策记录（grill-me）

| 议题 | 决策 |
|---|---|
| 领取时机 | 校验通过后自动领取入包 |
| 结果页信息 | 成功后展示氛围信息，不露 GPS/编号/扫码参数 |
| 非会员 | 静默回首页 |
| 位置校验 | 进结果页 Loading 内完成；前端总是定位上报 |
| 已领再扫 | 已领态；未开可继续立即打开 |
| 码分流 | 统一入口，多参数原样上报，服务端解析 |
| 外扫 | 与内扫同一规则 |
| 不可领 | 有效会员分类失败结果页 |
| 定位拒权 / 系统失败 | 分开：去设置 vs 可重试 |
| 拒权恢复 | 去设置返回后自动重试当前动作（相机重开扫码 / 结果页重走 Loading） |
| 相机中断返回 | 返回/关闭 → 首页 Tab（权限规划） |
| 外扫与相机 | 仅内扫需要小程序相机权限 |
| 成功态 CTA | 主立即打开、次查看背包 |
| 距离不足 | 展示真实直线距离米，不展示坐标 |
| 结果页模型 | 单页偏开箱语义：Loading → 终态；开箱用弹窗 |
| 开箱形态 | 当前页「宝箱开启」弹窗，多入口共用 |
| 范围 -1 | 配置仅后台；前端仍定位；**拒权不可领（含 -1）** |
| Loading 超时 | 回首页 |
| 未登录外扫 | 不强制登录 |
| 日志 | 全量服务端日志 |
| 并发 | 事务防超发 + 已领/领完语义 |
| 隐藏箱 | 扫码规则一致 |
| 会员校验 | 领取与开箱两次以服务端状态为准 |
