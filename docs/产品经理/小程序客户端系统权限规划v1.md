> 状态：已确认（grill-me 定稿）  
> 关联草稿：[我的城市探秘产品需求文档v1-草稿v1.md](./我的城市探秘产品需求文档v1-草稿v1.md)

> 说明：本文件为**用户端**微信小程序的**系统能力 / 隐私合规**专题规格（定位、相机、登录资料、订阅消息、隐私协议等）。业务角色权限（有效会员 / 过期等）以总 PRD 为准。

> **联动**：v1 **无**消息通知 / 站内消息中心入口（见导航规划）；到期触达 = 微信订阅消息 + 首页过期软提示。

## 1. 结论摘要

| 维度 | v1 决策 |
|---|---|
| 能力清单 | 前台定位 + 地图蓝点、内扫相机、登录昵称头像、到期订阅消息、分享；**不做**手机号 / 相册 / 麦克风 / 剪贴板 / 后台定位 |
| 定位时机 | **进首页即申请**（含访客）；扫码 Loading 再取点上报 |
| 定位形态 | `<map>` + `show-location` 蓝点 + `wx.getLocation`（gcj02）；单次取点 +「定位到我」，不持续跟随 |
| 相机时机 | **进入内扫扫码页**再申请；外扫不占用小程序相机权限 |
| 失败分级 | **拒权 → 去设置**；**系统失败/超时 → 可重试**（全局统一） |
| 拒权恢复 | 去设置后返回当前页 **自动重试**当前动作；相机中断页**返回/关闭 → 首页 Tab** |
| 隐私合规 | **双层并存**：产品协议**仅登录页**勾选 + 微信隐私保护指引 API；**访客定位不额外弹产品协议** |
| 登录资料 | 强制昵称头像（chooseAvatar + 昵称填写）；协议**默认不勾选** |
| 访客 | 可逛公开百科；收藏等触发登录；进首页同样申请定位（**仅本机**城市/蓝点，**不落库**） |
| 订阅消息 | **仅会员到期提醒**；每次「非有效→有效」激活成功后**按需**拉授权（未 accept 才弹）；**仅有效会员**设置可补开；过期不发；活动上新延后 |
| 定位上报 | 扫码当次校验上报；**首页定位仅已登录**可落库，**每会话最多 1 条**（可覆盖更新）；不做连续轨迹画像 |
| 消息中心 | v1 **无**消息通知入口 / 站内消息中心 |

## 2. 能力清单

| 能力 | v1 | 主要用途 | 说明 |
|---|---|---|---|
| 前台定位 `wx.getLocation`（建议 gcj02） | 要 | 默认城市匹配、地图「我」、扫码领取上报、**已登录**首页位置上报 | 不申请后台定位 / 轨迹 |
| 地图组件 `<map>` + `show-location` | 要 | 首页底图与系统蓝点 | 画地图本身不需定位；蓝点需要定位授权 |
| 相机（**仅内扫**） | 要 | 扫码页识别小程序码 | 外扫走微信扫一扫，无小程序相机授权 |
| 登录用户信息 | 要 | 头像选择 + 昵称填写，登录强制 | 不用旧版一键 getUserProfile 作为主路径 |
| 订阅消息 | 要 | **仅**会员到期提醒 | 活动上新 v1 不做订阅模板；无站内消息中心 |
| 转发好友 / 可选朋友圈 | 要 | 内容分享 | 无额外系统权限；不做海报存相册 |
| 手机号 | **不做** | — | v1 不暴露绑定入口；总 PRD 字段可预留 |
| 相册写入 / 选图、麦克风、剪贴板、后台定位 | **不做** | — | — |

- `app.json` 权限声明与公众平台《用户隐私保护指引》**用途文案**由开发 + 法务按上表场景后补；产品只锁定场景。

## 3. 隐私与登录

### 3.1 双层合规

| 层级 | 内容 | 交互 |
|---|---|---|
| 产品协议 | 《用户协议》《隐私政策》 | **仅登录页**链接 + 勾选；设置页可再次查看；后台管理版本 |
| 微信隐私保护指引 | 平台强制：调用定位 / 相机等前须用户同意 | `wx.requirePrivacyAuthorize` / 隐私弹窗组件等现行 API |

- **产品勾选不能替代**微信隐私授权。
- 首次触发需隐私授权的接口前，若用户未同意微信指引，先完成微信隐私授权再继续该能力。
- **访客**进首页申请定位：**不**额外弹产品协议；仅走微信隐私指引（若需要）+ 系统定位授权。产品协议在登录时勾选，覆盖登录后上报等处理说明。

### 3.2 登录交互

- 访客可浏览首页百科地图等公开能力；收藏、欢迎码等既有「需登录」场景再拉登录。
- 登录流程：`wx.login`（openid）+ **头像**（`chooseAvatar`）+ **昵称**（昵称输入类型）+ 协议勾选。
- 协议**默认不勾选**；未勾选不可点登录。
- 昵称、头像未齐全不可完成登录。
- v1 **不**提供手机号绑定入口。
- 访客会话内**登录成功**：若本会话已有本机定位成功点，可按 §4.4 上报该会话**唯一**首页定位记录（**登录后才落库**）。

### 3.3 设置页（权限相关）

「我的 → 设置」至少包含：

- 用户协议、隐私政策入口
- 定位 / 相机「去开启」引导（跳转微信设置）
- **到期提醒**开关（订阅消息补授权）：**仅有效会员展示**；过期会员 / 未开通过会员 / 待激活 / 普通 / 未登录**不展示**

不做完整系统权限总开关列表。v1 **无**消息通知入口（见导航规划）。

## 4. 定位

### 4.1 申请与使用

| 场景 | 行为 |
|---|---|
| 进入首页（含未登录访客） | 申请定位；成功则匹配已开通城市并展示蓝点；失败/拒权 → 默认运营城市并提示 |
| 地图展示 | 使用地图组件；开启 `show-location` 系统蓝点 |
| 「定位到我」 | 再取一次最新点；若用户已**手动切换其他城市**，控件落到**该城中心**（与导航规划一致） |
| 首页在前台 | **不**持续跟随轨迹；非前台持续定位 |
| 扫码结果页 Loading | **一律** `getLocation` 并上报 `lat/lng`；前端不知是否 `-1`；范围只信服务端 |
| 重新定位（结果页） | 停留结果页，重新走 Loading 校验 |

### 4.2 失败分级（全局）

| 类型 | 用户侧 | 典型动作 |
|---|---|---|
| 拒权 / 未授权 | 说明需开启定位 + **去设置** | 首页落默认城；**扫码不入包（含范围配置为 `-1` 的宝箱）** |
| 系统失败 / 超时 | **可重试**（与距离不足区分） | 首页可重试或继续默认城策略；扫码结果页可重试定位 |
| 距离不足（扫码业务） | 真实直线距离（米）+ 重定位；不展示真实坐标 | 见扫码规划 |

**拒定位 × 范围 `-1`**：`-1` 仅表示服务端在**已有有效 lat/lng** 时不拦距离；**不表示**可无定位领取。客户端无定位或用户拒权时：**一律不入包**（与是否配置 `-1` 无关）。

### 4.3 拒权恢复

1. 展示说明 +「去设置」打开微信设置页。  
2. 用户从设置返回当前页后，**自动重试**当前动作：  
   - 首页 → 重新 `getLocation` / 刷新蓝点与城市匹配  
   - 扫码页 → 重新打开扫码（相机场景）  
   - 结果页 → 重新走 Loading 定位校验  
3. 用户**不**去设置、点返回/关闭：  
   - 相机中断页 → **回首页 Tab**  
   - 首页定位说明 → 留在首页并保持默认运营城市策略  
   - 扫码结果页定位拒权态 → 可留在结果页失败态，或按扫码规划次要导航回首页  

### 4.4 服务端使用与存储

| 来源 | 用途 | 存储边界 |
|---|---|---|
| 扫码领取 | 当次范围校验；日志可含定位 | **校验必带有效 lat/lng**；缺定位 / 拒权不入包；失败原因可记 |
| 首页定位（**仅已登录**） | 城市匹配辅助、运营分析等 | **允许上报并落库**；字段建议：坐标 + 时间 + **用户 ID** + 场景枚举 `home`；**每小程序会话最多 1 条**，同会话内再次成功取点（含「定位到我」）**覆盖更新**该条，不无限追加 |
| 首页定位（**访客 / 未登录**） | 仅本机默认城市 + 蓝点 | **不上报、不落库** |
| 轨迹 | — | v1 **不建**连续轨迹画像、不做路径回放 |
| 防作弊 | — | v1 不做强 GPS 精度门槛 / 强防篡改（总 PRD） |

- 保留周期、脱敏与是否可关：实现阶段最小合规落地（建议有限期保留，后台可配）。
- 漂移坐标仅用于首页宝箱层展示，**不参与**领取距离计算与距离展示。

## 5. 相机（内扫）

| 项 | 规则 |
|---|---|
| 申请时机 | 用户进入**内扫扫码页**时申请；不在首页预申请 |
| 扫码页职责 | 只开相机、识别小程序码；不展示业务结果 |
| 拒权 | **相机权限中断页**（说明 + 去设置），**不进入**扫码结果页 |
| 恢复 | 去设置返回后**自动重试**打开扫码 |
| 返回/关闭 | 用户不点去设置、点返回或关闭 → **回首页 Tab** |
| 外扫 | 微信扫一扫冷启动；**不经过**小程序相机权限；业务规则与内扫一致（见扫码规划） |

## 6. 订阅消息

| 项 | 规则 |
|---|---|
| 模板范围 | **仅会员到期提醒** |
| 活动上新 | v1 **不做**微信订阅模板；**也不做**站内消息中心承载活动上新 |
| 授权触点 | 每次用户从**非有效会员 → 有效会员**（欢迎页/组合页点击「开始探索」等**激活成功**）后：先 `getSetting({ withSubscriptions: true })` 看到期模板状态 → **已 accept 则跳过**；**未 accept**（含从未询问、可再请求）则 `requestSubscribeMessage`；**reject / ban** 不追弹 |
| 不拉起 | 有效会员期内的**纯顺延续期**（已是有效会员再次激活顺延）**不**自动拉订阅 |
| 设置补开 | **仅有效会员**在「我的 → 设置」展示「到期提醒」开关，可再次请求授权 |
| 发送策略 | 到期前若干天，以后台配置为准（总 PRD 系统配置）；**仅当前有效会员**且模板仍为可发状态时发送；**过期后不发送** |

## 7. 分享

- 支持转发好友（`onShareAppMessage` / `button open-type=share`）。
- 可选朋友圈（`onShareTimeline`）；失败不影响主路径。
- v1 **不**做分享海报写入相册。

## 8. 与账号态 / 既有链路关系

| 账号态 | 定位 | 首页定位落库 | 相机（内扫入口） | 订阅到期 |
|---|---|---|---|---|
| 未登录 | 进首页申请；可拒权落默认城 | **否**（仅本机） | 无有效会员扫码入口；外扫实体箱静默回首页 | 无；设置无到期开关 |
| 普通 / 待激活 | 同首页定位 | **是**（已登录，每会话 ≤1） | 无实体箱内扫入口；欢迎码等另链 | 待激活无发送；**非有效→有效**激活成功后按需拉；设置无到期开关（非有效会员） |
| 有效会员 | 首页 + 扫码 Loading | **是**（每会话 ≤1） | 有扫码入口，进页申请相机 | 设置可补开；按配置发送到期提醒 |
| 过期会员 | 首页定位仍可 | **是**（已登录，每会话 ≤1） | 无扫码入口；外扫实体箱静默回首页 | **不发送**；设置**隐藏**到期开关 |

业务领取 / 开箱会员校验仍以扫码规划与总 PRD 为准。

## 9. 明确不做（本专题 v1）

- 手机号授权 / 绑定入口  
- 相册、麦克风、剪贴板、后台定位、实时轨迹  
- 登录默认勾选协议  
- 活动上新微信订阅消息  
- **消息通知入口 / 站内消息中心**（到期靠订阅 + 首页过期软提示）  
- 海报保存到相册  
- 首页前台持续跟随定位  
- 访客首页定位上报落库  
- 无定位 / 拒权时按范围 `-1` 放行领取  
- 强 GPS 精度拒领  
- 管理员小程序权限清单（另案）  
- `app.json` / 隐私指引具体审核文案定稿（开发 + 法务）

## 10. 验收场景

- 访客打开小程序进首页：申请定位；拒权落默认运营城市并提示；同意后蓝点与城市匹配；**坐标不上报、不落库**；**不**额外弹产品协议（仅微信隐私指引 + 系统授权）。  
- 访客登录成功：若本会话已有本机定位点，可上报**该会话唯一**首页定位记录。  
- 已登录用户进首页：成功定位后上报；同会话内再次进首页 /「定位到我」**覆盖更新**，不追加多条。  
- 登录：未勾选协议不可登录；缺昵称或头像不可登录；齐备可登录。  
- 首次调定位/相机前：未同意微信隐私则先完成隐私授权。  
- 内扫拒相机：中断页 → 去设置 → 返回自动重进扫码；**返回/关闭 → 首页 Tab**。  
- 扫码 Loading：拒定位「去设置」且**不入包**；系统失败可重试；范围外距离 + 重定位；范围 `-1` 在**有定位**时可领；**拒权 + `-1` 仍不可领**。  
- 外扫实体宝箱码：无需小程序相机授权即可进入同一业务链（有效会员进结果页）。  
- 首次开通：待激活 → 有效激活成功后，未 accept 则拉到期订阅；已 accept 跳过。  
- 过期后再激活：非有效 → 有效成功后同样按需拉；过期期间不发送到期提醒；过期时设置无到期开关，成为有效会员后出现。  
- 有效期内纯顺延：不自动拉订阅。  
- 「我的 → 设置」：协议入口 + 定位/相机引导；到期提醒开关**仅有效会员**可见。  
- 导航与「我的」：**无**消息通知入口。  
- 无手机号 / 相册等能力入口；分享不申请相册权限。

| 相机时机 | 进扫码页再申请 |
| 外扫与相机 | 仅内扫需要小程序相机 |
| 拒权恢复 | 去设置 + 返回自动重试 |

| 访客 | 可逛；进首页也定位（本机） |

| 订阅范围 | 仅到期；活动上新延后；无站内消息中心 |

| 过期后订阅开关（G7） | **仅有效会员**设置展示；过期隐藏；过期不发送 |
| 消息中心（G8） | v1 **去掉**消息通知入口 |

| 分享 | 好友 + 可选朋友圈，无存相册 |
| 用途文案 | 场景锁定，声明文案后补 |
