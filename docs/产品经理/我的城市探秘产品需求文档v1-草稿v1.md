# 我的城市探秘产品需求文档

---

## 一、产品概述

一款基于 LBS（地理位置）+ 微信小程序的线下寻宝互动产品。核心玩法：

**宝箱 → 用户扫码发现 → 开启获得宝物 → 收藏/继续解谜**

同时叠加「副本 - 线索 - 答案」的解谜玩法，以及会员订阅制的商业模式。

注1: 产品界面不显示任何价格, 产品不支持线上购买和支持, 以及相关说明, 本产品的销售完全由线下进行.

## 四、功能需求补充

### 4.1 用户侧

- 微信授权登录（暂不需要手机号强绑定，可预留字段, 为后期开发做准备）
- 浏览公开百科：**列表模式 + 地图模式 + 标签筛选 + 关键词搜索**
- 收藏（需登录）
- 内容分享（转发好友/朋友圈），可考虑分享裂变激励

### 4.2 会员侧

- 周边宝箱热力图（会员专属）：**接口不返回真实 GPS 坐标, 返回飘移后的坐标(±10米)**，用户无法反推具体位置，需实地探索发现
- 扫码开箱完整流程：扫码 -> 查询宝瓶有效范围 → 校验用户 GPS 是否在有效范围内 → 开箱动画 → 宝物自动入账背包
- 图鉴管理：已获得图鉴墙、未解锁图鉴以剪影/灰色展示、图鉴详情、集齐进度、图集图鉴管理查看筛选
- 背包管理细化：
  - 道具：数量、使用记录、购买入口
- 线索/副本管理：副本列表（未解锁/进行中/已完成状态）、线索详情、提示使用计次、指引消耗道具、答案提交与校验反馈
- 消息通知：会员到期提醒、活动/上新通知

### 4.3 管理侧

- 数据看板：宝箱投放/发现统计、宝箱开启率、副本完成率、营收统计、活跃用户数、留存率
- 角色权限分级：目前仅超级管理员
- 合规管理：隐私政策与用户协议版本管理
- 道具管理："道具"的创建/管理/查看

## 五、关键业务规则确认情况

1. 宝箱被发现后是否需要员工线下回收，还是可以长期放置不回收？
  宝箱由用户发现并保存或丢弃或忽略, 属于消耗品, 对于长期未被发现的, 自动标记为异常情况. 对于可重复多次开启的宝箱, 一般会由工作人员放置在无法移动的位置, 系统目前无需考虑该问题.

# 十一、防作弊

不考虑防作弊, 用户愿意修改GPS就让他修改, 愿意把宝箱码发给其他人就发, 愿意一个设备登录多个账号就登, 无所谓, 对我们来说, 没有任何损失, 甚至这也可以是游戏乐趣一部分.

我们只需要考虑做好数据权限和安全控制即可, 比如可开启次数为1的宝箱被开启了10次, 这就是我们的服务bug了, 这种情况必须杜绝. 比如接口返回了宝瓶真实的GPS数据这种情况, 必须杜绝等等.
