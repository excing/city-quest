# 我的城市探秘产品需求文档

---

## 一、产品概述

一款基于 LBS（地理位置）+ 微信小程序的线下寻宝互动产品。核心玩法：

**宝箱（虚拟，游戏内容主体）→ 以宝瓶为现实载体投放到线下 → 用户扫码发现 → 开启获得宝物 → 收藏/兑换/继续解谜**

同时叠加「副本 - 线索 - 答案」的解谜玩法，以及会员订阅制的商业模式。

---

## 二、核心业务闭环说明

**宝瓶是宝箱在现实世界中的映射载体，二者始终一一对应，宝瓶本身不承载独立的游戏意义。** 换言之：

- 宝箱是内容主体（等级、宝物、概率权重都挂在宝箱上）；宝瓶只是宝箱投放到线下时生成的"壳"——一个带 GPS 坐标和二维码的物理/展示标识，供用户在现实世界定位、扫码。
- 因此宝瓶实体应尽量精简，只保留与"投放、定位、发现"相关的字段（见 3.1）。

---

## 三、实体模型

### 3.1 宝瓶（Bottle）

> 宝瓶是宝箱的现实载体，字段只服务于"投放、定位、发现"这条线，不承载宝物/等级等游戏内容（这些属于宝箱）。

| 字段 | 说明 |
|---|---|
| 6位数字编号 | 用于投放时快速识别和记录 |
| 宝箱码（二维码/小程序码） | 用于用户拾取宝瓶后取出扫码打开宝箱 |
| GPS 经纬度 | 宝瓶在现实世界的精确位置 |
| 指引 | 寻找宝瓶的精确提示, 结合投放照片一起，需寻宝罗盘 |
| 关联宝箱ID | 一一对应，投放宝箱时自动生成对应宝瓶 |
| 状态 | 待投放 / 已投放 / 已被发现 / 已回收 / 异常（丢失、损坏） |
| 投放人 & 投放时间 | 关联员工/管理员账号 |
| 投放照片 | 宝瓶实物照片 + 环境照片各一张 |
| 发现人 & 发现时间 | 记录被谁开启，用于统计 |
| 所属区域/城市 | 用于员工权限分区、热力图统计 |

### 3.2 宝箱（Box）
注: 同一个宝箱同一个用户只能开启一次

| 字段 | 说明 |
|---|---|
| 唯一编号 | |
| 等级 | 普通, 稀有, 史诗, 传说 |
| 宝物 | 图鉴/兑换券/空箱/线索/百科/道具 |
| 是否为隐藏宝箱 | 隐藏宝箱, 对所有用户(包括会员)不可见 |
| 状态 | 未开启 / 已开启 |
| 开启位置范围 | 用户距离宝箱位置范围内时开启有效, -1 表示不限制范围, 最小值 10米, 默认值 10米 |
| 总开启次数 | 宝箱总开启次数, -1 表示不限制次数, 默认值为 1 |
| 开启人 & 开启时间 | 列表, 记录所有开启宝箱的用户 |
| 关联宝瓶ID | 反向关联，一一对应 |

## 四、功能需求补充

### 4.1 用户侧

- 微信授权登录（暂不需要手机号强绑定，可预留字段, 为后期开发做准备）
- 浏览公开百科：**列表模式 + 地图模式 + 标签筛选 + 关键词搜索**
- 收藏（需登录）
- 内容分享（转发好友/朋友圈），可考虑分享裂变激励

### 4.2 会员侧

- 不支持线上购买和支持
- 周边宝瓶热力图（会员专属）：**接口不返回宝瓶真实 GPS 坐标, 返回飘移后的坐标(±10米)**，用户无法反推具体位置，需实地探索发现
- 扫码开箱完整流程：扫码 -> 查询宝瓶有效范围 → 校验用户 GPS 是否在宝瓶有效范围内 → 开箱动画 → 宝物自动入账背包
- 图鉴管理：已获得图鉴墙、未解锁图鉴以剪影/灰色展示、图鉴详情、集齐进度、图集图鉴管理查看筛选
- 背包管理细化：
  - 兑换券：列表、使用入口、核销记录查询
  - 道具：数量、使用记录、购买入口
- 线索/副本管理：副本列表（未解锁/进行中/已完成状态）、线索详情、提示使用计次、指引消耗道具、答案提交与校验反馈
- 消息通知：新宝瓶投放提醒、会员到期提醒、活动/上新通知

### 4.3 管理侧

- 数据看板：宝瓶投放/发现统计、宝箱开启率、副本完成率、营收统计、活跃用户数、留存率
- 角色权限分级：超级管理员 / 投宝人，各自权限范围
- 兑换券核销记录查询与对账
- 合规管理：隐私政策与用户协议版本管理
- 道具管理："道具"的创建/管理/查看

### 4.4 员工(投宝人)侧

- 投放宝瓶："领取待投放宝瓶库存"记录（员工从仓库/管理端领取多少个宝瓶待投放）
- 查看宝瓶：状态筛选（待投放/已投放/已被发现）
- 查看投放区域：地图可视化 + 按区域/城市统计
- 异常处理：宝瓶丢失、损坏、需回收时的上报与状态变更流程

---

## 五、关键业务规则确认情况

1. 宝瓶（实物）被发现后是否需要员工线下回收，还是可以长期放置不回收（不影响宝箱一一对应关系，仅涉及实物运维）？
  宝瓶由用户发现并保存或丢弃, 属于一次性消耗品, 对于长期未被发现的宝瓶, 自动标记为异常情况, 由投宝人线下回收, 当然视具体情况而定, 可能被环卫工或其他人(非会员)拿走了, 总之宝瓶属于投放后就无需管理的载体.

# 十一、防作弊

不考虑防作弊, 用户愿意修改GPS就让他修改, 愿意把宝箱码发给其他人就发, 愿意一个设备登录多个账号就登, 无所谓, 对我们来说, 没有任何损失, 甚至这也可以是游戏乐趣一部分.

我们只需要考虑做好数据权限和安全控制即可, 比如可开启次数为1的宝箱被开启了10次, 这就是我们的服务bug了, 这种情况必须杜绝. 比如接口返回了宝瓶真实的GPS数据这种情况, 必须杜绝等等.
