## 一、产品概述

「我的城市探秘」是一款基于 LBS（地理位置）+ 微信小程序的线下寻宝互动产品。

核心玩法：

**用户线下购买权益 → 扫专属欢迎小程序码进入开始探索页 → 激活会员或开始解密地图 → 到现场扫码发现宝箱 → Loading 自动入包 → 立即打开（弹窗）或稍后在背包打开 → 获得图鉴/道具/线索/百科解锁/虚拟宝箱/空箱 → 继续收集或推进副本解谜**

产品叠加「副本 - 线索 - 答案」的解谜玩法，并通过线下销售的会员套餐完成商业闭环。

目标城市: 特一线城市, 热门旅游旅居城市.

目标人群: 旅游和旅游人群, 好奇心重的人群, 

关键原则：

- 用户端不展示价格，不提供线上支付，不展示线上售卖说明。
- 所有销售在小程序外完成(线下或微信私域),由超级管理员在后台开通或续期会员。
- 管理员完成会员、解密地图或组合售卖后，系统生成绑定用户或临时用户的欢迎小程序码，用户扫码进入指定欢迎界面。
- 非会员可浏览公开百科；**非有效会员**（未登录 / 普通 / 待激活 / 过期）扫实体宝箱码时**静默回首页**：不进扫码结果页、不作提示、不领取、不发放奖励、不强制登录。
- 会员过期后保留账号、背包、图鉴、线索和历史记录，但失去首页宝箱层、扫码领取、打开宝箱、使用罗盘等会员权限（导航仍为 5 Tab 只读，见导航规划）。

## 二、版本范围与关键约束

### 2.1 v1 范围

- 微信小程序用户登录、公开百科浏览、收藏、分享。
- 用户端系统能力（详见 [小程序客户端系统权限规划v1.md](./小程序客户端系统权限规划v1.md)）：前台定位 + 地图蓝点、内扫相机、登录强制昵称头像、仅会员到期订阅消息、转发分享。
- 用户端信息架构：首页地图为主内容面；底部导航按账号态 2 Tab / 5 Tab（详见导航规划）；**无探索 Tab / 探索信息流**。
- 首页宝箱层（原「热力图」表述）：有效会员可见已投放非隐藏宝箱的漂移坐标点。
- 扫码领取实体宝箱、扫码结果页、背包开箱与「宝箱开启」弹窗、奖励入账（详见宝箱扫码规划）。
- 图鉴墙、图鉴详情、图鉴系列进度。
- 道具背包与「寻宝罗盘」使用。
- 会员欢迎小程序码、解密地图欢迎小程序码、VIP+解密地图组合欢迎小程序码。
- 解密地图道具：购买后进入指定地图欢迎页，并在有效会员状态下开始指定副本或地图玩法。
- 副本最小闭环：副本列表、线索解锁、答案提交、完成状态、完成奖励。
- 超级管理员后台：用户查询、会员开通/续期、套餐配置、宝箱管理、奖励池管理、百科管理、图鉴管理、道具管理、副本/线索管理、数据看板、操作日志。

### 2.2 v1 不做

- 用户端线上支付。
- 用户端价格展示。
- 用户端道具售卖。
- 探索 Tab 与探索综合信息流。
- 多后台角色或员工分区权限。
- 用户投稿百科。
- 强防作弊系统。
- 线下实体宝箱自动回收、自动巡检、自动判定丢失。
- 用户端手机号授权 / 绑定入口（字段可预留，不暴露入口）。
- 相册写入 / 选图、麦克风、剪贴板、后台定位、连续轨迹画像。
- 活动上新微信订阅消息；消息通知入口 / 站内消息中心；分享海报保存到相册。
- 访客首页定位上报落库；无定位 / 拒权时按范围 `-1` 放行领取。

### 2.3 运营人工处理范围

线下实体宝箱的回收、固定放置、丢失、损坏、长期未发现等问题，由超级管理员或运营人员在线下人工处理；系统只提供后台状态、备注和操作日志，不自动推动线下处置流程。

## 三、术语与核心定义

| 术语 | 定义 |
|---|---|
| 用户 | 微信小程序账户。注册后即为普通用户，会员只是该账户的权限状态。 |
| 临时用户 | 管理员线下售卖时可先创建的待绑定对象，仅记录称呼和备注；用户首次扫码欢迎码后自动绑定微信账户。 |
| 待激活会员 | 管理员已完成会员开通或组合售卖，但用户尚未在欢迎页点击「开始探索」；未激活前不能使用会员能力。 |
| 有效会员 | 当前时间早于会员到期时间的用户。 |
| 过期会员 | 曾开通过会员但当前已过期的用户；历史数据保留，会员权限失效。 |
| 探秘入口小程序码 | 管理员完成线下权益发放后生成的用户入口码，类型包括会员激活码、解密地图开始码、VIP+解密地图组合码。 |
| 解密地图 | 一种道具，绑定指定副本或地图；用户扫码进入专用欢迎页，并在有效会员状态下开始地图玩法。 |
| 组合售卖 | VIP 会员套餐 + 解密地图的线下组合权益，生成一个组合欢迎码。 |
| 实体宝箱 | 线下投放的真实宝箱，用户需要到现场扫码领取。 |
| 虚拟宝箱 | 作为奖励进入背包的宝箱，不对应线下点位，不需要 GPS 校验。 |
| 领取 | 用户扫码实体宝箱，Loading 内通过会员、位置、次数等校验后**自动**将宝箱加入背包并消耗一次可领取次数。 |
| 打开 | 用户打开未打开的背包宝箱，系统根据宝箱关联的奖励池发放奖励；UI 为当前页「宝箱开启」弹窗。 |
| 立即打开 | 扫码结果页成功待开态的主操作：领取已在 Loading 完成，点击后在**当前页**弹出「宝箱开启」弹窗执行开箱。 |
| 扫码结果页 | 仅有效会员进入；Loading（定位上报 + 校验 + 可能入包）后展示成功待开 / 已领 / 分类失败。 |
| 首页宝箱层 | 首页地图上有效会员可见的实体宝箱点位层，使用漂移坐标；旧称「热力图」，合稿后优先用本词。 |
| 奖励池 | 宝箱或副本完成奖励的发放规则容器，承载固定奖励、权重、库存、等级等规则。 |
| 寻宝罗盘 | 会员道具。对指定宝箱或线索消耗 1 个后，永久解锁真实坐标或精确指引。 |
| 漂移坐标 | 实体宝箱投放时生成并持久保存的展示坐标，用于首页宝箱层；不是每次请求随机生成。 |

核心规则：

- 有效会员才能领取实体宝箱。
- 有效会员才能打开任何宝箱，包括已领取的实体宝箱和奖励获得的虚拟宝箱。
- 待激活会员不等于有效会员，不能使用首页宝箱层、扫码领取、开箱、罗盘和副本完整能力。
- 会员有效期从用户在会员欢迎页或组合欢迎页点击「开始探索」时开始计算。
- 解密地图必须在有效会员状态下开始；VIP+解密地图组合码可在点击「开始探索」时同时激活会员并开始地图。
- 同一用户对同一个实体宝箱只能领取一次。
- 实体宝箱领取后可在任意位置打开，不再要求回到宝箱现场。
- 实体宝箱领取时原子消耗可领取次数，避免并发超发。
- 宝箱的等级、奖励类型、概率、库存主要由奖励池承载，实体宝箱只关联奖励池。
- 6 位数字编号只用于运营识别；用户扫码使用不可枚举的小程序码**多参数**（前端原样上报，服务端解析），不作 6 位编号查询。
- 首页宝箱层只返回漂移坐标；真实 GPS 坐标只在用户使用寻宝罗盘解锁后返回。

## 四、实体模型

### 4.1 用户

| 字段 | 说明 |
|---|---|
| 用户ID | 系统内部唯一编号。 |
| 微信 openid / unionid | 微信小程序授权登录必需。 |
| 用户码 | 用户端展示给线下运营识别的编号或二维码，用于后台开通会员。 |
| 昵称 / 头像 | 登录强制：`chooseAvatar` 选头像 + 昵称填写组件；不用旧版一键 `getUserProfile` 作为主路径（详见权限规划）。 |
| 手机号 | v1 **不**暴露绑定入口；数据模型可预留字段。 |
| 会员状态 / 到期时间 | 普通、待激活会员、有效会员、过期会员；到期时间决定权限。 |
| 背包 | 关联用户拥有的图鉴、道具、宝箱、线索、空箱记录。 |
| 百科收藏 | 用户收藏的百科内容。 |
| 百科解锁记录 | 通过奖励或运营发放解锁的百科。 |
| 注册时间 / 最后登录时间 | 用户行为统计。 |

### 4.2 会员套餐

| 字段 | 说明 |
|---|---|
| 套餐ID | 后台唯一编号。 |
| 套餐标题 | 后台和管理记录使用。 |
| 套餐时长 | 按天配置，如 3 天、2 周、1 个月。 |
| 赠送寻宝罗盘数量 | 开通或续期后一次性发放。 |
| 后台可见标价 | 仅供后台配置和运营参考，用户端不展示。 |
| 欢迎页配置 | 欢迎页标题、说明、封面、开始目标。 |
| 欢迎码有效期配置 | 本套餐生成的会员欢迎码未使用前的有效期。 |
| 状态 | 启用 / 停用。 |

会员叠加规则：

- 管理员开通会员后，用户进入待激活会员状态。
- 用户在会员欢迎页点击「开始探索」时，系统才计算新到期时间并转为有效会员。
- 用户当前为有效会员时，再次激活从原到期时间继续顺延。
- 用户当前非有效会员时，从激活时间开始计算新套餐时长。
- 每次开通或续期都按套餐配置发放寻宝罗盘。

### 4.3.1 临时用户

| 字段 | 说明 |
|---|---|
| 临时用户ID | 后台唯一编号。 |
| 称呼 | 管理员线下识别使用。 |
| 备注 | 线下订单、现场说明或工作人员备注。 |
| 绑定用户ID | 首次扫码欢迎码后绑定的微信小程序用户。 |
| 状态 | 未绑定 / 已绑定 / 已取消。 |
| 创建管理员 / 创建时间 | 后台审计。 |

临时用户规则：

- 新用户可先由管理员创建临时用户并完成售卖发码。
- 临时用户不要求手机号或短信校验。
- 首次扫码欢迎码并登录的小程序用户自动绑定该临时用户和对应权益。
- 已绑定临时用户不可再次绑定其他微信账户。

### 4.3.2 探秘入口小程序码

| 字段 | 说明 |
|---|---|
| 入口码ID | 系统内部唯一编号。 |
| 小程序码参数 | 不可枚举随机参数。 |
| 类型 | 会员激活 / 解密地图开始 / VIP+解密地图组合。 |
| 绑定用户ID / 临时用户ID | 二选一，确定该码归属。 |
| 目标对象ID | 会员开通记录、解密地图购买记录或组合销售记录。 |
| 欢迎页配置快照 | 生成时的标题、说明、封面、开始目标。 |
| 有效期 | 按套餐或解密地图配置生成。 |
| 状态 | 未使用 / 已进入 / 已完成 / 已过期 / 已作废。 |
| 扫码日志 | 记录扫码用户、时间、结果和失败原因。 |

入口码规则：

- 管理员完成开通、解密地图售卖或组合售卖后，系统立即生成入口小程序码。
- Web 后台和管理员小程序均可展示该码给管理员转交用户。
- 未完成前，本人或绑定后的同一用户可重复扫码进入欢迎页。
- 已完成后，本人复扫进入对应的已激活或继续探索状态，不重复激活、不重复开始。
- 非归属用户扫码时，用户端统一提示无法进入该探秘入口并返回首页，后台记录具体失败原因。

### 4.4 实体宝箱

| 字段 | 说明 |
|---|---|
| 宝箱ID | 系统内部唯一编号。 |
| 6 位数字编号 | 运营投放、盘点、沟通时快速识别。 |
| 扫码参数 | 不可枚举的小程序码多参数，前端原样上报、服务端解析分流。 |

## 五、功能需求

### 5.1 用户侧信息架构与基础能力

底部导航与首页细则以 [首页与全局导航规划v1.md](./首页与全局导航规划v1.md) 为准，摘要如下：

| 账号态 | 底部 Tab |
|---|---|
| 未登录 / 普通 / 待激活 | 首页 \| 我的 |
| 有效会员 / 过期会员 | 首页 \| 副本 \| 图鉴 \| 背包 \| 我的 |

- 打开小程序默认「首页」；首页为**地图模式**主内容面，叠加公开百科点；（仅有效会员）首页宝箱层。
- **无探索 Tab**、无探索综合信息流；百科路径：首页地图 / 首页搜索 /「我的 → 百科收藏」（**无**「我的 → 百科浏览」；详见导航与「我的」规划）。
- **定位（首页）**：进入首页即申请（含访客）；`map` + `show-location` 蓝点 + `getLocation`（建议 gcj02）；单次取点 +「定位到我」，不持续跟随；GPS → 逆地理 **adcode**（市级归一）匹配已开通运营城；失败/拒权/未开通 → 默认运营城市并提示；切城/刷点后地图视野由客户端按该城**点位集合** `includePoints` 适配（非城心）；**访客仅本机、不落库**；**已登录**首页定位可上报，**每会话 ≤1 条**（细则见导航与权限规划）。
- 微信授权登录：`wx.login` + 强制昵称头像 + 产品协议勾选（**默认不勾选**，未勾选不可登录）；产品协议与微信隐私保护指引**双层并存**；访客定位不额外弹产品协议（详见权限规划）。
- 展示用户码，供线下运营识别（「我的」：文本 + 复制 + 二维码；详见「我的」规划）。
- 浏览公开百科：首页地图点、搜索（仅百科标题/标签，不搜宝箱编号）、详情；收藏需登录；「我的」仅收藏入口。
- 分享内容：支持转发好友或可选朋友圈；**不**申请相册、不做海报存相册。
- **无**消息通知 / 站内消息中心入口（v1）。
- 「我的」页：分态内容、设置/关于、登录退出落点见 [我的页面规划v1.md](./我的页面规划v1.md)。摘要：未登录极简（登录 + 百科收藏 + 关于）；设置（仅已登录）= 定位/相机去开启 + 退出登录；协议在关于；**到期提醒**开关在身份区旁、**仅有效会员**。
- 用户端不展示价格，不提供线上支付 / 开通购买入口。
- 扫会员欢迎小程序码：登录并通过归属校验后进入会员欢迎页，点击「开始探索」后激活会员。
- 扫解密地图欢迎小程序码：登录并通过归属校验后进入解密地图欢迎页，点击「开始解密」后校验有效会员并开始地图。
- 扫 VIP+解密地图组合码：登录并通过归属校验后进入组合欢迎页，点击「开始探索」后同时激活会员并开始地图。
- **非有效会员**扫实体宝箱码：**静默回首页**（不进结果页、无提示、不领取、不强制登录）。

## 六、关键业务规则

### 6.1 会员开通

1. 用户在线下完成付款。
2. 已登录用户展示用户码；新用户可由管理员创建临时用户。
3. 超级管理员在后台扫码、输入用户码或选择临时用户定位对象。
4. 超级管理员选择套餐，填写实收金额和备注。
5. 系统创建待激活会员开通记录，发放套餐赠送的寻宝罗盘，生成会员欢迎小程序码。
6. 用户扫码进入会员欢迎页，点击「开始探索」后系统计算到期时间并转为有效会员。
7. 若本次为**非有效→有效**跃迁：激活成功后按需请求**会员到期**订阅授权（已 accept 跳过；未 accept 才弹；reject/ban 不追弹；**仅有效会员**可在「我的」身份区旁补开，见权限规划与「我的」规划）。有效期内纯顺延不自动拉订阅。
8. 系统写入会员开通记录、欢迎码扫码日志和操作日志。

用户端不展示价格，不提供线上支付路径。

### 6.1.1 欢迎小程序码与激活

- 欢迎小程序码参数必须不可枚举。
- 欢迎小程序码必须绑定用户或临时用户，不允许未绑定权益的通用激活码。
- 用户扫码后必须登录；已绑定用户码必须与当前登录用户匹配。
- 临时用户欢迎码首次扫码登录后自动绑定当前微信小程序用户。
- 会员欢迎码扫码验证通过后进入欢迎页；只有点击「开始探索」才完成会员激活。
- 会员有效期从「开始探索」成功时起算；若用户已是有效会员，则从原到期时间顺延。
- 欢迎码未使用前的有效期按套餐或解密地图配置生成。
- 已完成的欢迎码本人可复扫，但不得重复激活、重复发放道具或重复开始地图。

### 6.1.2 解密地图售卖与开始

- 解密地图是道具，必须绑定指定副本或地图起点。
- 解密地图购买记录必须记录实收金额、管理员、备注，并进入线下营收统计。
- 单独售卖解密地图时，管理员端必须校验目标用户为有效会员，否则不允许提交，需改用 VIP+解密地图组合售卖。
- 用户扫码进入解密地图欢迎页后，点击「开始解密」时再次校验有效会员。
- 开始成功后，系统标记该地图已开始，后续用户可从欢迎码复扫、背包或副本入口继续进度。

### 6.1.3 VIP+解密地图组合售卖

- 组合售卖生成一条组合销售记录，内部关联会员开通记录和解密地图购买记录。
- 组合售卖只生成一个组合欢迎小程序码。
- 用户扫码进入组合欢迎页，点击「开始探索」后同时完成会员激活和解密地图开始。
- 组合欢迎码已完成后本人复扫进入继续探索状态，不重复激活会员、不重复开始地图。

### 6.2 实体宝箱领取

领取必须同时满足：

- 用户已登录。
- 用户为有效会员。
- 扫码参数有效且不可被枚举。
- 宝箱状态为已投放。
- 用户在宝箱开启位置范围内；范围为 -1 时服务端在**已有有效 lat/lng** 时跳过距离拦截；**缺定位 / 拒权一律不可领（含 -1）**（前端仍上报定位，见宝箱扫码规划 / 权限规划）。
- 用户未领取过该实体宝箱。
- 宝箱未达到可领取总次数；总次数为 -1 时不限制。

领取成功后（扫码结果页 Loading 内自动完成）：

- 系统原子增加已领取次数。
- 系统创建领取记录。
- 系统向用户背包写入一个未打开的实体宝箱物品。
- 结果页进入成功待开：主「立即打开」（当前页弹窗开箱）、次「查看背包」。

### 6.3 背包宝箱打开

打开必须满足：

- 用户已登录。
- 用户为有效会员。
- 背包宝箱属于当前用户。
- 背包宝箱状态为未打开。

打开成功后：

- 系统按奖励池规则生成奖励。
- 奖励分类入账。
- 宝箱状态变为已打开。
- 写入开箱结果和操作日志或用户行为日志。

### 6.4 背包与奖励入账

| 类型 | 入账方式 |
|---|---|
| 图鉴 | 写入背包，并在图鉴墙展示。 |
| 道具 | 写入背包，增加道具数量或生成道具实例。 |
| 实体宝箱扫码领取结果 | 写入背包，状态为未打开；这不是奖励池抽取结果。 |
| 虚拟宝箱 | 写入背包，状态为未打开。 |
| 线索 | 写入背包，同时写入用户线索解锁记录。 |
| 百科解锁 | 写入用户百科解锁记录。 |
| 空箱 | 写入背包，作为开箱历史和展示记录。 |

### 6.5 隐藏宝箱

- 隐藏宝箱不进入首页宝箱层。
- 隐藏宝箱仍可扫码领取，规则与普通实体箱一致（会员、位置、次数等）。
- 隐藏宝箱的发现路径来自副本线索、百科内容或线下物料。

### 6.6 首页宝箱层与坐标

- 实体宝箱投放时生成一次漂移坐标并持久保存。
- 首页宝箱层接口只返回漂移坐标，不返回真实 GPS；仅有效会员可拉取。
- 使用寻宝罗盘解锁后，才可查看指定宝箱真实 GPS 和精确指引及投放照片。
- 未登录 / 普通 / 待激活 / 过期：不加载宝箱层。
- 旧文档「热力图」与「首页宝箱层」同义时，以「首页宝箱层」为准。

### 6.7 寻宝罗盘

- 使用前校验用户为有效会员。
- 使用前校验用户拥有足够罗盘数量。
- 对同一用户和同一目标对象，解锁记录永久有效。
- 已解锁对象再次查看不消耗罗盘。
- 罗盘消耗和解锁记录必须在同一事务内完成。

### 6.8 副本与线索

- 副本可配置线性解锁或自由探索。
- 线性解锁时，未完成前置线索不得提交后续线索答案。
- 自由探索时，用户可查看已解锁的任意线索。
- 答案提交后按规范化规则匹配。
- 副本完成后按配置发放完成奖励。

## 七、业务流程

### 7.1 线下开通会员流程

线下收款 → 用户展示用户码或管理员创建临时用户 → 超级管理员后台识别对象 → 选择套餐并填写实收金额 → 系统创建待激活会员记录 → 发放寻宝罗盘 → 生成会员欢迎小程序码 → 用户扫码进入欢迎页 → 点击开始探索 → 系统激活会员并计算到期时间。

### 7.2 扫码领取实体宝箱流程

完整规格见 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md)。摘要：

1. **入口**：有效会员首页悬浮扫码或「我的」扫一扫普通按钮；或微信扫一扫外扫。内扫与外扫业务规则一致。
2. **扫码界面**：进入内扫页再申请相机；相机拒权 → 中断页（去设置），不进结果页；去设置返回后自动重试；**返回/关闭 → 首页 Tab**；外扫不占小程序相机权限。
3. **分流**：多参数原样上报服务端；宝箱码 / 欢迎码等按类型路由。
4. **非有效会员**（未登录 / 普通 / 待激活 / 过期）扫实体宝箱码：**静默回首页**（不进结果页、无提示、不领取、不强制登录）；服务端记日志。
5. **有效会员 → 扫码结果页 Loading**：始终取定位并上报 → 服务端校验会员、参数、宝箱状态、位置、单用户已领、总次数 → 全通过则**原子入包**；定位拒权去设置后返回自动重试 Loading；**拒权 / 无 lat/lng 一律不入包（含范围 `-1`）**。
6. **Loading 之后**：
   - 成功待开：氛围信息 + 主「立即打开」+ 次「查看背包」
   - 已领取：未开可「立即打开」；已开则「查看背包」
   - 分类失败：距离不足（展示真实直线距离米 + 重定位）、定位拒权/失败、次数已满、状态不可领、参数无效等；均不入包
   - Loading 超时 / 网络失败：**直接回首页**
7. **立即打开**：当前页弹出「宝箱开启」弹窗（与背包开箱共用）；开箱时再次校验有效会员。

### 7.3 背包开箱流程

用户打开背包宝箱 → 校验会员有效 → 校验物品归属与未打开状态 → 执行奖励池 → 奖励入账 → 宝箱标记已打开 → 当前页弹出「宝箱开启」弹窗展示奖励（与扫码「立即打开」共用）。

### 7.4 寻宝罗盘流程

用户选择目标宝箱或线索 → 校验会员有效 → 校验是否已解锁 → 校验罗盘数量 → 消耗 1 个罗盘 → 写入解锁记录 → 展示真实坐标或精确指引。

### 7.5 副本流程

用户进入副本列表 → 查看可进入副本 → 解锁或查看线索 → 查看免费提示或使用罗盘查看精确指引 → 提交答案 → 更新线索状态 → 全部必要线索完成后标记副本完成 → 发放完成奖励。

### 7.6 解密地图购买与开始流程

已有有效会员线下购买解密地图 → 管理员后台识别用户 → 选择解密地图并填写实收金额和备注 → 系统创建解密地图购买记录和背包道具 → 生成解密地图欢迎小程序码 → 用户扫码进入解密地图欢迎页 → 点击开始解密 → 系统标记地图已开始并进入指定地图起点。

### 7.7 VIP+解密地图组合售卖流程

线下收款 → 管理员识别用户或创建临时用户 → 选择会员套餐和解密地图 → 填写组合实收金额和备注 → 系统创建组合销售记录、待激活会员记录和解密地图购买记录 → 生成一个组合欢迎小程序码 → 用户扫码进入组合欢迎页 → 点击开始探索 → 系统激活会员、标记地图已开始并进入指定地图起点。

| 状态 | 进入条件 | 权限 |

### 8.7 解密地图开始状态

## 十一、异常处理

| 场景 | 处理 |
|---|---|
| 非有效会员扫码实体宝箱 | 静默回首页；不进结果页、无提示、不领取；不返回宝箱敏感信息；记日志。 |
| 扫码参数无效或不存在 | 仅有效会员：结果页提示宝箱不存在或已失效；非有效会员仍静默回首页。 |
| 宝箱未投放、停用、已回收、异常 | 有效会员结果页提示不可领取；不入包。 |
| 用户拒绝定位授权 | 有效会员：需定位失败态 + 去设置；返回后自动重试 Loading；不入包。 |
| 定位系统失败 | 有效会员：可重试定位；不入包。 |
| 用户不在范围内 | 有效会员：提示需靠近，展示真实直线距离（米），可重定位；不展示真实坐标；不入包。 |
| 用户已领取过该实体宝箱 | 有效会员：已领态；未开可立即打开弹窗；已开引导查看背包；不二次扣次。 |
| 宝箱领取次数已满 | 有效会员：提示已被领取完；不入包。 |
| 并发领取冲突 | 以服务端事务结果为准，失败方走已领完/已领/业务失败语义。 |
| Loading 超时 / 网络失败 | 直接回首页，用户再次扫码。 |
| 相机权限拒绝（内扫） | 中断页（说明 + 去设置），不进结果页；返回后自动重试打开扫码。 |
| 会员过期时打开宝箱 | 阻断打开，奖励不发放，宝箱保持未打开。 |
| 奖励库存不足 | 从本次候选集中移除后继续抽取；全部耗尽则发放空箱记录。 |
| 罗盘数量不足 | 提示罗盘不足，不创建解锁记录。 |
| 已解锁对象重复查看 | 直接展示已解锁内容，不重复消耗罗盘。 |
| 待激活会员使用会员能力 | 按非有效会员阻断，不开放首页宝箱层、领取、开箱、罗盘和副本完整流程。 |
| 欢迎码用户不匹配 | 用户端统一提示无法进入该探秘入口并返回首页；后台记录具体原因。 |
| 欢迎码过期或作废 | 用户端统一提示无法进入该探秘入口并返回首页；后台记录具体原因。 |
| 解密地图开始时用户不是有效会员 | 直接阻断并返回首页，不开始地图。 |
| 单独售卖解密地图给非有效会员 | 管理员端不允许提交，提示需选择 VIP+解密地图组合售卖。 |
| 后台开通会员找不到用户 | 可要求用户先登录小程序并展示用户码，或由管理员创建临时用户。 |
| 后台配置缺少必要字段 | 不允许启用对应宝箱、奖励池、副本或线索。 |

## 十二、防作弊与安全边界

v1 不做强防作弊，接受以下玩法性作弊风险：

- 用户修改 GPS。
- 用户把宝箱码分享给其他人。
- 用户一个设备登录多个账号。

必须防止的服务端问题：

- 可领取总次数为 1 的宝箱被领取多次。
- 同一用户重复领取同一实体宝箱。
- 奖励库存并发超发。
- 首页宝箱层或普通接口泄漏真实 GPS。
- 用户通过枚举 6 位编号或扫码参数获取宝箱信息。
- 非会员绕过客户端直接调用会员接口。
- 用户伪造背包物品、会员状态、罗盘数量或奖励结果。
- 用户伪造欢迎码参数激活会员或开始解密地图。
- 非归属用户使用他人的会员欢迎码、解密地图欢迎码或组合欢迎码。
- 已完成欢迎码被重复用于重复激活、重复发放道具或重复开始地图。
- 后台操作缺少日志导致无法追责。

安全要求：

- 所有会员、位置、次数、库存、归属校验必须在服务端完成。
- 客户端传入的用户状态、坐标、背包物品、奖励结果不可直接信任。
- 扫码多参数使用不可枚举随机值；6 位编号不作为用户端查询凭证。
- 欢迎码参数使用不可枚举随机值；用户归属、临时用户绑定、有效期、权益状态必须在服务端校验。
- 欢迎码验证失败的用户端提示统一处理；后台必须保留具体失败原因用于审计。
- 真实 GPS 只在罗盘解锁后按指定对象返回。
- 涉及领取、开箱、库存扣减、罗盘消耗、会员激活、地图开始的操作必须使用事务或等效原子机制。
- 对高频接口做服务保护限频，限频目的为保护系统稳定，不作为强反作弊承诺。

## 十五、验收场景

### 15.1 导航与首页（详见导航规划）

- 未登录打开小程序：默认首页，底部仅「首页 | 我的」，地图仅百科点，无扫码入口；收藏触发登录。
- 普通 / 待激活：2 Tab；无宝箱层、无扫码入口；待激活欢迎码激活成功后切 5 Tab 并出现宝箱层与扫码入口。
- 有效会员：5 Tab；首页百科 + 宝箱双 icon；首页悬浮扫码 + 「我的」扫码按钮；点位卡片不完成领取；搜索只出百科。
- 过期会员：5 Tab 只读；无宝箱层、无扫码入口；顶部到期软提示无价格。
- 首页宝箱层接口：仅有效会员获得非隐藏宝箱漂移坐标，不返回真实 GPS。

### 15.2 宝箱扫码（详见宝箱扫码规划）

- 非有效会员扫实体宝箱码（内扫/外扫）：静默回首页，不领取、不消耗次数、不返回真实 GPS、不强制登录。
- 有效会员范围内扫码：Loading 自动领取成功，已领取次数 +1，背包出现未打开宝箱，结果页成功待开。
- 有效会员点「立即打开」：当前页「宝箱开启」弹窗开箱，奖励按规则入账。
- 有效会员点「查看背包」：进入背包列表。
- 范围外：失败态展示真实直线距离（米），可重定位；不展示真实坐标；不入包。
- 已领再扫：已领态；未开可弹窗开箱；不二次扣次。
- 可领取总次数为 1 的实体宝箱被多人并发扫码：只有一个用户领取成功。
- Loading 超时 / 网络失败：回首页。
- 隐藏宝箱：不进宝箱层，扫码规则与普通箱一致。

### 15.3 系统权限（详见权限规划）

- 访客打开小程序进首页：申请定位；拒权落默认运营城市并提示；同意后蓝点与 adcode 匹配运营城；**坐标不上报、不落库**；不额外弹产品协议。
- 已登录进首页：成功定位可上报；同会话最多 1 条（可覆盖更新）。
- 登录：未勾选协议不可登录；缺昵称或头像不可登录；齐备可登录；无手机号绑定入口。
- 首次调定位 / 相机前：未同意微信隐私保护指引则先完成隐私授权。
- 内扫拒相机：中断页 → 去设置 → 返回自动重进扫码；**返回/关闭 → 首页**；外扫不占用小程序相机权限。
- 扫码 Loading：拒定位「去设置」且不入包；系统失败可重试；范围 `-1` 有定位时可领；**拒权 + `-1` 仍不可领**。
- 非有效→有效激活成功：未 accept 则拉到期订阅；已 accept 跳过；有效期内纯顺延不拉；过期不发送；到期开关仅有效会员身份区旁可见。
- 导航与「我的」：**无**消息通知入口；无活动上新订阅模板；**无**「我的 → 百科浏览」。
- 「我的」：未登录极简；设置（已登录）= 定位/相机去开启 + 退出；协议在关于；详见「我的」规划。
- 分享不申请相册权限；无相册 / 麦克风 / 剪贴板 / 后台定位能力入口。

### 15.4 其它核心场景

- 用户领取实体宝箱后离开现场：可在任意位置打开，但必须仍是有效会员；开箱为弹窗。
- 会员过期后打开背包宝箱：打开失败，宝箱保持未打开，奖励不发放。
- 使用寻宝罗盘解锁宝箱：消耗 1 个罗盘，永久显示该宝箱真实 GPS 和精确指引。
- 重复查看已解锁宝箱位置：不再消耗罗盘。
- 奖励池库存部分耗尽：系统移除耗尽候选并继续抽取。
- 奖励池全部候选耗尽：开箱结果为空箱，并写入背包记录。
- 线索答案提交：大小写、首尾空格、全半角差异不影响正确答案匹配。
- 后台开通会员：通过用户码定位用户，记录套餐、实收金额、备注，正确顺延到期时间并发放罗盘。
- 会员欢迎码激活：管理员开通后用户为待激活会员；用户扫本人欢迎码进入欢迎页，点击开始探索后转为有效会员，到期时间从点击时间起算；非有效→有效成功后按需请求会员到期订阅授权（未 accept 才弹）。
- 待激活会员使用会员能力：首页宝箱层、扫码领取、开箱、罗盘和副本完整流程均被阻断。
- 临时用户绑定：管理员创建临时用户并生成欢迎码；用户首次扫码登录后自动绑定临时用户和权益。
- 非归属用户扫欢迎码：用户端统一提示无法进入该探秘入口并返回首页，后台记录具体失败原因。
- 有效会员购买解密地图：管理员生成解密地图欢迎码；用户扫码进入地图欢迎页，点击开始解密后地图状态变为已开始。
- 非有效会员单独购买解密地图：管理员端不允许提交，提示选择 VIP+解密地图组合售卖。
- VIP+解密地图组合售卖：系统生成一个组合欢迎码；用户点击开始探索后同时激活会员并开始指定地图。
- 已完成欢迎码复扫：本人可进入已激活或继续探索状态，不重复激活、不重复发放道具、不重复开始地图。
- 欢迎码过期：用户端不可进入欢迎页，管理员可在记录详情重新生成。
- 用户端全流程：不展示价格，不提供线上支付入口。
