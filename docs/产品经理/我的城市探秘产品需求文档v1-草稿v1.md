# 我的城市探秘产品需求文档 v1

---

## 一、产品概述

「我的城市探秘」是一款基于 LBS（地理位置）+ 微信小程序的线下寻宝互动产品。

核心玩法：

**会员到现场扫码发现宝箱 → 领取到背包或立即打开 → 获得图鉴/道具/线索/百科解锁/虚拟宝箱/空箱 → 继续收集或推进副本解谜**

产品叠加「副本 - 线索 - 答案」的解谜玩法，并通过线下销售的会员套餐完成商业闭环。

关键原则：

- 用户端不展示价格，不提供线上支付，不展示线上售卖说明。
- 所有销售在线下完成，由超级管理员在管理后台为用户开通或续期会员。
- 非会员可浏览公开百科；非会员扫码宝箱时阻断并返回首页，不领取宝箱、不发放奖励。
- 会员过期后保留账号、背包、图鉴、线索和历史记录，但失去领取宝箱、打开宝箱、查看宝箱位置等会员权限。

---

## 二、版本范围与关键约束

### 2.1 v1 范围

- 微信小程序用户登录、公开百科浏览、收藏、分享。
- 会员热力图、扫码领取宝箱、背包开箱、奖励入账。
- 图鉴墙、图鉴详情、图鉴系列进度。
- 道具背包与「寻宝罗盘」使用。
- 副本最小闭环：副本列表、线索解锁、答案提交、完成状态、完成奖励。
- 超级管理员后台：用户查询、会员开通/续期、套餐配置、宝箱管理、奖励池管理、百科管理、图鉴管理、道具管理、副本/线索管理、数据看板、操作日志。

### 2.2 v1 不做

- 用户端线上支付。
- 用户端价格展示。
- 用户端道具售卖。
- 多后台角色或员工分区权限。
- 用户投稿百科。
- 强防作弊系统。
- 线下实体宝箱自动回收、自动巡检、自动判定丢失。

### 2.3 运营人工处理范围

线下实体宝箱的回收、固定放置、丢失、损坏、长期未发现等问题，由超级管理员或运营人员在线下人工处理；系统只提供后台状态、备注和操作日志，不自动推动线下处置流程。

## 三、术语与核心定义

| 术语 | 定义 |
|---|---|
| 用户 | 微信小程序账户。注册后即为普通用户，会员只是该账户的权限状态。 |
| 有效会员 | 当前时间早于会员到期时间的用户。 |
| 过期会员 | 曾开通过会员但当前已过期的用户；历史数据保留，会员权限失效。 |
| 实体宝箱 | 线下投放的真实宝箱，用户需要到现场扫码领取。 |
| 虚拟宝箱 | 作为奖励进入背包的宝箱，不对应线下点位，不需要 GPS 校验。 |
| 领取 | 用户扫码实体宝箱，通过会员、位置、次数等校验后，将宝箱加入背包并消耗一次宝箱可领取次数。 |
| 打开 | 用户打开背包中的宝箱，系统根据宝箱关联的奖励池发放奖励。 |
| 立即打开 | 扫码页上的快捷操作。本质是先完成领取，再立即打开刚领取的背包宝箱。 |
| 奖励池 | 宝箱或副本完成奖励的发放规则容器，承载固定奖励、权重、库存、等级等规则。 |
| 寻宝罗盘 | 会员道具。对指定宝箱或线索消耗 1 个后，永久解锁真实坐标或精确指引。 |
| 漂移坐标 | 实体宝箱投放时生成并持久保存的展示坐标，用于会员热力图；不是每次请求随机生成。 |

核心规则：

- 有效会员才能领取实体宝箱。
- 有效会员才能打开任何宝箱，包括已领取的实体宝箱和奖励获得的虚拟宝箱。
- 同一用户对同一个实体宝箱只能领取一次。
- 实体宝箱领取后可在任意位置打开，不再要求回到宝箱现场。
- 实体宝箱领取时原子消耗可领取次数，避免并发超发。
- 宝箱的等级、奖励类型、概率、库存主要由奖励池承载，实体宝箱只关联奖励池。
- 6 位数字编号只用于运营识别；用户扫码使用不可枚举的随机 token 或小程序码参数。
- 热力图只返回漂移坐标；真实 GPS 坐标只在用户使用寻宝罗盘解锁后返回。

| 字段 | 说明 |
|---|---|
| 虚拟宝箱ID | 系统内部唯一编号。 |
| 奖励池ID | 打开时使用的奖励规则。 |
| 来源 | 奖励池发放、后台发放、活动发放等。 |
| 背包物品ID | 进入用户背包后的实例记录。 |

虚拟宝箱不关联线下 GPS，不需要扫码或位置校验，但打开时仍要求用户为有效会员。

### 4.6 奖励池

| 字段 | 说明 |
|---|---|
| 奖励池ID | 系统内部唯一编号。 |
| 名称 | 后台识别。 |
| 模式 | 固定奖励 / 权重库存。 |
| 奖励条目 | 奖励类型、目标对象、数量、等级、权重、库存等配置。 |
| 状态 | 草稿 / 启用 / 停用。 |
| 创建人 / 更新时间 | 后台审计。 |

奖励类型：

- 图鉴
- 道具
- 线索
- 百科解锁
- 空箱

奖励池模式：

- 固定奖励：支持配置多条奖励，打开时全部发放。
- 权重库存：支持按奖励类型配置权重，也支持在类型下按等级配置权重。例如图鉴 90%、道具 10%；图鉴内普通 90%、稀有 8%、史诗 2%。

库存耗尽规则：

- 抽取时如果命中的候选类型、等级或具体奖励库存不足，则从本次候选集中移除后继续抽取。
- 如果所有候选都耗尽，则结果为空箱。
- 成功发放有库存奖励时，必须原子扣减库存，避免并发超发。

### 4.14 寻宝罗盘

寻宝罗盘是 v1 的核心道具。

使用规则：

- 使用对象为指定实体宝箱或指定线索。
- 每次解锁消耗 1 个寻宝罗盘。
- 同一用户对同一对象已解锁时，再次查看不重复消耗。
- 对实体宝箱使用后，永久解锁该宝箱真实 GPS 和精确指引和投放照片。
- 对线索使用后，永久解锁该线索精确指引。
- 用户会员过期后，已解锁记录保留；再次成为有效会员后可继续查看会员权限内容。

## 五、功能需求

### 5.1 用户侧

- 微信授权登录。
- 展示用户码，供线下运营在后台识别用户。
- 浏览公开百科：列表模式、地图模式、标签筛选、关键词搜索。
- 收藏百科：需登录。
- 分享内容：支持转发好友或朋友圈。
- 非会员扫码宝箱：阻断并直接返回首页，不作任何提示，不展示宝箱详情，不领取宝箱。

### 5.2 会员侧

- 周边宝箱热力图：只展示已投放且非隐藏宝箱的漂移坐标。
- 扫码领取宝箱：扫码后校验会员、GPS 范围、可领取次数、是否已领取。
- 扫码页操作：加入背包、立即打开。
- 背包管理：查看图鉴、道具、宝箱、线索、空箱。
- 背包开箱：有效会员可打开未打开宝箱并获得奖励。
- 图鉴管理：已获得图鉴墙、未解锁剪影、图鉴详情、系列进度、筛选。
- 道具管理：查看数量、使用记录、寻宝罗盘解锁记录。
- 副本管理：副本列表、进行中/已完成状态、线索详情、答案提交、完成奖励。
- 消息通知：会员到期提醒、活动上新通知。

### 5.3 管理侧

- 用户管理：通过用户码或后台搜索定位用户。
- 会员开通/续期：选择套餐、记录实收金额、备注、发放罗盘。
- 套餐管理：配置标题、时长、赠送罗盘数量、后台可见标价、启停状态。
- 宝箱管理：创建宝箱、生成随机 token、配置位置、漂移坐标、范围、次数、隐藏状态、奖励池、状态、照片和备注。
- 奖励池管理：配置固定奖励、权重库存、类型权重、等级权重、库存、启停状态。
- 内容管理：百科、图鉴、图鉴系列、副本、线索。
- 道具管理：道具创建、启停、后台发放、库存和使用记录查询。
- 数据看板：用户、会员、营收、宝箱、奖励、副本、图鉴、罗盘等指标。
- 合规管理：隐私政策与用户协议版本管理。
- 操作日志：记录所有关键后台操作。

## 六、关键业务规则

### 6.1 会员开通

1. 用户在线下完成付款。
2. 用户在小程序展示用户码。
3. 超级管理员在后台扫码或输入用户码定位用户。
4. 超级管理员选择套餐，填写实收金额和备注。
5. 系统计算新到期时间并发放套餐赠送的寻宝罗盘。
6. 系统写入会员开通记录和操作日志。

用户端不展示价格，不提供线上支付路径。

### 6.2 实体宝箱领取

领取必须同时满足：

- 用户已登录。
- 用户为有效会员。
- 扫码 token 有效且不可被枚举。
- 宝箱状态为已投放。
- 用户在宝箱开启位置范围内；范围为 -1 时跳过位置校验。
- 用户未领取过该实体宝箱。
- 宝箱未达到可领取总次数；总次数为 -1 时不限制。

领取成功后：

- 系统原子增加已领取次数。
- 系统创建领取记录。
- 系统向用户背包写入一个未打开的实体宝箱物品。
- 用户可选择停留在背包，也可立即打开。

### 6.3 背包宝箱打开

打开必须满足：

- 用户已登录。
- 用户为有效会员。
- 背包宝箱属于当前用户。
- 背包宝箱状态为未打开。

打开成功后：

- 系统按奖励池规则生成奖励。
- 奖励分类入账。
- 宝箱状态变为已打开。
- 写入开箱结果和操作日志或用户行为日志。

### 6.4 背包与奖励入账

| 类型 | 入账方式 |
|---|---|
| 图鉴 | 写入背包，并在图鉴墙展示。 |
| 道具 | 写入背包，增加道具数量或生成道具实例。 |
| 实体宝箱扫码领取结果 | 写入背包，状态为未打开；这不是奖励池抽取结果。 |
| 虚拟宝箱 | 写入背包，状态为未打开。 |
| 线索 | 写入背包，同时写入用户线索解锁记录。 |
| 百科解锁 | 写入用户百科解锁记录。 |
| 空箱 | 写入背包，作为开箱历史和展示记录。 |

### 6.5 隐藏宝箱

- 隐藏宝箱不进入会员热力图。
- 隐藏宝箱仍可扫码领取，但必须满足会员、位置、次数等领取规则。
- 隐藏宝箱的发现路径来自副本线索、百科内容或线下物料。

### 6.6 热力图与坐标

- 实体宝箱投放时生成一次漂移坐标并持久保存。
- 热力图接口只返回漂移坐标，不返回真实 GPS。
- 使用寻宝罗盘解锁后，才可查看指定宝箱真实 GPS 和精确指引及投放照片。
- 非会员不可查看热力图。

### 6.7 寻宝罗盘

- 使用前校验用户为有效会员。
- 使用前校验用户拥有足够罗盘数量。
- 对同一用户和同一目标对象，解锁记录永久有效。
- 已解锁对象再次查看不消耗罗盘。
- 罗盘消耗和解锁记录必须在同一事务内完成。

### 6.8 副本与线索

- 副本可配置线性解锁或自由探索。
- 线性解锁时，未完成前置线索不得提交后续线索答案。
- 自由探索时，用户可查看已解锁的任意线索。
- 答案提交后按规范化规则匹配。
- 副本完成后按配置发放完成奖励。

## 七、业务流程

### 7.1 线下开通会员流程

线下收款 → 用户展示用户码 → 超级管理员后台识别用户 → 选择套餐并填写实收金额 → 系统续期会员 → 发放寻宝罗盘 → 写入开通记录。

### 7.2 扫码领取实体宝箱流程

用户扫码 → 服务端解析 token → 校验登录与会员 → 校验宝箱状态 → 校验位置范围 → 校验单用户领取限制 → 校验总次数 → 原子领取 → 写入背包 → 用户选择加入背包或立即打开。

### 7.3 背包开箱流程

用户打开背包宝箱 → 校验会员有效 → 校验物品归属与未打开状态 → 执行奖励池 → 奖励入账 → 宝箱标记已打开 → 展示开箱结果。

### 7.4 寻宝罗盘流程

用户选择目标宝箱或线索 → 校验会员有效 → 校验是否已解锁 → 校验罗盘数量 → 消耗 1 个罗盘 → 写入解锁记录 → 展示真实坐标或精确指引。

### 7.5 副本流程

用户进入副本列表 → 查看可进入副本 → 解锁或查看线索 → 查看免费提示或使用罗盘查看精确指引 → 提交答案 → 更新线索状态 → 全部必要线索完成后标记副本完成 → 发放完成奖励。

## 八、状态机

### 8.1 会员状态

| 状态 | 进入条件 | 权限 |
|---|---|---|
| 普通用户 | 注册后未开通过会员 | 浏览公开百科、收藏、分享、展示用户码。 |
| 有效会员 | 后台开通且未到期 | 普通用户权限 + 热力图、扫码领取、背包开箱、罗盘解锁、副本完整玩法。 |
| 过期会员 | 当前时间超过到期时间 | 保留历史数据；失去热力图、扫码领取、背包开箱、罗盘解锁等会员权限。 |

### 8.2 实体宝箱状态

| 状态 | 说明 |
|---|---|
| 待投放 | 后台已创建，尚未线下投放，不可领取。 |
| 已投放 | 可被有效会员扫码领取。 |
| 停用 | 后台临时关闭，不可领取。 |
| 已回收 | 运营人工标记已回收，不可领取。 |
| 异常 | 运营人工标记丢失、损坏或其他异常，不可领取。 |

「是否已领完」由可领取总次数和已领取次数计算，不作为必须手动维护的状态。

### 8.3 背包宝箱状态

| 状态 | 说明 |
|---|---|
| 未打开 | 已进入用户背包，可在会员有效时打开。 |
| 已打开 | 已发放奖励，不可重复打开。 |
| 已失效 | 后台处理异常时使用。 |

### 8.4 奖励池状态

| 状态 | 说明 |
|---|---|
| 草稿 | 可编辑，不可用于线上发放。 |
| 启用 | 可被宝箱或副本引用。 |
| 停用 | 不可新增引用，也不可执行发放；已关联对象需后台改绑启用中的奖励池或重新启用原奖励池。 |

### 8.5 副本进度状态

| 状态 | 说明 |
|---|---|
| 未解锁 | 用户尚未达到进入条件。 |
| 进行中 | 用户已有线索或答题记录。 |
| 已完成 | 用户完成必要线索并领取完成奖励。 |

## 十一、异常处理

| 场景 | 处理 |
|---|---|
| 非会员扫码实体宝箱 | 阻断并返回首页，不返回宝箱敏感信息。 |
| token 无效或不存在 | 提示宝箱不存在或已失效，不返回内部原因。 |
| 宝箱未投放、停用、已回收、异常 | 不允许领取，提示不可领取。 |
| 用户拒绝定位授权 | 提示需要授权定位后才能领取。 |
| 用户不在范围内 | 提示距离不满足要求，不领取、不消耗次数。 |
| 用户已领取过该实体宝箱 | 提示已领取，可引导查看背包。 |
| 宝箱领取次数已满 | 提示已被领取完。 |
| 并发领取冲突 | 以服务端事务结果为准，失败方提示已被领取完或稍后重试。 |
| 会员过期时打开宝箱 | 阻断打开，奖励不发放，宝箱保持未打开。 |
| 奖励库存不足 | 从本次候选集中移除后继续抽取；全部耗尽则发放空箱记录。 |
| 罗盘数量不足 | 提示罗盘不足，不创建解锁记录。 |
| 已解锁对象重复查看 | 直接展示已解锁内容，不重复消耗罗盘。 |
| 后台开通会员找不到用户 | 要求用户先登录小程序并展示用户码。 |
| 后台配置缺少必要字段 | 不允许启用对应宝箱、奖励池、副本或线索。 |

## 十二、防作弊与安全边界

v1 不做强防作弊，接受以下玩法性作弊风险：

- 用户修改 GPS。
- 用户把宝箱码分享给其他人。
- 用户一个设备登录多个账号。

必须防止的服务端问题：

- 可领取总次数为 1 的宝箱被领取多次。
- 同一用户重复领取同一实体宝箱。
- 奖励库存并发超发。
- 热力图或普通接口泄漏真实 GPS。
- 用户通过枚举 6 位编号或 token 获取宝箱信息。
- 非会员绕过客户端直接调用会员接口。
- 用户伪造背包物品、会员状态、罗盘数量或奖励结果。
- 后台操作缺少日志导致无法追责。

安全要求：

- 所有会员、位置、次数、库存、归属校验必须在服务端完成。
- 客户端传入的用户状态、坐标、背包物品、奖励结果不可直接信任。
- 扫码 token 使用不可枚举随机值；6 位编号不作为用户端查询凭证。
- 真实 GPS 只在罗盘解锁后按指定对象返回。
- 涉及领取、开箱、库存扣减、罗盘消耗的操作必须使用事务或等效原子机制。
- 对高频接口做服务保护限频，限频目的为保护系统稳定，不作为强反作弊承诺。

建议关键唯一约束：

- 用户 openid 唯一。

- 同一用户 + 同一罗盘解锁对象唯一。

## 十五、验收场景

- 非会员扫码实体宝箱：阻断并返回首页，不领取、不消耗次数、不返回真实 GPS。
- 有效会员在范围内扫码并选择加入背包：领取成功，实体宝箱已领取次数增加 1，背包出现未打开宝箱。
- 有效会员扫码并选择立即打开：先领取，再开箱，奖励按规则入账。
- 用户领取实体宝箱后离开现场：可在任意位置打开，但必须仍是有效会员。
- 会员过期后打开背包宝箱：打开失败，宝箱保持未打开，奖励不发放。
- 同一用户重复扫码同一实体宝箱：不能再次领取。
- 可领取总次数为 1 的实体宝箱被多人并发扫码：只有一个用户领取成功。
- 热力图接口：只返回非隐藏宝箱的漂移坐标，不返回真实 GPS。
- 使用寻宝罗盘解锁宝箱：消耗 1 个罗盘，永久显示该宝箱真实 GPS 和精确指引。
- 重复查看已解锁宝箱位置：不再消耗罗盘。
- 奖励池库存部分耗尽：系统移除耗尽候选并继续抽取。
- 奖励池全部候选耗尽：开箱结果为空箱，并写入背包记录。
- 线索答案提交：大小写、首尾空格、全半角差异不影响正确答案匹配。
- 后台开通会员：通过用户码定位用户，记录套餐、实收金额、备注，正确顺延到期时间并发放罗盘。
- 用户端全流程：不展示价格，不提供线上支付入口。
