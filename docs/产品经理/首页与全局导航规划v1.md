## 1. 结论摘要

用户端 v1 以**首页地图**为唯一主内容面，**取消探索 Tab 与探索综合信息流**。

底部导航按账号态切换：

| 账号态 | 底部 Tab（从左到右） |
|---|---|
| 未登录、普通登录用户、待激活会员 | 首页 \| 我的 |
| 有效会员、过期会员 | 首页 \| 副本 \| 图鉴 \| 背包 \| 我的 |

- 打开小程序**一律默认「首页」**。
- 首页地图叠加**公开百科点**与（仅有效会员）**宝箱点**。
- 扫码为**有效会员**专属入口：**首页悬浮按钮** + **「我的」普通按钮**（非悬浮）；细则见 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md)。
- 用户端**不展示价格**，**不提供线上支付 / 开通购买入口**。

## 2. 底部导航细则

### 2.1 Tab 结构

- **2 Tab（未登录 / 普通 / 待激活）**：`首页` | `我的`
- **5 Tab（有效会员 / 过期会员）**：`首页` | `副本` | `图鉴` | `背包` | `我的`

### 2.2 状态切换规则

- **待激活**：与普通用户相同，仅 2 Tab；激活成功（欢迎页点击「开始探索」）后切换为 5 Tab；激活前**不预览** 5 Tab。
- **过期会员**：保留 5 Tab，能力**只读**（见第 6 节）。
- **欢迎码 / 会员激活 / 解密地图开始**等全屏流程独立于底部导航，不占用 Tab。

### 2.3 v1 明确不做

- 探索 Tab
- 探索综合信息流（百科 + 副本 + 公告混排页）
- 用户端价格展示与线上支付

## 3. 首页地图

首页是用户端主内容面，**地图模式**，无探索信息流。

### 3.1 顶栏

- 固定控件：`城市切换` + `搜索框`
- 不展示价格，不提供线上开通 / 续费购买文案
- 搜索范围：**仅公开百科**（标题 / 标签）；不搜索宝箱编号

### 3.2 图层与可见性

- 百科点 + 宝箱点使用**不同 icon 叠加**，**不提供图层开关**
- **百科点**：未登录、普通、待激活、有效会员、过期会员凡可浏览公开内容的用户均可见
- **宝箱点（漂移坐标）**：仅**有效会员**可见；展示已投放且非隐藏宝箱
- **未登录 / 普通 / 待激活 / 过期**：均**不加载宝箱层**
- **隐藏宝箱**：不进入首页地图（发现路径仍为线索 / 百科 / 线下物料）
- **本人已领取**实体宝箱：有效会员地图仍显示，样式或角标标识「已领」

### 3.3 城市与定位

> 系统定位能力细则（申请时机、失败分级、上报、拒权恢复）以 [小程序客户端系统权限规划v1.md](./小程序客户端系统权限规划v1.md) 为准；运营城市主数据（含 adcode）以 [后台管理-运营城市v1.md](./后台管理-运营城市v1.md) 为准；本节约定首页地图侧表现。

- 支持手动切换后台**已开通城市**（状态 = 启用）；地图数据与点位跟随所选城市
- **进入首页即申请定位**（含未登录访客）；形态为 `<map>` + `show-location` 蓝点 + `wx.getLocation`（建议 gcj02）
- **GPS 匹配运营城**：`getLocation` → 逆地理得到行政区 adcode → **市级归一**后与启用运营城的 **adcode** 精确匹配；**不以城市名称**做主匹配
- **默认城市**：优先 GPS 匹配已开通城市；定位失败、拒绝授权或当前城市未开通（adcode 无对应启用城）→ 后台配置的**默认运营城市**并提示
- **首页定位强度**：单次取点 +「定位到我」；**不**前台持续跟随轨迹
- **切城 / 点位刷新后的地图视野**（客户端 map 负责，后台**不**配置城心）
  - 该城有业务点位（百科 / 宝箱等）：用 map `include-points` / `MapContext.includePoints`，将视野适配为**该城当前可见点位集合**的包围区域（可加 padding）；**不是**飞到单一城心坐标
  - 该城无点位（空城）：不依赖城心；地图可滑动 + 中央轻提示（见下）
- **「定位到我」**
  - 当前使用系统定位（未手动切城）时：飞到用户**真实 GPS**
  - 用户已**手动切换到其他城市**后：控件语义为**适应该城点位可见区域**（`includePoints` 该城 markers）；有点位则包住全部点；空城则保持筹备中空态、不强制跳转虚假城心
- **失败分级**：拒权 / 未授权 → 说明 + 去设置；系统失败 / 超时 → 可重试（与扫码业务「距离不足」区分）
- **拒权恢复**：去设置后返回当前页，自动重试 `getLocation` / 刷新蓝点与城市匹配
- **首页位置上报**：**仅已登录**可上报落库；**每会话最多 1 条**（可覆盖更新）；**访客仅本机**用于城市/蓝点、**不落库**；**不建**连续轨迹画像（详见权限规划）
- **空城**：地图仍可滑动，中央轻提示「这座城市的探秘内容筹备中」

### 3.4 百科点交互

- 点击标记：轻量卡片展示**图片 + 标题 + 标签**
- 可从卡片进入百科详情
- 收藏需登录；未登录触发收藏时要求登录

### 3.5 宝箱点交互（仅有效会员）

点击宝箱标记 → 轻量卡片（宝箱图片 + 主操作区）：

| 状态 | 卡片表现 |
|---|---|
| 未使用寻宝罗盘 | 主按钮「使用罗盘」；库存 > 0 消耗 1 个并**永久**解锁真实坐标 / 距离 / 指南针 / 指引图文案；库存 = 0 提示不足；**使用前不展示精确距离、不暴露真实 GPS** |
| 已使用寻宝罗盘 | 展示指南针方位 + 到真实坐标的距离 + 指引图 / 文案；再次查看不消耗 |
| 已领取 | 卡片区分「已领取」；罗盘展示规则不变 |

- 点位卡片**不完成扫码领取**；领取实体宝箱仍走扫码链路（完整规则见 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md)）
- 罗盘解锁规则与总 PRD 一致：同一用户 + 同一宝箱永久有效，事务内扣减库存并写解锁记录

### 3.6 扫码入口

> 完整状态机、结果页、定位、开箱弹窗等以 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md) 为准；本节只约定导航可见入口。

- **仅有效会员**展示扫码入口：
  - **首页**：悬浮扫码按钮
  - **「我的」**：普通按钮「扫一扫」（非悬浮；入口位置见 [我的页面规划v1.md](./我的页面规划v1.md)）
- 未登录、普通、待激活、过期：**不展示**上述扫码入口
- 扫码界面只开相机、识别小程序码；码多参数原样上报，**服务端按码型分流**，界面本身不展示业务结果
- 非有效会员（含未登录）扫实体宝箱码：**静默回首页**——不进入扫码结果页、不作任何提示、不展示宝箱详情、不领取、**不强制登录**
- 内扫与微信「扫一扫」外扫：**同一套业务规则**

## 4. 公开百科完整路径

在取消探索 Tab 后，百科闭环为：

1. **首页地图**：卡片预览 → 详情  
2. **首页搜索**：百科搜索结果 → 详情  
3. **我的 → 百科收藏**：收藏列表管理（**无**「我的 → 百科浏览」；详见 [我的页面规划v1.md](./我的页面规划v1.md)）

v1 不提供独立探索列表信息流。

## 5. 「我的」页

> 完整分态清单、身份区、设置/关于分工、登录退出落点以 [我的页面规划v1.md](./我的页面规划v1.md) 为准；本节只保留导航层摘要。

无论 2 Tab 或 5 Tab，「我的」**始终**为底部最右 Tab。导航层要点：

- 身份：头像、昵称、用户码、会员状态 / 到期（未登录为登录 CTA；细则见「我的」规划）
- **百科收藏**入口（**无**百科浏览入口）
- 图鉴、背包、副本入口（与一级 Tab **可重复**；未登录 / 普通 / 待激活不显示）
- **无**消息通知 / 站内消息中心入口（v1；到期靠微信订阅 + 首页过期软提示）
- 有效会员：**扫一扫**普通按钮（非悬浮；行为见扫码规划）
- 设置 / 关于 / 到期提醒开关位置：见「我的」规划与权限规划（到期提醒在身份区旁，**不在**设置；协议在关于）

约束：不展示价格，不提供线上支付入口。

## 6. 过期会员只读边界

### 6.1 可

- 使用 5 Tab 导航
- 浏览公开百科、首页百科点与详情、百科收藏
- 查看图鉴墙与图鉴详情、背包列表与历史记录、副本进度与已解锁线索文案

### 6.2 不可

- 查看首页宝箱层
- 使用扫码领取实体宝箱（无扫码入口；误扫静默回首页）
- 打开背包中未打开宝箱
- 消耗寻宝罗盘或新解锁坐标
- 提交副本答案、领取副本完成奖励

### 6.3 首页提示

过期会员进入首页时，顶部软提示：

> 会员已到期，图鉴 / 背包可继续查看

- 不展示价格  
- 不展示续费购买按钮  
- 续费在小程序外完成

## 7. 未登录与普通用户

## 11. User Journey / Flow / Use Case

> 本节从用户视角归纳本专题路径；业务规则与边界以本文其它章节为准。  
> 角色（Actor）账号态：未登录 / 普通 / 待激活 / 有效会员 / 过期会员（与总 PRD 一致）。  
> 可对照 §12 验收场景执行；扫码领取细则见 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md)，系统定位见 [小程序客户端系统权限规划v1.md](./小程序客户端系统权限规划v1.md)。

### 11.1 User Journey / 用户旅程

| ID | 名称 | 角色 | 目标 | 旅程步骤 | 成功结果 | 关联 |
|---|---|---|---|---|---|---|
| NAV-J-01 | 访客逛城与百科 | 未登录 | 打开小程序浏览公开内容 | 打开小程序 → 默认首页（2 Tab）→ 定位/默认运营城 → 地图百科点或搜索 → 详情；收藏时触发登录 | 完成百科浏览；未登录无宝箱层/扫码 | §3、§4、§7；登录见权限 /「我的」 |
| NAV-J-02 | 会员日常探索壳层 | 有效会员 | 在导航壳层内发现宝箱并准备领取 | 打开小程序 → 5 Tab 首页 → 百科+宝箱双层 → 点宝箱用罗盘解锁真实距离/指引 → 可见首页悬浮扫码与「我的」扫一扫 | 地图探索就绪；领取走 SCAN 链路 | §2、§3.5–3.6；领取见 SCAN |
| NAV-J-03 | 过期只读回访 | 过期会员 | 查看历史资产，无会员写操作 | 打开小程序 → 5 Tab → 首页无宝箱层/扫码 + 顶部到期软提示（无价格）→ 图鉴/背包/副本只读 | 历史可看；领箱/开箱/罗盘/交答案不可用 | §6 |

### 11.2 User Flow / 用户流程

**F-导航分态**

1. 冷启动 / 切回前台 → 读取账号态  
2. 未登录 / 普通 / 待激活 → **2 Tab**（首页 \| 我的）  
3. 有效会员 / 过期会员 → **5 Tab**（首页 \| 副本 \| 图鉴 \| 背包 \| 我的）  
4. 默认落点：**首页**（非「我的」）  
5. 待激活在欢迎页点「开始探索」成功 → 切换为 5 Tab（激活前不预览 5 Tab）  
6. 欢迎码 / 激活 / 解密地图等全屏流程独立于底部导航

**F-首页地图**

1. 进入首页 → 申请定位（含访客；细则 PERM）  
2. GPS → 逆地理 adcode（市级归一）匹配已开通城市；失败/拒权/未开通 → 默认运营城市并提示  
3. 加载公开百科点；**仅有效会员**加载宝箱层（漂移坐标，非隐藏）  
4. 点百科标记 → 轻量卡片 → 可进详情  
5. 点宝箱标记（仅有效）→ 卡片；未用罗盘不露精确距离；用罗盘后永久显示真实距离/指南针/指引；**卡片不完成领取**  
6. 「定位到我」：未手动切城 → 真 GPS；已手动切他城 → 该城点位可见区域（includePoints）；空城不飞虚假城心  

**F-百科路径**

1. 首页地图卡片 → 详情  
2. 首页搜索（仅百科标题/标签，不搜宝箱编号）→ 结果 → 详情  
3. 「我的 → 百科收藏」→ 列表（见 ME；**无**「我的 → 百科浏览」）  
4. v1 **无**探索 Tab / 探索信息流  

**F-切城与空城**

1. 用户手动切换已开通城市 → 地图点位随城刷新 → 有点则 includePoints 适配可见区域  
2. 空城 → 可滑动 + 中央「筹备中」轻提示  
3. 定位失败路径与默认城见 §3.3 / PERM  

**F-扫码入口（导航层）**

1. 仅有效会员展示：首页悬浮扫码 + 「我的」扫一扫（普通按钮）  
2. 非有效会员无入口；误扫实体箱 → **静默回首页**（业务见 SCAN-UC-06）

## 12. 验收场景

- 未登录打开小程序：默认首页，底部仅「首页 | 我的」，地图仅百科点，无扫码入口；收藏触发登录
- 普通登录用户：2 Tab；首页无宝箱层、无扫码入口；可首页百科浏览 / 收藏；「我的」含用户码与百科收藏、无扫一扫与玩法入口；扫实体宝箱码静默回首页（不强制登录）
- 待激活会员：2 Tab；无宝箱层、无扫码入口；欢迎码激活成功后切换 5 Tab，出现宝箱层、首页悬浮扫码与「我的」扫一扫按钮
- 有效会员：底部为「首页 | 副本 | 图鉴 | 背包 | 我的」；首页百科 + 宝箱双 icon；首页悬浮扫码 + 「我的」扫一扫；点宝箱未用罗盘不显示精确距离；使用罗盘后永久显示真实距离 / 指南针 / 指引；已领宝箱有已领样式；点位卡片不完成领取；搜索只出百科；扫码领取/开箱见宝箱扫码规划；「我的」分态见「我的」规划
- 过期会员：5 Tab；首页无宝箱层、无扫码入口；顶部到期软提示无价格；可查看图鉴 / 背包历史；开箱、罗盘、领箱不可用；误扫实体宝箱码静默回首页
- 手动切城：点位随城市变化；有点则 includePoints 适配可见区域；空城显示筹备中提示；切到他城后「定位到我」适配该城点位区域（非城心）；定位失败落默认运营城市
- 访客 / 任意账号进首页：申请定位；拒权落默认城并提示；同意后蓝点与城市匹配；**访客坐标不落库**；**已登录**首页位置可按每会话 ≤1 上报（详见权限规划）
- 「我的」：未登录极简（登录 + 百科收藏 + 关于）；到期提醒开关仅有效会员身份区旁；设置无协议（协议在关于）；详见「我的」规划
- **无**消息通知入口（导航与「我的」）；**无**「我的 → 百科浏览」
- 首页宝箱层接口：仅有效会员可获得非隐藏宝箱的漂移坐标，不返回真实 GPS；非有效会员不返回宝箱点位
- 用户端全导航与首页路径：不展示价格，不提供线上支付入口
