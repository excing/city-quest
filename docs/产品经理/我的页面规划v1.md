## 1. 结论摘要

「我的」是底部导航**最右 Tab**，在 2 Tab / 5 Tab 下**始终存在**。职责是**身份与账号**、**收藏入口**、**会员态下的玩法快捷入口与扫码**、**设置与关于**——不是第二首页，也不承载价格/支付/消息中心。

| 维度 | v1 决策 |
|---|---|
| 未登录 | **极简**：去登录 + 百科收藏 + 关于 |
| 已登录身份 | 头像、昵称、**用户码**（文本 + 复制 + 二维码 icon→全屏码）；会员态文案按账号态 |
| 百科 | **仅百科收藏**入口；**无**「百科浏览」（公开百科 = 首页地图 + 首页搜索） |
| 玩法入口 | 图鉴 / 背包 / 副本：仅**有效·过期**；`switchTab` 到对应 Tab 根页 |
| 扫码 | 仅**有效会员**；普通按钮「**扫一扫**」（非悬浮）；链路同首页悬浮 |
| 到期提醒 | 仅**有效会员**，在**身份区旁**开关（订阅补授权）；**不在**设置内 |
| 设置 | **仅已登录**独立子页：定位/相机「去开启」+ 退出登录 |
| 关于 | 全账号态可有：用户协议、隐私政策、（可选）版本号；**协议只在关于** |
| 改资料 | 点头像/昵称调微信能力；无自定义资料页 |
| 退出 | 二次确认 → 清登录态 → **强制回首页 Tab** |
| 明确不做 | 资产摘要、消息中心、价格/支付、帮助/客服、开通历史、账号注销 |

## 2. 页面职责与边界

### 2.1 做什么

- 展示与管理登录身份（访客引导登录 / 已登录资料与用户码）
- 展示会员状态与到期信息（按账号态，无价格/续费）
- 提供百科收藏入口（未登录点击触发登录）
- 有效会员：扫一扫入口；有效·过期：图鉴/背包/副本快捷入口
- 有效会员：到期提醒订阅补授权开关
- 已登录：设置（系统权限引导 + 退出）
- 全账号态：关于（协议 / 隐私 / 可选版本号）

### 2.2 不做什么

- 不作为打开小程序的默认落点（默认首页，见导航规划）
- 不展示价格、线上支付、开通/续费购买入口
- 不提供消息通知 / 站内消息中心
- 不在本页展示资产摘要（罗盘数、图鉴数、未开箱数等）
- 不提供帮助 / 客服 / FAQ / 开通历史 / 订单
- 不提供账号注销（v1 延后）
- 不做像素级 UI 规格（原型另案）

## 3. 信息架构与分态内容

### 3.1 主页面产品区块顺序

自上而下（产品层，非视觉稿）：

1. **身份区**（未登录为登录 CTA 占位）
2. **扫一扫**（仅有效会员）
3. **功能入口**：百科收藏；有效·过期另含图鉴 / 背包 / 副本
4. **设置**（仅已登录）
5. **关于**

2 Tab 态无扫一扫、无玩法三入口时，身份后直接接百科收藏与底栏入口。

### 3.2 分态主页面清单

| 账号态 | 主页面包含 |
|---|---|
| 未登录 | 登录 CTA \| 百科收藏 \| 关于 |
| 普通登录 | 身份（头像、昵称、用户码；**不写**「普通用户」）\| 百科收藏 \| 设置 \| 关于 |
| 待激活会员 | 身份（待激活 + 弱入口 + 用户码）\| 百科收藏 \| 设置 \| 关于 |
| 有效会员 | 身份（会员有效 + 到期日 + **到期提醒开关** + 用户码）\| **扫一扫** \| 百科收藏 \| 玩法三入口 \| 设置 \| 关于 |
| 过期会员 | 身份（会员已到期 + 原到期日 + 用户码；**无**到期开关、**无**扫一扫）\| 百科收藏 \| 玩法三入口 \| 设置 \| 关于 |

- 未登录：**不展示**设置、扫一扫、玩法入口、用户码、会员态。
- 过期保留 5 Tab 只读能力边界见导航规划；本页不重复首页到期软提示条。

## 4. 身份区

### 4.1 未登录

- 默认头像占位 + 「未登录」类文案 + 主 CTA **「去登录」**
- 无用户码、无会员状态 / 到期
- 点击身份区或「去登录」→ 拉起登录流（见权限规划：协议勾选 + 头像昵称强制）

### 4.2 已登录通用

- 展示：头像、昵称、**用户码**
- **改资料**：点击头像或昵称 → 调微信 `chooseAvatar` + 昵称输入能力更新资料；**不**做独立自定义资料页，**不**支持非微信体系的自定义头像/昵称方案
- 用户码旁：
  - 可读文本码 + **复制**
  - **二维码 icon** → 点击展示**全屏二维码**（供线下运营扫码识别）
- 所有已登录态均有用户码：普通 / 待激活 / 有效 / 过期

### 4.3 会员状态文案

| 账号态 | 展示 |
|---|---|
| 普通 | 不展示「普通用户」等会员标签；仅昵称区 + 用户码 |
| 待激活 | 「待激活」+ 副文案弱引导（见 §4.4） |
| 有效 | 「会员有效」+ 到期日 **YYYY-MM-DD**（如：会员有效 · 至 2026-12-31） |
| 过期 | 「会员已到期」+ 原到期日 **YYYY-MM-DD** |

- 全态：**不**展示价格、**不**展示续费/购买按钮。
- 过期：**仅**身份区状态表达；**不**再出首页式全宽软提示条（首页软提示仍按导航规划）。

### 4.4 待激活弱入口

- 身份区副文案示例语义：「扫欢迎码完成激活」（具体文案可原型定）
- **弱入口**（非强 banner）：
  - 若存在本人**未完成**欢迎/激活流程记录 → 进入**本人会员欢迎页**
  - 若无记录 → toast 说明需使用购买时获得的欢迎码扫码激活
- 激活前**不预览** 5 Tab（与导航一致）

### 4.5 到期提醒开关（仅有效会员）

- 位置：**身份区旁**（产品归属在身份区，不在设置）
- 可见性：**仅当前有效会员**；过期 / 待激活 / 普通 / 未登录 **不展示**
- 行为：订阅消息**补授权**（`requestSubscribeMessage` 等细则与触点策略见权限规划）；本文件只锁定**入口位置与可见性**
- 发送：仅有效会员可发；过期不发（权限规划）

## 5. 扫一扫入口

| 项 | 规则 |
|---|---|
| 可见 | **仅有效会员** |
| 形态 | 普通按钮（**非悬浮**）；产品文案 **「扫一扫」** |
| 位置 | 身份区之后、功能入口之前（产品顺序） |
| 行为 | 与首页悬浮扫码**同一套**内扫链路；结果页 / 入包 / 失败以扫码规划为准 |
| 其它账号态 | 不展示该入口；外扫实体箱仍按扫码规划静默回首页 |

## 6. 百科收藏入口

| 项 | 规则 |
|---|---|
| 入口 | 「我的」提供 **百科收藏** 一行/入口；**不提供「百科浏览」** |
| 公开百科路径 | **首页地图**、**首页搜索**（及详情）；不再经「我的 → 浏览」 |
| 未登录 | 入口**始终展示**；点击 → **拉登录** → 成功后进入收藏列表 |
| 已登录 | 进入本人百科收藏列表 |
| 列表 IA | 本专题**只定入口与登录门闸**；列表空态、取消收藏、筛选等**另案** |

---

## 7. 玩法入口（图鉴 / 背包 / 副本）

| 项 | 规则 |
|---|---|
| 可见 | **仅有效会员、过期会员** |
| 不可见 | 未登录、普通、待激活（与无对应一级 Tab 一致） |
| 跳转 | `switchTab` 到对应一级 Tab **根页**；**不**深链子页 |
| 与 Tab 关系 | 与一级 Tab **可重复**，作快捷入口；过期只读边界仍由各 Tab / 导航规划约束 |

## 8. 设置子页

| 项 | 规则 |
|---|---|
| 形态 | **独立子页**；主页面入口文案「设置」 |
| 谁可见 | **仅已登录**；未登录主页面**无**设置入口 |
| 内容 | 定位「去开启」、相机「去开启」（跳转微信设置）；**退出登录** |
| 不含 | 用户协议 / 隐私政策（改在**关于**）；到期提醒开关（在身份区）；消息通知；完整系统权限总开关列表 |

### 8.1 退出登录

1. 用户点「退出登录」  
2. **二次确认**（如「确定退出登录？」）  
3. 确认后：清理本地登录态；服务端会话失效  
4. **强制 `switchTab` 回首页 Tab**  
5. 底部导航按**未登录**变为 2 Tab（首页 \| 我的）

## 9. 关于子页

| 项 | 规则 |
|---|---|
| 谁可见 | **未登录与已登录**均可有「关于」入口 |
| 内容 | 《用户协议》《隐私政策》入口；**可选**小程序版本号 |
| 协议勾选 | 仍**仅登录页**强制勾选（权限规划）；关于仅供再次查看 |
| 与设置分工 | **协议只在关于**；设置只管权限引导 + 退出 |

## 10. 登录 / 退出路径与落点

### 10.1 从「我的」发起登录

| 触发 | 登录成功后落点 |
|---|---|

见 §8.1：二次确认 → 清会话 → **回首页 Tab**。

## 12. User Journey / Flow / Use Case

> 本节从用户视角归纳本专题路径；业务规则与边界以本文其它章节为准。  
> 角色（Actor）账号态：未登录 / 普通 / 待激活 / 有效会员 / 过期会员（与总 PRD 一致）。  
> 可对照 §13 验收场景执行；登录协议与订阅见 [小程序客户端系统权限规划v1.md](./小程序客户端系统权限规划v1.md)；扫码链路见 [宝箱扫码规划v1.md](./宝箱扫码规划v1.md)。

### 12.1 User Journey / 用户旅程

| ID | 名称 | 角色 | 目标 | 旅程步骤 | 成功结果 | 关联 |
|---|---|---|---|---|---|---|
| ME-J-01 | 访客登录与身份建立 | 未登录→已登录 | 建立账号身份 | 进「我的」极简页 → 点去登录 → 协议勾选+头像昵称（PERM）→ 成功后仍在「我的」已登录分态 | 可见用户码、设置等 | §3、§4、§10；PERM |
| ME-J-02 | 有效会员账号中枢 | 有效会员 | 管理身份并快捷进入玩法 | 身份区（有效+到期日+提醒开关+用户码）→ 扫一扫 / 百科收藏 / 图鉴·背包·副本 switchTab → 设置或关于 | 完成身份查看与入口跳转 | §3–§9 |
| ME-J-03 | 退出回到访客导航 | 已登录 | 安全退出 | 设置 → 退出二次确认 → 清登录态 → 强制回首页 Tab（2 Tab） | 会话清除；导航回访客壳 | §8.1、§10.2 |

### 12.2 User Flow / 用户流程

**F-分态渲染**

1. 进入「我的」Tab  
2. 按账号态渲染 §3.2 清单（未登录极简 … 有效含扫一扫与玩法 … 过期无扫一扫/无到期开关）  
3. 区块顺序：身份 →（扫一扫）→ 功能入口 → 设置 → 关于  

**F-登录落点**

1. 点「去登录」/ 身份区 → 登录成功 → **留在「我的」**主页面，刷新已登录分态  
2. 未登录点「百科收藏」→ 登录成功 → 进入**百科收藏列表**  
3. 登录交互（协议默认不勾选、强制头像昵称）→ PERM  

**F-退出**

1. 已登录 → 设置 → 退出登录  
2. 二次确认 → 清会话 → **回首页 Tab**  

**F-扫一扫入口（本页）**

1. 仅有效会员展示普通按钮「扫一扫」（非悬浮）  
2. 点击 → 与首页悬浮一致的内扫链路（SCAN / PERM）  
3. 过期 / 非有效：**不展示**  

**F-玩法入口**

1. 仅有效·过期展示图鉴 / 背包 / 副本  
2. 点击 → `switchTab` 到对应 Tab 根页  
3. 只读边界（过期）以导航规划为准

## 13. 验收场景

- 未登录进「我的」：仅登录 CTA + 百科收藏 + 关于；无用户码、无设置、无扫一扫、无玩法入口  
- 未登录点百科收藏 → 登录 → 进入收藏列表  
- 未登录点去登录 → 登录成功 → 仍在「我的」且为普通（或对应）已登录态  
- 普通登录：有用户码（可复制 + 二维码全屏）；无会员标签；有设置与关于；无扫一扫、无玩法三入口  
- 待激活：身份「待激活」+ 弱入口；有记录进欢迎页，无记录 toast；2 Tab；无扫一扫/玩法  
- 有效会员：身份有效 + YYYY-MM-DD 到期 + 到期提醒开关；扫一扫；百科收藏；图鉴/背包/副本可 switchTab；设置无协议无到期开关  
- 点扫一扫：进入与首页一致的内扫链路（详见扫码规划）  
- 过期：身份已到期 + 原到期日；无到期开关、无扫一扫；有玩法入口只读；无首页式软提示条在「我的」  
- 点头像/昵称：可更新微信头像昵称；无独立资料页  
- 设置：定位/相机去开启；退出需确认 → 回首页且 2 Tab  
- 关于：可打开用户协议与隐私政策  
- 全「我的」路径：无价格、无支付、无消息中心、无资产摘要、无客服/开通历史
