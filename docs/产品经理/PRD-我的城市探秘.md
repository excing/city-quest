## 1. Summary

「我的城市探秘」是一款个人主体微信小程序。v1 只覆盖云南大理。用户在首页地图上查看运营录入的本地「百科」（美食、景点、其他等地点），可打开详情、记录浏览；登录后可收藏。用户不能发布内容。运营通过简易 Web 后台维护百科并控制上下架。

## 3. Background

### 3.1 这是什么

到一座陌生或半熟城市时，人们常需要「地图上看看有什么值得去的点」，而不是刷一长串广告化信息流。本产品把运营筛选过的本地地点做成地图上的百科点位，支持浏览与登录收藏。

### 3.2 为什么现在做

- 原始需求极简：地图浏览 + 收藏 + 我的记录，明确不做 UGC。
- 个人主体微信小程序适合轻量、可上线验证的 MVP。
- 先单城（大理）把路径跑通，再考虑扩城与能力扩展。

### 3.3 约束

- 平台：微信小程序（个人主体）。
- 内容不能依赖用户生产，必须运营录入。
- 技术选型不在本 PRD 锁定，另见技术方案（若有）。

## 4. Objective

### 4.1 目标

让用户在大理用一张地图快速发现运营推荐的本地地点，需要时登录收藏，并在「我的」中回看浏览与收藏；同时让运营不改代码也能维护内容。

### 4.2 对谁有价值

- **用户**：少踩坑、少刷广告，用地图找点、用收藏稍后安排行程。
- **项目本身**：验证「策展型本地百科 + 地图」是否值得继续投入。

### 4.3 关键结果（KR）

| KR | 说明 | 如何判定达成 |
|----|------|----------------|
| KR1 | 真实用户可使用 | 小程序通过发布流程，外网可打开核心页面 |
| KR2 | 核心路径顺畅 | 自己/朋友可完成：打开地图 → 预览 → 详情 →（登录）收藏 → 我的中查看浏览/收藏，过程无阻断性缺陷 |
| KR3 | 运营可独立维护 | 后台可完成百科新增/编辑/上下架与图片维护，无需改客户端代码即可让上架内容出现在地图上 |
| KR4 | 首发有可读内容 | 上线时大理至少有一批已上架百科（建议 ≥ 20 条，可按实际准备调整） |

不设 v1 的 DAU/商业化硬指标；先保证能用、好逛、能运营。

## 5. Market Segment(s)

本产品按「要完成的任务」划分用户，而不是按年龄性别。

### 5.1 主场景人群

| 人群 | 要完成的工作（Jobs） | 约束与情境 |
|------|----------------------|------------|
| 到大理游玩的人 | 快速知道附近/城里有哪些值得去的吃、景、物；记下想去的点 | 时间有限；可能不熟悉路；希望少广告 |
| 在大理短住/生活的人 | 收藏常去或想去的地点，偶尔回看 | 需要可回访的收藏列表 |
| 运营者（本人） | 稳定录入与调整推荐点，控制是否对外展示 | 个人时间有限，需要简单后台 |

### 5.2 产品侧约束（影响方案）

- 仅大理，不做多城市切换。
- 内容量 v1 按「可全量打在一张地图上」设计。
- 游客可逛、可记浏览；收藏与收藏入口仅登录用户。

## 6. Value Proposition(s)

### 6.1 用户获得什么

- **一眼看全**：打开即地图，自动框住全部已上架百科点位。
- **策展感**：内容是运营录入的地点百科，不是用户随便发的帖。
- **轻收藏**：登录后收藏存在服务端，换设备仍可看（在同一微信账号体系下）。
- **可回看**：浏览记录在本地保留最近轨迹，方便「刚才看过哪个」。

### 6.2 避开什么痛

- 信息流里难辨真假、广告多。
- 收藏散落在聊天记录/截图里找不到。
- 大而全 App 太重，只想快速看地图点位。

### 6.3 相对常见替代方案

| 替代 | 用户常怎么做 | 我们更轻/更聚焦的点 |
|------|----------------|---------------------|
| 大众点评/高德等 | 搜店、看评价 | 更少噪音，强调本地策展清单式点位 |
| 攻略笔记/聊天记录 | 收藏文章、截图 | 地图空间关系清晰，收藏结构统一 |
| 朋友口头推荐 | 问人 | 可自助、可回看、可分享详情卡片 |

v1 不追求功能全面，只把「地图发现 + 详情 + 登录收藏 + 运营维护」做稳。

## 7. Solution

### 7.1 UX / 用户流程

#### 7.1.1 信息架构

```
底部 Tab
├── 首页（地图）
└── 我的
    ├── 用户信息（未登录：登录入口 / 已登录：头像昵称、退出）
    ├── 浏览记录（游客与登录均有）
    ├── 我的收藏（仅登录可见）
    └── 设置/关于/协议

二级页（示意）
├── 百科详情
├── 登录页
├── 浏览记录列表
├── 收藏列表
└── 协议/关于等静态页
```

#### 7.1.2 主路径：逛地图

1. 用户打开小程序，进入首页地图。
2. 客户端拉取**全部已上架**百科。
3. 地图根据全部点位自动缩放，使所有点位落在可见范围内。
4. 点位颜色来自该百科所属类型的配置色 `color`。
5. 用户点击某点位 → 出现预览卡（**名称 + 介绍**）。
6. 用户再点预览卡/「查看详情」→ 进入百科详情。
7. 进入详情即写入**本地浏览记录**（规则见 7.2.5）。

#### 7.1.3 收藏路径（含未登录）

1. 用户在详情点击收藏。
2. **已登录**：请求服务端收藏成功，按钮变为已收藏态。
3. **未登录**：
   - 弹出确认（说明需要登录才能收藏）。
   - 用户确认后进入登录页。
   - 登录成功后回到原页，**不自动完成收藏**；用户需再次点击收藏。
4. 该「需登录能力 → 弹窗确认 → 登录 → 回原页不自动续办」为**通用逻辑**，后续其他需登录能力同样适用。
5. 「我的」页也提供主动登录入口。

#### 7.1.4 我的：浏览与收藏

1. 浏览记录：所有人可见入口；数据仅本地。
2. 我的收藏：仅登录后显示入口；数据仅服务端。
3. 列表点击进入详情（下架表现见 7.2.7）。

#### 7.1.5 运营路径

1. 管理员用可配置的账号密码登录 Web 后台。
2. 创建/编辑百科，上传图片，设置类型与标签，填写字段。
3. 上架后，小程序地图与详情可见；下架后对普通浏览不可见（收藏侧特殊规则见 7.2.7）。

#### 7.1.6 分享

- 百科详情支持分享给朋友/朋友圈能力（按微信小程序规范）。
- 分享卡片带**标题与封面**（标题默认百科名称，封面默认百科封面图）。
- 从分享进入应能打开对应百科详情（若已下架，则按失效提示处理）。

### 7.2 Key Features

#### 7.2.1 百科（核心内容模型）

- **定义**：百科 = 绑定真实经纬度的 **POI 地点卡片**。
- **命名**：产品内统一称「百科」，不再把「美食景点其他」当成三套产品，而是类型配置的初始示例。
- **类型**
  - 来源：后台**静态文件配置**。
  - 配置字段：`key`、`name`、`color`。
  - 每个百科只能选 **一个类型**。
- **标签**
  - 与类型独立。
  - 运营在后台**自由输入**多个标签。
  - 小程序详情展示；v1 **不按标签筛选**。
- **上下架**
  - 支持上架 / 下架。
  - 仅**上架**内容出现在首页地图与正常详情访问路径中。

##### 字段：必填 / 选填（代拟，可改）

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | 是 | 展示名 |
| 类型 | 是 | 单选，对应配置 key |
| 经度、纬度 | 是 | 地图打点；坐标系 **GCJ-02**（与微信地图一致）。运营后台以 **地图选点为主**、输入框微调为辅；底图可用开源 OSM（展示 WGS84，保存前转换为 GCJ-02） |
| 图片 | 否 | 若有, 第一张可作为封面/分享封面, 若没有图片, 则不展示图片 |
| 介绍 | 是 | 详情正文；预览卡展示介绍（过长时截断，完整在详情） |
| 地址 | 否 | 文本地址；详情用于展示与调起导航. 若没有, 则不展示相关信息和功能 |
| 标签 | 否 | 多个，自由文本 |
| 营业时间 | 否 | 空则详情不展示该行 |
| 人均 | 否 | 空则不展示 |
| 电话 | 否 | 空则不展示 |
| 上架状态 | 是 | 上架 / 下架 |

> 若你调整必填策略，优先改本表，并同步详情展示规则。

#### 7.2.2 首页地图

| 项 | v1 行为 |
|----|---------|
| 数据 | 一次获取全部已上架百科 |
| 视野 | 根据全部点位自动缩放，保证都在可见范围（地图组件自动处理） |
| 点位样式 | 圆点，使用类型 `color` 区分（运行时按色绘制，分类扩展不依赖静态图标资源） |
| 筛选 | **不做**类型筛选 |
| 搜索 | **不做** |
| 路线 | **不做** |
| 点击点位 | 先出预览卡，再进详情 |

**预览卡内容**：名称 + 介绍（不强制封面；介绍过长截断）。

#### 7.2.3 百科详情

**始终展示（有数据的核心信息）**

- 名称
- 类型（名称，可用类型色点缀）
- 标签（有则展示）
- 介绍

**可选扩展（有值才展示）**

- 地址 + 打开地图/导航
- 封面 / 多图
- 营业时间、人均、电话等

**操作**

- 收藏 / 取消收藏（登录用户；游客点收藏走登录通用逻辑）
- 分享（标题 + 封面）

**浏览**

- 进入详情页即记一次浏览（本地规则见下）。

#### 7.2.4 登录与账号

**登录页元素**

- 头像
- 昵称
- 登录按钮
- 用户协议、隐私政策复选框（未勾选不可登录，或登录时校验并提示）

**触发**

- 「我的」提供登录按钮, 点击后进入登录页
- 点击需登录能力（如收藏）→ 弹窗确认 → 登录页

**成功后**

- 返回原页面
- **不自动执行**刚才被拦截的动作（不自动收藏）

**退出**

- 在「我的」用户信息区提供退出；退出后收藏入口隐藏，收藏态按未登录处理

#### 7.2.5 浏览记录（本地）

| 规则 | v1 |
|------|-----|
| 触发 | 进入百科详情即记录 |
| 存储 | 仅本地，不云端 |
| 游客 | 同样记录 |
| 去重 | 同一百科只保留一条，再次浏览则更新为最新并排到最前 |
| 排序 | 最近浏览在前 |
| 容量 | 最多 200 条，超出丢弃最旧 |
| 清空 | 支持一键清空 |
| 入口 | 「我的」→ 浏览记录（游客与登录都显示） |

列表点击：进入对应详情；若百科已下架或不可用，按失效提示处理（可从记录中仍看到历史名称等本地缓存信息，具体展示以实现简洁为准）。

#### 7.2.6 收藏（服务端）

| 规则 | v1 |
|------|-----|
| 前提 | 必须登录 |
| 存储 | 仅服务端 |
| 入口 | 详情收藏按钮；「我的」→ 我的收藏（仅登录可见） |
| 列表 | 展示收藏的百科；默认按收藏时间倒序（最近在前） |
| 取消收藏 | 正常情况在**详情页**取消 |
| 游客 | 无法收藏；我的页**不显示**收藏入口 |

#### 7.2.7 下架后的用户侧表现

| 场景 | 行为 |
|------|------|
| 首页地图 | 不展示该点位 |
| 正常浏览入口 | 不可当有效内容推广 |
| 收藏列表 | **仍显示**该条目；置灰或角标「已下架」 |
| 点击已下架收藏 | 提示不可用；**允许在列表取消收藏**，或进入失效详情页后取消收藏 |
| 失效详情（若进入） | 不展示完整运营内容亦可；至少提供明确「已下架」说明 + 取消收藏 |

说明：正常在架内容「取消收藏只在详情」；**已下架**时允许列表取消，避免用户无法清理失效收藏。

#### 7.2.8 「我的」页

| 模块 | 游客 | 已登录 |
|------|------|--------|
| 用户信息区 | 显示登录入口 | 头像、昵称、退出 |
| 浏览记录 | 有 | 有 |
| 我的收藏 | **无入口** | 有 |
| 设置/关于/协议 | 有 | 有 |

#### 7.2.9 运营后台（Web）

| 能力 | v1 |
|------|-----|
| 登录 | 简单账号密码，账号密码可配置 |
| 百科 CRUD | 创建、编辑、删除或等效管理 |
| 上下架 | 支持 |
| 图片 | 支持上传/维护多图 |
| 类型 | 使用静态配置（key/name/color），后台录入时单选类型 |
| 标签 | 自由输入多标签 |
| 位置录入 | **地图选点为主**（开源 OSM 瓦片），经纬度输入可微调；存库 GCJ-02 |
| 字段 | 与 7.2.1 表一致 |

后台以「能独立维护内容」为完成标准，不追求复杂权限与工作流。手动抄写经纬度几乎不可行，位置字段必须支持在地图上点选。

#### 7.2.10 权限矩阵（摘要）

| 能力 | 游客 | 登录用户 |
|------|------|----------|
| 看地图点位（上架） | ✓ | ✓ |
| 看预览卡 / 详情（上架） | ✓ | ✓ |
| 浏览记录读写（本地） | ✓ | ✓ |
| 清空浏览 | ✓ | ✓ |
| 收藏 / 取消收藏 | ✗（引导登录） | ✓ |
| 看「我的收藏」入口 | ✗ | ✓ |
| 分享详情 | ✓（按微信能力） | ✓ |

#### 7.2.11 v1 明确不做

- 用户发布内容、评论、点赞
- 搜索
- 类型筛选（仅颜色区分）
- 路线规划
- 多城市
- 消息推送
- 复杂分享海报玩法（详情分享卡片即可）

---

### 7.3 Technology（产品约束，不锁实现栈）

本 PRD **不指定**前端框架、后端语言或云厂商。实现时需满足以下产品级约束：

1. 运行于**微信小程序**（个人主体合规与能力范围需自行满足审核要求）。
2. 首页使用**地图**展示点位，并支持根据点位集合自动调整视野。
3. 登录能识别用户，以支撑**服务端收藏**。
4. 浏览记录仅存**客户端本地**。
5. 运营后台为 **Web**，支持账号密码登录与百科维护。
6. 类型配置为**静态配置文件**形态（key/name/color），与百科数据分离。

具体技术选型写入独立技术方案，不在此展开。

### 7.4 Assumptions（假设）

| 假设 | 若被推翻则影响 |
|------|----------------|
| v1 百科数量适合「全量拉取 + 全图展示」 | 需分页、聚合点或视野内加载 |
| 地图组件能按点位集合自动缩放到合适级别 | 需手写默认中心点与缩放兜底 |
| 用户能接受「登录后不自动续收藏」 | 需改为登录后自动完成动作 |
| 个人主体能力与审核允许地图、登录、分享等路径 | 需裁剪能力或调整主体 |
| 运营有精力维持大理内容质量 | 地图会空或价值不足 |
| 代拟必填表符合实际录入习惯 | 修改 7.2.1 字段表即可 |

## 8. Release

### 8.1 相对阶段（不设死日期）

| 阶段 | 目标 | 主要交付 |
|------|------|----------|
| 阶段 A | 可浏览 | 百科数据模型、首页地图全量打点、预览卡、详情、本地浏览、类型配置色 |
| 阶段 B | 可登录收藏 | 登录页与通用登录拦截、服务端收藏、我的收藏、详情取消收藏 |
| 阶段 C | 可运营上线 | Web 后台 CRUD/上下架/图、协议与关于、详情分享、发布与真实内容填充 |
| 阶段 D | 验收 | 对照 KR1–KR4 走通自己/朋友路径，修阻断问题 |

三阶段可并行部分工作，但对外发布至少完成 A+B+C 的闭环。

### 8.2 v1 内 / v1 后

| 属于 v1 | 放到以后 |
|---------|----------|
| 单城大理 | 多城市 |
| 地图全量点位 + 颜色区分 | 类型筛选、搜索、路线 |
| 运营录入百科 | UGC 发布/评论/点赞 |
| 本地浏览 + 云端收藏 | 浏览云同步、个性化推荐 |
| 详情分享卡片 | 海报、活动、推送 |
| 简易后台 | 复杂权限、审核流、数据分析大盘 |

### 8.3 依赖与风险

- 微信个人主体能力与类目限制
- 地图与定位相关合规文案（隐私政策）
- 首发内容数量与质量
- 图片存储与加载体验

### 8.4 验收清单（摘要）

- [ ] 首页能展示全部上架点位，视野可覆盖全部点
- [ ] 点位点出预览卡（名称+介绍），可进详情
- [ ] 详情字段与空值隐藏规则正确
- [ ] 进入详情写入本地浏览；去重、100 条上限、清空可用
- [ ] 游客无收藏入口；点收藏弹窗登录且成功后不自动收藏
- [ ] 登录用户可收藏/在详情取消；我的收藏列表可用
- [ ] 下架后地图消失；收藏列表置灰可取消
- [ ] 详情分享带标题与封面
- [ ] 后台可配置账号登录并完成 CRUD 与上下架
- [ ] 协议/隐私政策可访问
