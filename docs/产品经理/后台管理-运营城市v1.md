## 1. 结论摘要

| 维度 | v1 决策 |
|---|---|
| 端 | 仅 Web |
| 菜单 | 系统/运营配置 → 城市 |
| 权限 | 整站超级管理员；无分城权限 |
| 状态 | 草稿 / 启用 / 停用（可任意互转） |
| 默认运营城 | 全局恰好 1 个；必须为启用；设新默认自动取消旧默认 |
| 删除 | 不提供 |
| 页面 | 列表 + 新建/编辑同一表单 |
| 坐标 / 边界 / adcode / 城心 | 不做 |

## 2. 实体模型

| 字段 | 说明 |
|---|---|
| cityId | 系统生成，稳定主键；业务关联用 |
| 名称 | 必填；≤20 字；全局唯一；可改 |
| 介绍 | 可选；纯文本；≤500 字；预留扩展 |
| 状态 | 草稿 / 启用 / 停用 |
| 是否默认运营城 | 布尔；全库最多一个为 true |
| 排序 | 整数；新建默认 0 |
| 创建时间 / 更新时间 | 审计 |
| 创建人 / 更新人 | 超级管理员 |

---

## 3. 业务规则

### 3.1 新建

- 默认状态：草稿  
- 默认排序：0  
- 默认非默认运营城  

### 3.2 状态

- 草稿 / 启用 / 停用可任意互转（受默认城约束）  
- 不可删除；误配用草稿或停用  

### 3.3 默认运营城

- 全局恰好 1 个默认城（正式环境验收）  
- 设为默认：目标必须已是启用；成功后原默认自动取消  
- 禁止停用当前默认城；须先将默认改到其他启用城  
- 配置期可暂时 0 个启用城；设默认或把默认城改为非启用时拦截并提示  
- 正式环境应存在恰好 1 个「启用且默认」的城市（后台不自动造城）  

### 3.4 列表排序

- 主序：排序升序  
- 同序：更新时间倒序  

### 3.5 名称

- 全库唯一；冲突保存失败并提示  

### 3.6 并发

- 后写覆盖；无乐观锁  

### 3.7 操作日志

记录：创建、编辑、状态变更、设默认（含旧默认取消）；含操作人、对象、前后关键字段、时间。

---

## 4. 页面规格

### 4.1 列表

- 搜索：名称  
- 筛选：状态  
- 分页  
- 展示：名称、状态、是否默认、排序、更新时间等  
- 行内：改状态、设默认、进入编辑  
- 空态：文案 + 新建入口  
- 不做：批量操作、拖拽排序  

### 4.2 新建 / 编辑表单

- 字段：名称、介绍、状态、是否默认、排序  
- 校验：名称必填/长度/唯一；介绍长度；设默认须启用；对当前默认城停用或改非启用 → 禁止并提示先换默认  

### 4.3 权限

- 菜单：系统/运营配置 → 城市  
- 超管可管；无模块级开关  

---

## 5. 与百科

- 业务通过 `cityId` 关联；改名不影响关联  
- 供百科等下拉的可选城：状态 = 启用  
- 城市停用不自动改百科状态；已绑数据保留 cityId  
- 细则见 [后台管理-百科v1.md](./后台管理-百科v1.md)

## 6. 用例

| ID | 名称 | 前置 | 主成功路径 | 异常 | 后置 |
|---|---|---|---|---|---|
| CITY-UC-01 | 新建草稿 | 超管登录 | 新建 → 填名称 → 存草稿 | 重名失败 | 列表可见草稿 |
| CITY-UC-02 | 启用并设默认 | 存在城 | 启用 → 设默认 | 非启用不可设默认 | 该城为唯一默认 |
| CITY-UC-03 | 切换默认 | 默认 A，启用 B | 对 B 设默认 | — | A 取消，B 为默认 |
| CITY-UC-04 | 停用默认被拒 | 当前为默认 | 尝试停用 | 提示先换默认 | 状态不变 |
| CITY-UC-05 | 改名 | 已有城 | 编辑名称保存 | 重名失败 | cityId 不变 |
| CITY-UC-06 | 列表筛选 | 多状态数据 | 状态筛 / 名称搜 | 无结果空态 | — |

---

## 7. 验收

- 新建默认为草稿、排序 0、非默认  
- 启用后可设默认；切换默认后全局仍仅一个默认  
- 停用当前默认城失败并提示  
- 名称超长 / 重名失败  
- 列表排序：排序升序，同序更新时间倒序  
- 关键写操作可在操作日志追溯  
- 正式环境：恰好 1 个启用且默认的城市  

---

## 8. v1 不做

- 删除  
- 城心 / 半径 / 边界 / adcode  
- 批量导入  
- 拖拽排序、批量改状态  
- 多级区域  
- 员工分城权限
