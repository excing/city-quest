## 1. 结论摘要

| 维度 | v1 决策 |
|---|---|
| 端 | 仅 Web |
| 菜单 | 系统/运营配置 → 城市 |
| 权限 | 整站超级管理员；无分城权限 |
| 状态 | 草稿 / 启用 / 停用（可任意互转） |
| 默认运营城 | 全局恰好 1 个；必须为启用；设新默认自动取消旧默认 |
| adcode | 配置；用于逆地理匹配运营城（见 §3.8） |
| 删除 | 不提供 |
| 页面 | 列表 + 新建/编辑同一表单 |
| 城心 / 半径 / 边界 | 不做；切城后地图视野由客户端按该城点位集合适配（见 §5.2） |

## 2. 实体模型

| 字段 | 说明 |
|---|---|
| cityId | 系统生成，稳定主键；业务关联用 |
| 名称 | 必填；≤20 字；全局唯一；可改；展示用，**不作** GPS 匹配主键 |
| adcode | 国标行政区划代码；字符串；用于逆地理结果匹配运营城；规则见 §3.8 |
| 介绍 | 可选；纯文本；≤500 字；预留扩展 |
| 状态 | 草稿 / 启用 / 停用 |
| 是否默认运营城 | 布尔；全库最多一个为 true |
| 排序 | 整数；新建默认 0 |
| 创建时间 / 更新时间 | 审计 |
| 创建人 / 更新人 | 超级管理员 |

## 3. 业务规则

### 3.1 新建

- 默认状态：草稿  
- 默认排序：0  
- 默认非默认运营城  
- adcode：可填；**启用前必须有效**（见 §3.8）  

### 3.2 状态

- 草稿 / 启用 / 停用可任意互转（受默认城与 adcode 约束）  
- 不可删除；误配用草稿或停用  
- **设为启用**：须已填有效 adcode；否则拦截并提示  

### 3.3 默认运营城

- 全局恰好 1 个默认城（正式环境验收）  
- 设为默认：目标必须已是启用；成功后原默认自动取消  
- 禁止停用当前默认城；须先将默认改到其他启用城  
- 配置期可暂时 0 个启用城；设默认或把默认城改为非启用时拦截并提示  
- 正式环境应存在恰好 1 个「启用且默认」的城市（后台不自动造城）  

### 3.4 列表排序

- 主序：排序升序  
- 同序：更新时间倒序  

### 3.5 名称

- 全库唯一；冲突保存失败并提示  
- 改名不影响 `cityId` 与 adcode 关联的匹配逻辑  

### 3.6 并发

- 后写覆盖；无乐观锁  

### 3.7 操作日志

记录：创建、编辑、状态变更、设默认（含旧默认取消）；含操作人、对象、前后关键字段（含 adcode）、时间。

### 3.8 adcode（逆地理匹配）

- **用途**：用户端 GPS → 逆地理编码得到行政区 adcode → 与启用运营城的 adcode 对齐，决定当前运营城；匹配失败 / 当前城未开通 → 默认运营城（用户端细则见导航规划）  
- **粒度**：v1 运营城按**地级市（或直辖市）**配置 adcode（如杭州 `330100`、上海 `310000`）；不按区县拆运营城  
- **格式**：6 位数字字符串（允许前导零以字符串存储）；非法格式保存失败  
- **唯一性**：全库 adcode 唯一（含草稿/停用，避免启用时冲突）；冲突保存失败并提示  
- **启用约束**：状态为启用时 adcode 必填且合法；草稿可暂时为空  
- **匹配归一（实现约定，非后台字段）**：逆地理若返回区县级 adcode，服务端/客户端归一到市级后再与运营城 adcode 比较（如区码映射到所属地级市码）；具体映射表由研发实现，不在本后台维护边界多边形  
- **名称不参与匹配**：展示与搜索用名称；GPS 落城以 adcode 为准

## 4. 页面规格

### 4.1 列表

- 搜索：名称；可选按 adcode 精确搜  
- 筛选：状态  
- 分页  
- 展示：名称、adcode、状态、是否默认、排序、更新时间等  
- 行内：改状态、设默认、进入编辑  
- 空态：文案 + 新建入口  
- 不做：批量操作、拖拽排序  

### 4.2 新建 / 编辑表单

- 字段：名称、adcode、介绍、状态、是否默认、排序  
- 校验：  
  - 名称必填/长度/唯一  
  - adcode：启用时必填；格式 6 位数字；全库唯一  
  - 介绍长度  
  - 设默认须启用  
  - 对当前默认城停用或改非启用 → 禁止并提示先换默认  

### 4.3 权限

- 菜单：系统/运营配置 → 城市  
- 超管可管；无模块级开关

## 5. 与其他模块

### 5.1 百科等业务

- 业务通过 `cityId` 关联；改名 / 改 adcode 不影响已有 `cityId` 绑定（改 adcode 只影响后续 GPS 匹配落点）  
- 供百科等下拉的可选城：状态 = 启用  
- 城市停用不自动改百科状态；已绑数据保留 cityId  
- 细则见 [后台管理-百科v1.md](./后台管理-百科v1.md)  

### 5.2 用户端城市与地图视野

> 用户端完整交互以 [首页与全局导航规划v1.md](./首页与全局导航规划v1.md) 为准；本节只约定与主数据的分工。

| 能力 | 主数据 / 服务 | 客户端 |
|---|---|---|
| 已开通城市列表 | 状态 = 启用；返回 cityId、名称、adcode、是否默认、排序 | 城市切换 UI |
| GPS 匹配运营城 | 启用城 adcode 集合；逆地理 + 市级归一后匹配 | 触发定位；匹配失败用默认城 |
| 默认运营城 | 唯一默认且启用 | 失败/拒权/未开通兜底 |
| 切城 / 刷新点位后的地图镜头 | **不**提供城心坐标 | 以该城**当前可见业务点位**（百科/宝箱等 markers）用 map `include-points` / `includePoints` 适配可见区域；**不是**单一城心点 |
| 空城（该城无点位） | — | 地图可滑动；中央轻提示筹备中；不依赖城心 |

## 6. 用例

| ID | 名称 | 前置 | 主成功路径 | 异常 | 后置 |

## 7. 验收

- 新建默认为草稿、排序 0、非默认；adcode 可空  
- 无有效 adcode 不可启用；启用后可设默认；切换默认后全局仍仅一个默认  
- 停用当前默认城失败并提示  
- 名称超长 / 重名失败  
- adcode 非法格式 / 与其他城冲突失败  
- 列表排序：排序升序，同序更新时间倒序  
- 关键写操作可在操作日志追溯（含 adcode 变更）  
- 正式环境：恰好 1 个启用且默认的城市，且该城（及所有启用城）具备合法唯一 adcode  

---

## 8. v1 不做

- 删除  
- 城心坐标 / 半径 / 边界多边形（地图可见区域由客户端按点位 `includePoints` 负责）  
- 批量导入  
- 拖拽排序、批量改状态  
- 多级区域（省-市-区运营树）  
- 员工分城权限  
- 后台维护逆地理区→市映射表 UI（由研发侧实现约定）
